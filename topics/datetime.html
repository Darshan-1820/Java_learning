Section 01

Old vs New: Why Java 8 Changed Everything

Java's original date/time classes (java.util.Date, java.util.Calendar) are universally considered one of the worst APIs in the language. Java 8 introduced java.time.* — an entirely new package that is clean, immutable, and actually makes sense.

What Was Wrong With the Old API

OldDateProblems.java 
// Problem 1: Months start at 0 (January = 0, not 1)
Date date = new Date(2026, 3, 23);  // This is actually April 3926!
// Year is offset from 1900, month is 0-indexed. Insane.

// Problem 2: Date is mutable — anyone can change it
Date bookingDate = booking.getDate();
bookingDate.setTime(0);  // Just mutated the booking's internal date!

// Problem 3: Date contains time, Calendar is verbose
Calendar cal = Calendar.getInstance();
cal.set(Calendar.YEAR, 2026);
cal.set(Calendar.MONTH, Calendar.APRIL);  // Why not just 4?
cal.set(Calendar.DAY_OF_MONTH, 23);
Date d = cal.getTime();  // 4 lines for one date

The New API: Clean and Immutable

NewDateAPI.java 
// One line. Month is 4 for April. Year is 2026. Done.
LocalDate date = LocalDate.of(2026, 4, 23);

// Immutable — plusDays returns a NEW object, never modifies the original
LocalDate tomorrow = date.plusDays(1);  // date is still April 23

// Clean, readable, no surprises
LocalDateTime now = LocalDateTime.now();
System.out.println(now);  // 2026-04-23T14:30:00
Feature Old (java.util.Date) New (java.time.*)
Months 0-indexed (Jan = 0) 1-indexed (Jan = 1)
Mutability Mutable (dangerous) Immutable (safe)
Thread safety Not thread-safe Thread-safe
API design Confusing, deprecated methods Clean, fluent methods
Date vs Time Date includes time always Separate: LocalDate, LocalTime, LocalDateTime
Rule

Never use java.util.Date or java.util.Calendar in new code. Always use java.time.*. The old classes exist only for backward compatibility with legacy code. Nimbus uses the new API exclusively.

The java.time Family

                      java.time.*
                          |
          ┌───────────────┼───────────────┐
          |               |               |
      LocalDate      LocalTime    LocalDateTime
    (date only)    (time only)   (date + time)
    2026-04-23      14:30:00     2026-04-23T14:30:00
          |               |               |
          └───────────────┼───────────────┘
                          |
                  ZonedDateTime
              (date + time + timezone)
          2026-04-23T14:30:00+05:30[Asia/Kolkata]
The core classes of java.time — each handles a different level of detail
Section 02

LocalDate — Date Without Time

LocalDate represents a calendar date — year, month, day — with no time and no timezone. Use it when you only care about the date: birthdays, booking dates, due dates.

Creating a LocalDate

LocalDateCreation.java 
// Today's date
LocalDate today = LocalDate.now();              // 2026-04-23

// Specific date
LocalDate launch = LocalDate.of(2026, 4, 23);   // 2026-04-23

// Using Month enum (more readable)
LocalDate launch2 = LocalDate.of(2026, Month.APRIL, 23);

// Parse from string
LocalDate parsed = LocalDate.parse("2026-04-23");  // ISO format: yyyy-MM-dd

Common Operations

LocalDateOps.java 
LocalDate today = LocalDate.of(2026, 4, 23);

// Adding and subtracting — returns NEW object, never modifies original
LocalDate tomorrow   = today.plusDays(1);        // 2026-04-24
LocalDate nextWeek   = today.plusWeeks(1);       // 2026-04-30
LocalDate nextMonth  = today.plusMonths(1);      // 2026-05-23
LocalDate lastYear   = today.minusYears(1);      // 2025-04-23

// Extracting parts
int year      = today.getYear();                 // 2026
Month month   = today.getMonth();                // APRIL
int monthVal  = today.getMonthValue();            // 4
int day       = today.getDayOfMonth();            // 23
DayOfWeek dow = today.getDayOfWeek();             // THURSDAY
int dayOfYear = today.getDayOfYear();             // 113

// Comparisons
boolean isBefore = today.isBefore(tomorrow);     // true
boolean isAfter  = today.isAfter(lastYear);      // true
boolean isEqual  = today.isEqual(today);          // true
boolean isLeap   = today.isLeapYear();            // false
Key methods explained
1
today.plusDays(1)
Returns a NEW LocalDate one day later. today is NOT modified — LocalDate is immutable. Every "modifier" method returns a new object. This is a critical difference from the old Date class.
2
today.getDayOfWeek()
Returns a DayOfWeek enum (MONDAY through SUNDAY), not a number. Much more readable than the old Calendar.DAY_OF_WEEK which returned integers where Sunday was 1.
3
today.isBefore(tomorrow)
Reads like English. Compare this to the old API where you had to call date1.compareTo(date2) < 0 or date1.getTime() < date2.getTime().
In Nimbus

When a user books a parcel, Nimbus records the booking date as LocalDate.now(). To calculate estimated delivery (e.g., 3 business days), you would use bookingDate.plusDays(3) and check if the result falls on a weekend using getDayOfWeek().

Section 03

LocalTime — Time Without Date

LocalTime represents a time of day — hours, minutes, seconds, nanoseconds — with no date and no timezone. Use it for opening hours, shift schedules, or time-only comparisons.

LocalTimeBasics.java 
// Current time
LocalTime now = LocalTime.now();               // 14:30:15.123456

// Specific time
LocalTime pickup = LocalTime.of(10, 30);       // 10:30
LocalTime precise = LocalTime.of(10, 30, 45);  // 10:30:45

// Parse from string
LocalTime parsed = LocalTime.parse("10:30:00");

// Operations
LocalTime later = pickup.plusHours(2);          // 12:30
LocalTime earlier = pickup.minusMinutes(30);    // 10:00

// Extract parts
int hour   = pickup.getHour();    // 10
int minute = pickup.getMinute();  // 30

// Comparisons
boolean isOpen = now.isAfter(LocalTime.of(9, 0))
              && now.isBefore(LocalTime.of(18, 0));
// true if current time is between 9 AM and 6 PM
In Nimbus

Nimbus could use LocalTime to enforce pickup windows: "Parcels can only be picked up between 9:00 AM and 6:00 PM." A simple isAfter/isBefore check on the requested pickup time handles this validation.

Section 04

LocalDateTime — Date + Time

LocalDateTime combines date and time into one object. This is the most commonly used class in business applications — booking timestamps, payment times, scheduling.

Creating and Using

LocalDateTimeBasics.java 
// Current date and time
LocalDateTime now = LocalDateTime.now();
// 2026-04-23T14:30:15.123

// Specific date and time
LocalDateTime booking = LocalDateTime.of(2026, 4, 23, 10, 30);
// 2026-04-23T10:30

// From separate LocalDate and LocalTime
LocalDate date = LocalDate.of(2026, 4, 23);
LocalTime time = LocalTime.of(10, 30);
LocalDateTime combined = LocalDateTime.of(date, time);
// 2026-04-23T10:30

// Extract date or time part
LocalDate datePart = now.toLocalDate();   // 2026-04-23
LocalTime timePart = now.toLocalTime();   // 14:30:15.123

// All the same operations work
LocalDateTime tomorrow = now.plusDays(1);
LocalDateTime laterToday = now.plusHours(3);
int hour = now.getHour();
DayOfWeek day = now.getDayOfWeek();

Nimbus Booking Example

BookingTimestamps.java 
public class Booking {
    private LocalDateTime pickupTime;
    private LocalDateTime dropoffTime;
    private LocalDateTime paymentTime;

    public void schedulePickup(LocalDateTime requestedTime) {
        // Validate: pickup must be in the future
        if (requestedTime.isBefore(LocalDateTime.now())) {
            throw new IllegalArgumentException("Pickup time cannot be in the past");
        }

        // Validate: pickup must be during business hours (9 AM - 6 PM)
        LocalTime timeOfDay = requestedTime.toLocalTime();
        if (timeOfDay.isBefore(LocalTime.of(9, 0)) ||
            timeOfDay.isAfter(LocalTime.of(18, 0))) {
            throw new IllegalArgumentException("Pickup must be between 9 AM and 6 PM");
        }

        this.pickupTime = requestedTime;

        // Auto-calculate estimated dropoff (pickup + 2 days)
        this.dropoffTime = requestedTime.plusDays(2);
    }

    public void recordPayment() {
        this.paymentTime = LocalDateTime.now();
    }
}
Line-by-line explanation
1
requestedTime.isBefore(LocalDateTime.now())
Compares the requested pickup with the current moment. If the user tries to schedule a pickup for yesterday, this catches it. Reads like English: "is the requested time before now?"
2
requestedTime.toLocalTime()
Extracts just the time part from the datetime. We only care about the hour for business-hours validation, not the date.
3
requestedTime.plusDays(2)
Estimated delivery is 2 days after pickup. This returns a new LocalDateTime — the original is untouched. Immutability means you can pass requestedTime around without worrying someone will modify it.
In Nimbus

LocalDateTime is the backbone of Nimbus's time tracking. Every booking has a pickupTime, dropoffTime, and paymentTime. The scheduling logic, delivery window calculations, and payment timestamps all use LocalDateTime. It replaced what would have been dozens of lines of Calendar manipulation with clean, readable one-liners.

Section 05

DateTimeFormatter

DateTimeFormatter converts dates to strings (formatting) and strings to dates (parsing). It controls how dates appear to users — "2026-04-23" vs "April 23, 2026" vs "23/04/2026".

Formatting: Date to String

Formatting.java 
LocalDateTime now = LocalDateTime.of(2026, 4, 23, 14, 30, 0);

// Built-in formatters
String iso = now.format(DateTimeFormatter.ISO_LOCAL_DATE_TIME);
// "2026-04-23T14:30:00"

// Custom patterns
DateTimeFormatter custom1 = DateTimeFormatter.ofPattern("dd/MM/yyyy");
String formatted1 = now.format(custom1);
// "23/04/2026"

DateTimeFormatter custom2 = DateTimeFormatter.ofPattern("dd MMM yyyy, hh:mm a");
String formatted2 = now.format(custom2);
// "23 Apr 2026, 02:30 PM"

DateTimeFormatter custom3 = DateTimeFormatter.ofPattern("EEEE, MMMM d, yyyy");
String formatted3 = now.format(custom3);
// "Thursday, April 23, 2026"

DateTimeFormatter custom4 = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm");
String formatted4 = now.format(custom4);
// "2026-04-23 14:30"

Common Pattern Letters

Letter Meaning Example
yyyy Year (4 digits) 2026
MM Month (2 digits) 04
MMM Month (abbreviated) Apr
MMMM Month (full) April
dd Day (2 digits) 23
EEEE Day of week (full) Thursday
HH Hour (24-hour, 00-23) 14
hh Hour (12-hour, 01-12) 02
mm Minutes 30
ss Seconds 45
a AM/PM PM

Parsing: String to Date

Parsing.java 
// Parse ISO format (default)
LocalDate date = LocalDate.parse("2026-04-23");

// Parse custom format — the pattern must match the input EXACTLY
DateTimeFormatter indian = DateTimeFormatter.ofPattern("dd/MM/yyyy");
LocalDate parsed = LocalDate.parse("23/04/2026", indian);

// Parse datetime
DateTimeFormatter dtFormat = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm");
LocalDateTime dt = LocalDateTime.parse("2026-04-23 14:30", dtFormat);

// What happens if the format doesn't match?
LocalDate bad = LocalDate.parse("23-04-2026");
// DateTimeParseException! Default format is yyyy-MM-dd, not dd-MM-yyyy
Watch Out

Parsing is strict. The string must exactly match the pattern. "23/04/2026" will not parse with pattern "yyyy-MM-dd". If the user provides input in a different format than expected, you get a DateTimeParseException. Always wrap parsing in a try-catch when dealing with user input.

In Nimbus

When displaying booking history to users, Nimbus formats dates as "dd MMM yyyy, hh:mm a" (e.g., "23 Apr 2026, 02:30 PM") for readability. When storing dates in the database, it uses ISO format "yyyy-MM-dd HH:mm:ss". Two different formatters for two different audiences.

Section 06

Duration and Period

Duration measures time-based amounts (hours, minutes, seconds). Period measures date-based amounts (years, months, days). They answer the question: "How much time between A and B?"

Duration: For Time

DurationExample.java 
// How long between two times?
LocalTime start = LocalTime.of(10, 0);
LocalTime end = LocalTime.of(14, 30);

Duration gap = Duration.between(start, end);
System.out.println(gap.toHours());     // 4
System.out.println(gap.toMinutes());   // 270

// Create durations directly
Duration twoHours = Duration.ofHours(2);
Duration thirtyMins = Duration.ofMinutes(30);

// Use with LocalDateTime
LocalDateTime pickup = LocalDateTime.of(2026, 4, 23, 10, 0);
LocalDateTime delivery = pickup.plus(Duration.ofHours(48));
// 2026-04-25T10:00

Period: For Dates

PeriodExample.java 
// How long between two dates?
LocalDate start = LocalDate.of(2026, 1, 1);
LocalDate end = LocalDate.of(2026, 4, 23);

Period gap = Period.between(start, end);
System.out.println(gap.getMonths());  // 3
System.out.println(gap.getDays());    // 22
System.out.println(gap);              // P3M22D (3 months, 22 days)

// Create periods directly
Period oneMonth = Period.ofMonths(1);
Period twoWeeks = Period.ofWeeks(2);  // stored as 14 days

// Use with LocalDate
LocalDate today = LocalDate.of(2026, 4, 23);
LocalDate renewal = today.plus(Period.ofMonths(6));
// 2026-10-23
Feature Duration Period
Measures Hours, minutes, seconds, nanos Years, months, days
Use with LocalTime, LocalDateTime LocalDate, LocalDateTime
Example "2 hours 30 minutes" "3 months 22 days"
ISO format PT2H30M P3M22D
In Nimbus

Duration is perfect for calculating delivery windows: "this parcel took 48 hours from pickup to delivery." Period is useful for subscription or billing cycles: "user's plan renews every 1 month." Both give you precise control over time arithmetic without manual calculations.

Section 07

Timestamps and JDBC

Databases store datetime values as TIMESTAMP. JDBC (Java's database connectivity) uses java.sql.Timestamp. You need to convert between java.sql.Timestamp and java.time.LocalDateTime when reading from and writing to the database.

Writing to Database: LocalDateTime to Timestamp

WriteTimestamp.java 
// Convert LocalDateTime → java.sql.Timestamp for database
LocalDateTime pickupTime = LocalDateTime.of(2026, 4, 23, 10, 30);

// Using PreparedStatement
String sql = "INSERT INTO bookings (tracking_id, pickup_time) VALUES (?, ?)";
PreparedStatement ps = conn.prepareStatement(sql);
ps.setString(1, "NIM-001");
ps.setTimestamp(2, Timestamp.valueOf(pickupTime));  // LocalDateTime → Timestamp
ps.executeUpdate();

Reading from Database: Timestamp to LocalDateTime

ReadTimestamp.java 
// Read from ResultSet and convert to LocalDateTime
String sql = "SELECT tracking_id, pickup_time, dropoff_time FROM bookings";
ResultSet rs = stmt.executeQuery(sql);

while (rs.next()) {
    String trackingId = rs.getString("tracking_id");

    // Timestamp → LocalDateTime
    Timestamp ts = rs.getTimestamp("pickup_time");
    LocalDateTime pickupTime = ts.toLocalDateTime();

    // Or in one line
    LocalDateTime dropoffTime = rs.getTimestamp("dropoff_time").toLocalDateTime();

    System.out.println(trackingId + " pickup: " + pickupTime);
}
The two key conversions
1
Timestamp.valueOf(localDateTime)
LocalDateTime → Timestamp. Static method on Timestamp. Use when WRITING to the database. The PreparedStatement.setTimestamp() method requires a Timestamp object.
2
timestamp.toLocalDateTime()
Timestamp → LocalDateTime. Instance method on Timestamp. Use when READING from the database. The ResultSet.getTimestamp() returns a Timestamp, and you convert it to the modern API.

For LocalDate and DATE columns

DateColumn.java 
// Writing LocalDate → java.sql.Date
LocalDate bookingDate = LocalDate.now();
ps.setDate(1, java.sql.Date.valueOf(bookingDate));

// Reading java.sql.Date → LocalDate
java.sql.Date sqlDate = rs.getDate("booking_date");
LocalDate date = sqlDate.toLocalDate();
In Nimbus

In BookingDAO, every time you read a booking from the database, you convert Timestamp to LocalDateTime. Every time you save a booking, you convert LocalDateTime to Timestamp. These two methods — Timestamp.valueOf() and .toLocalDateTime() — are the bridge between the modern API and the database.

ScheduleParser.java 
public LocalDateTime parsePickupInput(String userInput) {
    // User enters: "23/04/2026 14:30"
    DateTimeFormatter inputFormat =
        DateTimeFormatter.ofPattern("dd/MM/yyyy HH:mm");

    try {
        LocalDateTime requested = LocalDateTime.parse(userInput, inputFormat);

        // Validate
        if (requested.isBefore(LocalDateTime.now())) {
            System.out.println("Error: Cannot schedule in the past.");
            return null;

        return requested;

        return null;
    }
}

Calculating Delivery Windows

DeliveryCalculator.java 
public String estimateDelivery(LocalDateTime pickupTime, String serviceType) {
    LocalDateTime estimated;

    switch (serviceType) {
        case "EXPRESS":
            estimated = pickupTime.plusHours(24);
            break;
        case "STANDARD":
            estimated = pickupTime.plusDays(3);
            break;
        case "ECONOMY":
            estimated = pickupTime.plusDays(7);
            break;
        default:
            estimated = pickupTime.plusDays(5);
    }

    // Skip weekends
    while (estimated.getDayOfWeek() == DayOfWeek.SATURDAY ||
           estimated.getDayOfWeek() == DayOfWeek.SUNDAY) {
        estimated = estimated.plusDays(1);
    }

    DateTimeFormatter fmt = DateTimeFormatter.ofPattern("EEEE, dd MMM yyyy");
    return "Estimated delivery: " + estimated.format(fmt);
    // "Estimated delivery: Monday, 27 Apr 2026"
}

Section 09

Timezone Basics

Nimbus operates in India only, so timezones are not critical. But understanding the basics is important for interviews and future projects that span multiple timezones.

ZonedDateTime

TimezoneBasics.java 
// Current time in India
ZonedDateTime indiaTime = ZonedDateTime.now(ZoneId.of("Asia/Kolkata"));
// 2026-04-23T14:30:00+05:30[Asia/Kolkata]

// Same moment in New York
ZonedDateTime nyTime = indiaTime.withZoneSameInstant(ZoneId.of("America/New_York"));
// 2026-04-23T05:00:00-04:00[America/New_York]

// Convert LocalDateTime to ZonedDateTime
LocalDateTime local = LocalDateTime.of(2026, 4, 23, 14, 30);
ZonedDateTime zoned = local.atZone(ZoneId.of("Asia/Kolkata"));

// Convert ZonedDateTime to LocalDateTime (drops timezone info)
LocalDateTime backToLocal = zoned.toLocalDateTime();

// List all available zone IDs
Set<String> zones = ZoneId.getAvailableZoneIds();
// Contains "Asia/Kolkata", "America/New_York", "Europe/London", etc.

When to Use What

Class When to Use Nimbus Example
LocalDate Dates only (no time needed) Booking date, user registration date
LocalTime Time only (no date needed) Business hours check, shift schedules
LocalDateTime Date + time, single timezone Pickup time, dropoff time, payment time
ZonedDateTime Date + time, multiple timezones International parcel tracking (future)
Key Insight

LocalDateTime does NOT store timezone information. It is "local" to wherever the code runs. For Nimbus running on a server in India, LocalDateTime.now() returns IST. If the same server moved to the US, it would return EST. For single-timezone applications like Nimbus, this is fine. For multi-timezone applications, use ZonedDateTime or store in UTC and convert on display.

Section 10

Common Mistakes

Date/time code is a minefield. These are the mistakes that catch everyone.

Mistake 1: Thinking plusDays() Modifies the Original

Wrong
LocalDate date = LocalDate.now();
date.plusDays(5);  // Returns new object
// date is UNCHANGED! The result
// was thrown away.
Right
LocalDate date = LocalDate.now();
LocalDate future = date.plusDays(5);
// Assign the result to a variable

Mistake 2: Using java.util.Date in New Code

Wrong
Date now = new Date();
// Mutable, confusing API,
// months start at 0
Right
LocalDateTime now = LocalDateTime.now();
// Immutable, clean API,
// months start at 1

Mistake 3: Wrong Pattern Letters

Wrong
// mm is minutes, not months!
DateTimeFormatter f =
    DateTimeFormatter.ofPattern(
        "dd/mm/yyyy"
    );
// Formats month as minutes = wrong
Right
// MM is months (capital M)
DateTimeFormatter f =
    DateTimeFormatter.ofPattern(
        "dd/MM/yyyy"
    );
// 23/04/2026 — correct

Mistake 4: Parsing Without Error Handling

Wrong
// User input can be anything
String input = scanner.nextLine();
LocalDate date =
    LocalDate.parse(input);
// DateTimeParseException if
// input is "hello"
Right
String input = scanner.nextLine();
try {
    LocalDate date =
        LocalDate.parse(input);
} catch (DateTimeParseException e) {
    System.out.println("Invalid date");
}

Mistake 5: Comparing with == Instead of Methods

Wrong
LocalDate a = LocalDate.of(2026, 4, 23);
LocalDate b = LocalDate.of(2026, 4, 23);
if (a == b) {}  // false!
// == compares references, not values
Right
LocalDate a = LocalDate.of(2026, 4, 23);
LocalDate b = LocalDate.of(2026, 4, 23);
if (a.isEqual(b)) {}  // true
// Or a.equals(b) — also true
Section 11

Practice Exercises

Write these yourself. Each exercise uses a different part of the Date/Time API.

Exercise 1: Days Until Birthday

Ask the user for their birthday (as "dd/MM/yyyy"). Calculate how many days until their next birthday. If the birthday has already passed this year, calculate days until next year's birthday.

Solution
1
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("dd/MM/yyyy");
LocalDate birthday = LocalDate.parse("15/08/2000", fmt);
LocalDate today = LocalDate.now();

// This year's birthday
LocalDate nextBirthday = birthday.withYear(today.getYear());

// If it already passed, use next year
if (nextBirthday.isBefore(today) || nextBirthday.isEqual(today)) {
    nextBirthday = nextBirthday.plusYears(1);
}

long daysUntil = ChronoUnit.DAYS.between(today, nextBirthday);
System.out.println("Days until birthday: " + daysUntil);
birthday.withYear(today.getYear()) creates a copy of the birthday but in the current year. ChronoUnit.DAYS.between() gives the exact number of days between two dates.

Exercise 2: Booking Receipt Formatter

Given a LocalDateTime, format it in three different ways: ISO ("2026-04-23T14:30:00"), Indian ("23/04/2026 02:30 PM"), and friendly ("Thursday, 23 April 2026 at 2:30 PM"). Create a method that takes a LocalDateTime and a format name ("iso", "indian", "friendly") and returns the formatted string.

Solution
1
public static String formatBookingTime(LocalDateTime dt, String style) {
    switch (style) {
        case "iso":
            return dt.format(DateTimeFormatter.ISO_LOCAL_DATE_TIME);
        case "indian":
            return dt.format(DateTimeFormatter.ofPattern("dd/MM/yyyy hh:mm a"));
        case "friendly":
            return dt.format(DateTimeFormatter.ofPattern("EEEE, dd MMMM yyyy 'at' h:mm a"));
        default:
            return dt.toString();
    }
}

LocalDateTime booking = LocalDateTime.of(2026, 4, 23, 14, 30);
System.out.println(formatBookingTime(booking, "iso"));
// 2026-04-23T14:30:00
System.out.println(formatBookingTime(booking, "indian"));
// 23/04/2026 02:30 PM
System.out.println(formatBookingTime(booking, "friendly"));
// Thursday, 23 April 2026 at 2:30 PM
The single quotes around 'at' in the pattern mean literal text — they are not parsed as date tokens. EEEE gives full day name, MMMM gives full month name.

Exercise 3: Delivery Time Calculator

Write a method that takes a pickup LocalDateTime and a service type ("EXPRESS", "STANDARD", "ECONOMY") and returns the estimated delivery LocalDateTime. EXPRESS = +24 hours, STANDARD = +3 days, ECONOMY = +7 days. Skip weekends (if the estimated date falls on Saturday or Sunday, push to Monday).

Solution
1
public static LocalDateTime estimateDelivery(LocalDateTime pickup, String service) {
    LocalDateTime estimated;

    switch (service) {
        case "EXPRESS":  estimated = pickup.plusHours(24); break;
        case "STANDARD": estimated = pickup.plusDays(3);   break;
        case "ECONOMY":  estimated = pickup.plusDays(7);   break;
        default:         estimated = pickup.plusDays(5);   break;
    }

    // Skip weekends
    while (estimated.getDayOfWeek() == DayOfWeek.SATURDAY ||
           estimated.getDayOfWeek() == DayOfWeek.SUNDAY) {
        estimated = estimated.plusDays(1);
    }

    return estimated;
}

LocalDateTime pickup = LocalDateTime.of(2026, 4, 23, 10, 0); // Thursday
System.out.println(estimateDelivery(pickup, "STANDARD"));

public static String calculateAge(LocalDate birthdate) {

Exercise 5: JDBC Timestamp Round Trip

Simulate a database round trip: create a LocalDateTime, convert it to java.sql.Timestamp, then convert it back. Print all three to verify they match. This tests the two conversion methods you will use in every DAO.

Solution
1
import java.sql.Timestamp;
import java.time.LocalDateTime;

public class TimestampRoundTrip {
    public static void main(String[] args) {
        // Step 1: Create LocalDateTime
        LocalDateTime original = LocalDateTime.of(2026, 4, 23, 14, 30, 0);
        System.out.println("Original:  " + original);
        // 2026-04-23T14:30

        // Step 2: Convert to Timestamp (what you send to DB)
        Timestamp ts = Timestamp.valueOf(original);
        System.out.println("Timestamp: " + ts);
        // 2026-04-23 14:30:00.0

        // Step 3: Convert back (what you get from DB)
        LocalDateTime recovered = ts.toLocalDateTime();
        System.out.println("Recovered: " + recovered);
        // 2026-04-23T14:30

        // Verify they match
        System.out.println("Match: " + original.isEqual(recovered));
        // true
    }
}
Timestamp.valueOf() converts TO the database format. .toLocalDateTime() converts FROM the database format. These are the only two conversions you need to memorize for JDBC.

Exercise 6: Business Hours Validator

Write a method boolean isWithinBusinessHours(LocalDateTime dateTime) that returns true only if the datetime is on a weekday (Monday-Friday) AND between 9:00 AM and 6:00 PM. Test with various inputs including weekends and after-hours.

Solution
1
public static boolean isWithinBusinessHours(LocalDateTime dt) {
    DayOfWeek day = dt.getDayOfWeek();
    LocalTime time = dt.toLocalTime();

    // Check weekday
    boolean isWeekday = day != DayOfWeek.SATURDAY && day != DayOfWeek.SUNDAY;

    // Check business hours (9:00 AM to 6:00 PM)
    boolean isDuringHours = !time.isBefore(LocalTime.of(9, 0))
                         && time.isBefore(LocalTime.of(18, 0));

    return isWeekday && isDuringHours;
}

// Tests
System.out.println(isWithinBusinessHours(
    LocalDateTime.of(2026, 4, 23, 14, 30)));  // Thu 2:30 PM → true

System.out.println(isWithinBusinessHours(
    LocalDateTime.of(2026, 4, 25, 14, 30)));  // Sat 2:30 PM → false

System.out.println(isWithinBusinessHours(
    LocalDateTime.of(2026, 4, 23, 20, 0)));   // Thu 8:00 PM → false

System.out.println(isWithinBusinessHours(
    LocalDateTime.of(2026, 4, 23, 9, 0)));    // Thu 9:00 AM → true
toLocalTime() extracts the time part for hour comparison. getDayOfWeek() returns the enum value for day checking. The ! before time.isBefore(9:00) means "time is at or after 9:00." Combined with time.isBefore(18:00), this creates the 9 AM to 6 PM window (inclusive start, exclusive end).
