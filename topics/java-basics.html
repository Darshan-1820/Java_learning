Section 01

What is a Variable?

Every program needs to store data — a user's name, a parcel's weight, a price. Variables are how you give data a name so you can use it later.

The Named Box Analogy

Think of your computer's memory as a giant warehouse full of empty boxes. A variable is when you pick a box, write a label on it (the variable name), specify what kind of thing goes in it (the data type), and optionally put something inside (the value).

  MEMORY (warehouse)
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │   ┌─────────────┐   ┌─────────────┐             │
  │   │ parcelWeight │   │  userName   │             │
  │   │ ─────────── │   │ ─────────── │             │
  │   │  type: int   │   │ type: String│             │
  │   │  value: 500  │   │ value: "Raj"│             │
  │   └─────────────┘   └─────────────┘             │
  │                                                  │
  │   ┌─────────────┐   ┌─────────────┐             │
  │   │  baseCost    │   │  isPaid     │             │
  │   │ ─────────── │   │ ─────────── │             │
  │   │ type: double │   │type: boolean│             │
  │   │  value: 75.0 │   │ value: true │             │
  │   └─────────────┘   └─────────────┘             │
  │                                                  │
  └──────────────────────────────────────────────────┘
Variables are named boxes in memory, each with a type and a value

Declaration, Initialization, and Assignment

There are three distinct steps when working with variables. You can do them separately or combine them.

1. Declaration — creating the box

You tell Java: "I need a box that holds this type of data, and I want to call it this name." No value goes in yet — the box is empty.

Declaration 
int parWeightGram;         // box labeled "parWeightGram", holds integers
String fullName;           // box labeled "fullName", holds text
double parServiceCost;     // box labeled "parServiceCost", holds decimal numbers
boolean isDelivered;       // box labeled "isDelivered", holds true/false

2. Initialization — putting the first value in

The very first time you put a value into the box. This is called initialization.

Initialization 
parWeightGram = 500;
fullName = "Raj Kumar";
parServiceCost = 75.0;
isDelivered = false;

3. Assignment — changing the value later

Any time after initialization, you can put a new value in the box. The old value is thrown away.

Assignment (changing the value) 
parWeightGram = 750;       // was 500, now 750. Old value gone.
parServiceCost = 112.5;    // recalculated after weight change

Combined — declare + initialize in one line

This is what you will do 95% of the time in real code:

Declare + Initialize (most common) 
int parWeightGram = 500;
String fullName = "Raj Kumar";
double parServiceCost = 75.0;
boolean isDelivered = false;
Line-by-line explanation
1
int parWeightGram = 500;
int = data type (whole number). parWeightGram = variable name. = means "put this value in." 500 = the value. ; = end of statement.
2
String fullName = "Raj Kumar";
String = text data type (capital S — it is a class, not a primitive). Text goes in double quotes.
3
double parServiceCost = 75.0;
double = decimal number type. Used for money, measurements, anything with a fractional part.
4
boolean isDelivered = false;
boolean = true or false. Only two possible values. Named after mathematician George Boole.
In Nimbus

Every field in the Nimbus system is a variable. The User class has String userId, String fullName, String email, String mobile, String password, String role. The Booking class has String bookingId, int parWeightGram, double parServiceCost, String parStatus, LocalDateTime parPickupTime. Each of these is a named box holding one piece of data about a user or a booking.

Variable Naming Rules

Java has strict rules about what you can name a variable:

RuleValidInvalid
Must start with a letter, $, or _ userName, _count, $price 1stUser, -name
Cannot use Java keywords classType, intValue class, int, public
Case sensitive userId and UserId are different —
Use camelCase by convention parWeightGram, parServiceCost par_weight_gram, PARWEIGHT
Common Mistake

You cannot use a variable before initializing it inside a method. Java will refuse to compile. Class fields get default values automatically (0, null, false), but local variables inside methods do not.

Section 02

Primitive Data Types

Java has exactly 8 primitive types — the basic building blocks for storing simple values. They are built into the language and stored directly in memory (on the stack).

Analogy

Think of primitives as standard-size containers at a warehouse. A small jar holds a byte. A bucket holds an int. A barrel holds a long. You pick the right container based on how much data you need to store. Using a barrel when a jar would do wastes space.

The Complete Primitive Types Table

Type Size Range Default Example
byte 1 byte (8 bits) -128 to 127 0 byte statusCode = 1;
short 2 bytes (16 bits) -32,768 to 32,767 0 short pinCode = 411;
int 4 bytes (32 bits) -2.1 billion to 2.1 billion 0 int parWeightGram = 500;
long 8 bytes (64 bits) -9.2 quintillion to 9.2 quintillion 0L long trackingNum = 9876543210L;
float 4 bytes (32 bits) ~6-7 decimal digits precision 0.0f float rating = 4.5f;
double 8 bytes (64 bits) ~15-16 decimal digits precision 0.0 double parServiceCost = 75.0;
char 2 bytes (16 bits) 0 to 65,535 (Unicode) '\u0000' char grade = 'A';
boolean ~1 bit (JVM-dependent) true or false only false boolean isDelivered = true;

Which Type to Use When

In practice, you will use four of these constantly and the others rarely:

  • int — your go-to for whole numbers. Parcel weight, counts, IDs as numbers.
  • double — your go-to for decimals. Prices, rates, percentages. Never use float for money.
  • boolean — any yes/no decision. Is the parcel delivered? Is the user an admin?
  • char — rare. A single character. You will almost always use String instead.

byte and short are used when memory is extremely tight (embedded systems, processing millions of records). long is for numbers bigger than 2.1 billion (timestamps, large IDs). float is almost never used — double is better in every way.

In Nimbus

Nimbus uses int parWeightGram for parcel weight (a 2.1 billion gram limit is more than enough — that is 2,100 metric tons). It uses double parServiceCost for the cost because money needs decimal precision. String parStatus holds status text like "BOOKED", "IN_TRANSIT", "DELIVERED". The double baseRate = 0.15 stores the per-gram rate used in pricing calculations.

Literal Suffixes

Some types need a suffix letter so Java knows which type you mean:

Literal suffixes 
long trackingNumber = 9876543210L;   // L suffix = long (without L, compiler error)
float rating = 4.5f;                 // f suffix = float (without f, Java treats 4.5 as double)
double cost = 75.0;                  // no suffix needed, decimals are double by default
int weight = 500;                    // no suffix needed, whole numbers are int by default
Common Mistake

long bigNumber = 9876543210; will not compile. Java sees 9876543210 as an int literal, but that number exceeds int's max (2.1 billion). You must write 9876543210L with the L suffix. Use uppercase L, not lowercase l — lowercase looks like the number 1.

Section 03

Reference Types

Everything that is not one of the 8 primitives is a reference type. This includes Strings, arrays, and any object you create from a class. The difference is fundamental to understanding Java.

Primitives vs References — Where They Live

Primitives store the actual value directly in the variable. Reference types store an address (a pointer) that tells Java where the actual data lives in a separate area of memory called the heap.

  STACK (fast, small)                    HEAP (large, shared)
  ┌───────────────────────┐              ┌──────────────────────────┐
  │                       │              │                          │
  │  parWeightGram = 500  │  (direct)    │                          │
  │                       │              │   ┌──────────────────┐   │
  │  parServiceCost = 75.0│  (direct)    │   │ "Raj Kumar"      │   │
  │                       │              │   │ (String object)  │   │
  │  fullName = 0x3A7F ───┼──────────────┼──>│ chars: R,a,j,... │   │
  │                       │              │   └──────────────────┘   │
  │  isDelivered = true   │  (direct)    │                          │
  │                       │              │   ┌──────────────────┐   │
  │  booking = 0x5B2E ────┼──────────────┼──>│ Booking object   │   │
  │                       │              │   │ id: "BK001"      │   │
  │                       │              │   │ weight: 500      │   │
  └───────────────────────┘              │   └──────────────────┘   │
                                         └──────────────────────────┘
Primitives hold values directly. References hold addresses pointing to heap objects.

Why This Matters

When you copy a primitive, you get an independent copy. When you copy a reference, both variables point to the same object.

Primitive copy vs Reference copy 
// Primitive copy: independent
int weightA = 500;
int weightB = weightA;   // weightB gets its own copy of 500
weightB = 750;           // changing B does NOT affect A
System.out.println(weightA); // 500 (unchanged)

// Reference copy: shared object
int[] scoresA = {90, 85, 70};
int[] scoresB = scoresA;     // scoresB points to the SAME array
scoresB[0] = 100;            // changing via B also changes A
System.out.println(scoresA[0]); // 100 (changed!)

Common Reference Types You Will Use

TypeWhat It HoldsNimbus Example
String Text (sequence of characters) String userId = "USR001";
Arrays Fixed-size collection of same type String[] statuses = {"BOOKED", "IN_TRANSIT", "DELIVERED"};
Objects Instance of any class User customer = new User("USR001", "Raj");
LocalDateTime Date and time combined LocalDateTime parPickupTime = LocalDateTime.now();
Key Difference

A reference variable can be null — meaning the box exists but it does not point to anything. Primitives can never be null. If you try to use a null reference (like calling null.length()), you get the dreaded NullPointerException.

In Nimbus

The User and Booking classes are reference types. When the service layer passes a Booking object to the DAO layer, it is passing the address, not a copy. If the DAO modifies the booking's status, the service layer sees the change too — because both are looking at the same object in memory.

Section 04

Type Casting

Sometimes you need to convert a value from one type to another. Java handles some conversions automatically and requires you to be explicit about others.

Implicit Casting (Widening) — Automatic

When you go from a smaller type to a larger type, Java does it automatically because no data can be lost. It is like pouring water from a small glass into a large bucket — nothing spills.

byte1 byte
→
short2 bytes
→
int4 bytes
→
long8 bytes
→
float4 bytes
→
double8 bytes
Widening direction: automatic, safe, no data loss
Implicit (widening) casting 
int parWeightGram = 500;
double weightAsDouble = parWeightGram;  // int → double: automatic
System.out.println(weightAsDouble);     // 500.0

// Nimbus pricing calculation:
int weight = 500;
double baseRate = 0.15;
double cost = weight * baseRate;        // int * double = double (automatic)
System.out.println(cost);              // 75.0

Explicit Casting (Narrowing) — Manual

When you go from a larger type to a smaller type, Java refuses to do it automatically because data could be lost. You must explicitly tell Java: "I know I might lose data, do it anyway." This is like pouring from a bucket into a glass — some water will spill.

Explicit (narrowing) casting 
double parServiceCost = 112.75;
int roundedCost = (int) parServiceCost;  // explicit cast: double → int
System.out.println(roundedCost);         // 112 (decimal part LOST, not rounded!)

// Without the cast, this line would NOT compile:
// int roundedCost = parServiceCost;     // ERROR: incompatible types

double expressCost = 135.99;
int displayCost = (int) expressCost;     // 135 — .99 is chopped off, not rounded
// If you want proper rounding:
int properRound = (int) Math.round(expressCost);  // 136
Data Loss

Narrowing does not round — it truncates (chops off the decimal). (int) 99.99 gives you 99, not 100. If you need rounding, use Math.round() first.

In Nimbus

When calculating parServiceCost = parWeightGram * baseRate, the int weight is automatically widened to double before multiplication. No explicit cast needed. But if you wanted to display the cost without decimals (like "Rs 75" instead of "Rs 75.00"), you would need an explicit cast: (int) parServiceCost.

Section 05

Wrapper Classes

Every primitive type has a corresponding wrapper class — an object version of the primitive. They exist because some parts of Java only work with objects, not primitives.

Primitive ↔ Wrapper Mapping

PrimitiveWrapper ClassExample
byteByteByte b = 10;
shortShortShort s = 100;
intIntegerInteger weight = 500;
longLongLong id = 9876543210L;
floatFloatFloat rating = 4.5f;
doubleDoubleDouble cost = 75.0;
charCharacterCharacter grade = 'A';
booleanBooleanBoolean active = true;

Why Do Wrapper Classes Exist?

Java's generics and collections (like ArrayList, HashMap) only work with objects. You cannot write ArrayList<int> — it must be ArrayList<Integer>. Wrapper classes bridge this gap.

Why wrappers are needed 
// This does NOT compile:
// ArrayList<int> weights = new ArrayList<int>();  // ERROR

// This works:
ArrayList<Integer> weights = new ArrayList<Integer>();
weights.add(500);    // adding an int, Java wraps it to Integer automatically
weights.add(750);
weights.add(300);

int firstWeight = weights.get(0);  // returns Integer, Java unwraps to int automatically

Autoboxing and Unboxing

Autoboxing is when Java automatically converts a primitive to its wrapper. Unboxing is the reverse. You almost never have to do it manually.

Autoboxing and unboxing 
// Autoboxing: int → Integer (automatic)
Integer parWeight = 500;          // Java converts 500 (int) to Integer object

// Unboxing: Integer → int (automatic)
int rawWeight = parWeight;        // Java converts Integer back to int

// Useful methods on wrapper classes:
String weightText = "500";
int parsedWeight = Integer.parseInt(weightText);   // String → int
double parsedCost = Double.parseDouble("75.50");   // String → double
boolean isActive = Boolean.parseBoolean("true");   // String → boolean

// Reverse: number to String
String costString = String.valueOf(75.0);          // double → String
String weightString = Integer.toString(500);       // int → String
In Nimbus

When reading user input from a web form, everything arrives as a String. To use it as a number, you must parse it: int weight = Integer.parseInt(request.getParameter("weight"));. If the user types "abc" instead of a number, parseInt throws a NumberFormatException — which is why input validation matters.

Null Trap

A primitive int can never be null, but an Integer wrapper can. If you unbox a null Integer, you get a NullPointerException: Integer x = null; int y = x; — CRASH. Always check for null before unboxing.

Section 06

String Basics

Strings are the most common reference type in Java. They hold text. But they have special rules that trip up every beginner — especially around == vs .equals().

Creating Strings

Two ways to create a String 
// Way 1: String literal (uses the String Pool)
String role = "CUSTOMER";
String anotherRole = "CUSTOMER";  // reuses the SAME object from pool

// Way 2: new keyword (creates a NEW object on the heap, always)
String roleNew = new String("CUSTOMER");  // different object, same text

String Immutability

Once a String object is created, its content cannot be changed. When you think you are changing a String, Java is actually creating a new String object and pointing your variable to it.

Strings are immutable 
String status = "BOOKED";
status = "IN_TRANSIT";    // "BOOKED" is NOT modified. A new String "IN_TRANSIT"
                          // is created, and status now points to it.
                          // "BOOKED" is still in memory (until garbage collected).

The String Pool

Java maintains a special memory area called the String Pool. When you create a string using a literal (quotes), Java first checks if that exact text already exists in the pool. If yes, it reuses the existing object. This saves memory.

  HEAP MEMORY
  ┌────────────────────────────────────────────────────────┐
  │                                                        │
  │   STRING POOL                                          │
  │   ┌──────────────────────────────────────┐             │
  │   │                                      │             │
  │   │   ┌────────────┐  ┌─────────────┐   │             │
  │   │   │ "CUSTOMER"  │  │ "BOOKED"    │   │             │
  │   │   └─────▲──▲───┘  └─────────────┘   │             │
  │   │         │  │                         │             │
  │   └─────────┼──┼─────────────────────────┘             │
  │             │  │                                       │
  │   role ─────┘  │                                       │
  │   anotherRole ─┘   (both point to SAME object)        │
  │                                                        │
  │   OUTSIDE POOL                                         │
  │   ┌────────────┐                                       │
  │   │ "CUSTOMER"  │  ← roleNew (separate object)         │
  │   └────────────┘                                       │
  │                                                        │
  └────────────────────────────────────────────────────────┘
String Pool reuses literals. new String() always creates a separate object.

== vs .equals() — THE Critical Trap

This is one of the most common bugs in Java. You must understand this.

  • == compares addresses (are these the same object in memory?)
  • .equals() compares content (do these contain the same text?)
== vs .equals() demo 
String a = "CUSTOMER";
String b = "CUSTOMER";
String c = new String("CUSTOMER");

System.out.println(a == b);        // true  (same object in pool)
System.out.println(a == c);        // false (different objects!)
System.out.println(a.equals(c));   // true  (same content)

// In Nimbus — checking if a user is an admin:
String role = user.getRole();

// WRONG (might work sometimes, fail other times):
if (role == "ADMIN") { ... }

// CORRECT (always works):
if (role.equals("ADMIN")) { ... }

// SAFEST (avoids NullPointerException if role is null):
if ("ADMIN".equals(role)) { ... }
Why == sometimes works and sometimes does not
1
a == b → true
Both are string literals "CUSTOMER". Java puts the first one in the pool, then reuses it for b. Same object = same address = == returns true.
2
a == c → false
new String() always creates a new object outside the pool. Different object = different address = == returns false, even though the text is identical.
3
a.equals(c) → true
.equals() compares the actual characters. Both contain "CUSTOMER" so it returns true regardless of where the objects live in memory.
4
"ADMIN".equals(role)
By putting the literal first, if role is null, you get false instead of a NullPointerException. This is a defensive coding pattern used in production code.
The Rule

Always use .equals() to compare Strings. Never use == for String comparison. The only exception is comparing to null: if (str == null) is correct because null is not an object and has no .equals() method.

In Nimbus

The parStatus field is a String. When checking if a parcel is delivered, always write "DELIVERED".equals(parStatus), never parStatus == "DELIVERED". The status often comes from database queries or user input, which creates new String objects — meaning == would return false even when the text matches.

Integer Division Trap

500 / 3 gives 166, not 166.67. When both operands are integers, Java does integer division and discards the decimal part. To get a decimal result, make at least one operand a double: 500 / 3.0 gives 166.6667. Or cast: (double) 500 / 3.

// Nimbus pricing engine
int parWeightGram = 500;

double expressCost = standardCost * 1.8;         // 75.0 * 1.8  = 135.0
double overnightCost = standardCost * 3.0;       // 75.0 * 3.0  = 225.0

System.out.println("Standard:  Rs " + standardCost);   // Rs 75.0
System.out.println("Express:   Rs " + expressCost);     // Rs 135.0
System.out.println("Overnight: Rs " + overnightCost);   // Rs 225.0

// Modulus example: parcels per batch
int totalParcels = 47;
int batchSize = 10;
int fullBatches = totalParcels / batchSize;   // 4 full batches
int remaining = totalParcels % batchSize;     // 7 parcels left over

// Status check (for Strings, use .equals() not ==)

Logical Operators

Combine multiple conditions into one.

Logical operators in Nimbus 
boolean isBooked = true;
boolean isPaid = true;
boolean isOverweight = false;

// AND: both must be true
boolean canShip = isBooked && isPaid && !isOverweight;
System.out.println(canShip);  // true (booked + paid + not overweight)

// OR: at least one
String role = "OFFICER";
boolean canViewAll = "ADMIN".equals(role) || "OFFICER".equals(role);
System.out.println(canViewAll);  // true

// NOT: flip
boolean isDelivered = false;
boolean needsTracking = !isDelivered;
System.out.println(needsTracking);  // true
Short-Circuit Evaluation

&& stops evaluating as soon as it finds a false (because if one is false, the whole AND is false). || stops as soon as it finds a true. This is called short-circuiting. It matters when the second condition has a side effect or could throw an error: user != null && user.isAdmin() — if user is null, Java never calls isAdmin(), avoiding a crash.

Increment and Decrement (++ and --)

Add or subtract 1 from a variable. Two versions: prefix (before the variable) and postfix (after).

Prefix vs Postfix 
int deliveryCount = 10;

// Postfix: use current value FIRST, then increment
int x = deliveryCount++;   // x = 10, deliveryCount becomes 11
System.out.println("x = " + x);                // 10
System.out.println("count = " + deliveryCount); // 11

// Prefix: increment FIRST, then use new value
int y = ++deliveryCount;   // deliveryCount becomes 12, y = 12
System.out.println("y = " + y);                // 12
System.out.println("count = " + deliveryCount); // 12

// Same rules apply for --
int pending = 5;
pending--;    // now 4
--pending;    // now 3
Analogy

Postfix (i++) is like a cashier who gives you the current price, then updates the price tag after. Prefix (++i) is like a cashier who updates the price tag first, then charges you the new price.

Ternary Operator ( ? : )

A compact one-line if-else. Structure: condition ? valueIfTrue : valueIfFalse

Ternary operator 
// Standard if-else (3 lines)
String label;
if (parWeightGram > 10000) {
    label = "HEAVY";
} else {
    label = "STANDARD";
}

// Same thing with ternary (1 line)
String label = parWeightGram > 10000 ? "HEAVY" : "STANDARD";

// Nimbus: determine service multiplier
double multiplier = "EXPRESS".equals(serviceType) ? 1.8 : 1.0;

// Nested (use sparingly — hard to read)
String tier = parWeightGram > 10000 ? "HEAVY"
            : parWeightGram > 5000  ? "MEDIUM"
            : "LIGHT";
In Nimbus

The pricing engine uses the ternary extensively: double multiplier = "OVERNIGHT".equals(type) ? 3.0 : "EXPRESS".equals(type) ? 1.8 : 1.0; This picks the correct multiplier (3.0x for overnight, 1.8x for express, 1.0x for standard) in a single expression.

Section 08

Type Promotion in Expressions

Java has a surprising rule: when you do math with small types like byte and short, Java automatically promotes them to int. This catches beginners off guard.

The Surprising Rule

byte + byte = int (not byte!) 
byte a = 50;
byte b = 60;
// byte c = a + b;   // COMPILER ERROR! a + b produces an int, not a byte.
int c = a + b;       // Works: result is int
byte d = (byte)(a + b);  // Works: explicit cast back to byte

short x = 100;
short y = 200;
// short z = x + y;  // COMPILER ERROR! Same rule.
int z = x + y;       // Works

Why Java Does This

Safety. A byte can hold up to 127. If you add two bytes like 100 + 100 = 200, the result does not fit in a byte. Instead of silently overflowing and giving you a wrong answer, Java promotes everything to int (which can hold up to 2.1 billion) so the result is always safe.

Full Promotion Rules

  1. If any operand is double → the other is promoted to double
  2. Else if any operand is float → the other is promoted to float
  3. Else if any operand is long → the other is promoted to long
  4. Else everything (byte, short, char) is promoted to int
Promotion in Nimbus pricing 
int parWeightGram = 500;       // int
double baseRate = 0.15;         // double

// Rule 1 applies: int * double → int promoted to double
double cost = parWeightGram * baseRate;  // 500.0 * 0.15 = 75.0
// The result is double, which is what we want for a price.

// What if we did integer math by mistake?
int wrongCost = 500 / 3;       // 166 (not 166.667 — decimal lost!)
double rightCost = 500 / 3.0;  // 166.6667 (3.0 forces double division)
In Nimbus

The expression parWeightGram * baseRate works correctly because baseRate is a double, so the int weight gets promoted automatically. If both were int (like 500 * 0), you would lose all decimal precision. This is why baseRate is declared as double, not int.

Section 09

Constants — The final Keyword

Some values should never change after being set. The final keyword enforces this at the compiler level — if anyone tries to change a final variable, the code will not compile.

Constants in Nimbus 
// Naming convention: ALL_CAPS with underscores
final double BASE_RATE_PER_GRAM = 0.15;
final double EXPRESS_MULTIPLIER = 1.8;
final double OVERNIGHT_MULTIPLIER = 3.0;
final int MAX_WEIGHT_GRAMS = 30000;    // 30 kg limit
final String DEFAULT_STATUS = "BOOKED";

// These will NOT compile:
// BASE_RATE_PER_GRAM = 0.20;    // ERROR: cannot assign a value to final variable
// MAX_WEIGHT_GRAMS = 50000;     // ERROR: cannot assign a value to final variable

// Using constants in pricing:
int parWeightGram = 500;
double standardCost = parWeightGram * BASE_RATE_PER_GRAM;       // 75.0
double expressCost = standardCost * EXPRESS_MULTIPLIER;          // 135.0
double overnightCost = standardCost * OVERNIGHT_MULTIPLIER;     // 225.0

Why Use Constants?

  • Prevents accidents — if someone accidentally types BASE_RATE = 0, the compiler catches it instantly
  • Single source of truth — change the rate in one place, every calculation updates automatically
  • Self-documenting — BASE_RATE_PER_GRAM is clearer than a mystery number 0.15 scattered through the code
Magic Numbers

A "magic number" is a raw number in code with no explanation: cost = weight * 0.15 * 1.8;. What is 0.15? What is 1.8? Nobody knows without context. Always replace magic numbers with named constants: cost = weight * BASE_RATE_PER_GRAM * EXPRESS_MULTIPLIER;. Now anyone can read it.

Section 10

Scanner for Input

To make your programs interactive, you need to read input from the user. Java's Scanner class reads from the keyboard (System.in).

NimbusBookingInput.java 
import java.util.Scanner;

public class NimbusBookingInput {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Enter your full name: ");
        String fullName = scanner.nextLine();          // reads entire line

        System.out.print("Enter parcel weight (grams): ");
        int parWeightGram = scanner.nextInt();         // reads one integer

        System.out.print("Enter base rate per gram: ");
        double baseRate = scanner.nextDouble();        // reads one decimal

        double cost = parWeightGram * baseRate;

        System.out.println("\n--- Booking Summary ---");
        System.out.println("Customer: " + fullName);
        System.out.println("Weight:   " + parWeightGram + "g");
        System.out.println("Cost:     Rs " + cost);

        scanner.close();  // always close when done
    }
}
Line-by-line explanation
1
import java.util.Scanner;
Import the Scanner class from Java's utility library. Without this line, Java does not know what Scanner is.
2
Scanner scanner = new Scanner(System.in);
Create a Scanner object that reads from keyboard input (System.in).
3
scanner.nextLine()
Reads an entire line of text until the user presses Enter. Used for names, addresses, etc.
4
scanner.nextInt()
Reads one integer from input. If the user types "abc", throws InputMismatchException.
5
scanner.nextDouble()
Reads one decimal number. Same error risk as nextInt if user types non-numeric text.
6
scanner.close()
Release the Scanner resource. Good practice, though not strictly required for System.in in small programs.

The Newline Trap — nextInt() then nextLine()

This is the #1 Scanner bug that every beginner hits. When you call nextInt(), it reads the number but leaves the newline character (Enter key) in the buffer. The next nextLine() reads that leftover newline and returns an empty string.

The newline trap (BROKEN) 
Scanner scanner = new Scanner(System.in);

System.out.print("Enter weight: ");
int weight = scanner.nextInt();     // reads "500", leaves "\n" in buffer

System.out.print("Enter name: ");
String name = scanner.nextLine();   // reads the leftover "\n" immediately!
// name is now "" (empty string) — user never got to type!

System.out.println("Name: '" + name + "'");  // prints: Name: ''
The fix: consume the leftover newline 
Scanner scanner = new Scanner(System.in);

System.out.print("Enter weight: ");
int weight = scanner.nextInt();
scanner.nextLine();   // ← CONSUME the leftover newline (throwaway call)

System.out.print("Enter name: ");
String name = scanner.nextLine();   // now this works correctly

System.out.println("Name: '" + name + "'");  // prints: Name: 'Raj Kumar'
The Rule

After every nextInt(), nextDouble(), or nextFloat(), add a throwaway scanner.nextLine() call before using nextLine() to read actual text. This clears the leftover newline from the buffer.

Section 11

Putting It All Together

A complete Nimbus pricing calculator using every concept from this page — variables, types, casting, constants, operators, and Scanner input.

NimbusPricingCalculator.java 
import java.util.Scanner;

public class NimbusPricingCalculator {

    // Constants (final + ALL_CAPS)
    static final double BASE_RATE_PER_GRAM = 0.15;
    static final double EXPRESS_MULTIPLIER = 1.8;
    static final double OVERNIGHT_MULTIPLIER = 3.0;
    static final int MAX_WEIGHT_GRAMS = 30000;

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        // Read customer name (String - reference type)
        System.out.print("Customer name: ");
        String fullName = scanner.nextLine();

        // Read email (String - reference type)
        System.out.print("Email: ");
        String email = scanner.nextLine();

        // Read weight (int - primitive type)
        System.out.print("Parcel weight (grams): ");
        int parWeightGram = scanner.nextInt();
        scanner.nextLine();  // consume newline (the trap!)

        // Read service type (String - reference type)
        System.out.print("Service (STANDARD / EXPRESS / OVERNIGHT): ");
        String serviceType = scanner.nextLine().toUpperCase();

        // Weight validation (comparison operator)
        boolean isOverweight = parWeightGram > MAX_WEIGHT_GRAMS;
        if (isOverweight) {
            System.out.println("ERROR: Weight exceeds " + MAX_WEIGHT_GRAMS + "g limit.");
            scanner.close();
            return;
        }

        // Calculate cost (arithmetic + type promotion: int * double = double)
        double baseCost = parWeightGram * BASE_RATE_PER_GRAM;

        // Select multiplier (ternary operator + .equals() for Strings)
        double multiplier = "OVERNIGHT".equals(serviceType) ? OVERNIGHT_MULTIPLIER
                          : "EXPRESS".equals(serviceType)   ? EXPRESS_MULTIPLIER
                          : 1.0;

        double parServiceCost = baseCost * multiplier;

        // Status (constant String)
        String parStatus = "BOOKED";

        // Display results (String concatenation with + operator)
        System.out.println("\n============================");
        System.out.println("  NIMBUS BOOKING RECEIPT");
        System.out.println("============================");
        System.out.println("Customer:  " + fullName);
        System.out.println("Email:     " + email);
        System.out.println("Weight:    " + parWeightGram + "g");
        System.out.println("Service:   " + serviceType);
        System.out.println("Base cost: Rs " + baseCost);
        System.out.println("Multiplier: " + multiplier + "x");
        System.out.println("Total:     Rs " + parServiceCost);
        System.out.println("Status:    " + parStatus);
        System.out.println("============================");

        scanner.close();
    }
}
Concepts used in this program
1
static final double BASE_RATE_PER_GRAM = 0.15;
Constants (Section 9). final prevents changes. static makes it shared across the class. ALL_CAPS naming convention.
2
String fullName = scanner.nextLine();
Scanner (Section 10) + Reference type (Section 3). Reads a full line of text from keyboard.
3
int parWeightGram = scanner.nextInt();
Primitive type (Section 2) + Scanner. Reads an integer. Followed by nextLine() to fix the newline trap.
4
parWeightGram * BASE_RATE_PER_GRAM
Type promotion (Section 8). int * double = double. Java automatically widens the int.
5
"OVERNIGHT".equals(serviceType) ? ... : ...
Ternary operator (Section 7) + .equals() for String comparison (Section 6). Literal-first pattern prevents NullPointerException.
6
parWeightGram > MAX_WEIGHT_GRAMS
Comparison operator (Section 7). Returns a boolean used for validation.
