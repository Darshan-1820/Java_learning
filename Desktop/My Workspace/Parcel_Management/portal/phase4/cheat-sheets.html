Spring Boot Annotations

Every annotation used in Nimbus Phase 4, in one place.

Annotation Where What It Does
@SpringBootApplication Main class Combines @Configuration + @EnableAutoConfiguration + @ComponentScan. Entry point.
@RestController Controller class Marks class as a REST controller. All methods return data (JSON), not views.
@RequestMapping Controller class Sets the base URL path for all endpoints in the controller.
@GetMapping Method Handles HTTP GET requests. Used for reading/fetching data.
@PostMapping Method Handles HTTP POST requests. Used for creating new resources.
@PutMapping Method Handles HTTP PUT requests. Used for updating existing resources.
@DeleteMapping Method Handles HTTP DELETE requests. Used for removing resources.
@PathVariable Method param Extracts a value from the URL path: /bookings/{id} → id.
@RequestParam Method param Extracts a query parameter: ?userId=1 → userId.
@RequestBody Method param Reads the HTTP request body (JSON) and converts to Java object.
@Valid Method param Triggers bean validation on the parameter before method runs.
@Service Service class Marks class as a service layer bean. Spring auto-creates an instance.
@Repository DAO class Marks class as a data access bean. Translates SQL exceptions automatically.
@Configuration Config class Contains @Bean methods. Used for CORS config, RestTemplate, etc.
@Autowired Field/constructor Injects a dependency. Spring finds the matching bean and wires it in.
@ControllerAdvice Exception handler class Global exception handling across all controllers.
@ExceptionHandler Method Catches a specific exception type and returns a custom error response.
@CrossOrigin Controller/method Enables CORS for specific origins on a controller or method.

Angular Decorators

Decorators tell Angular what a class is and how it should behave.

DecoratorUsed OnPurpose
@Component Class Declares a component with its template, styles, and selector
@NgModule Class Declares a module — groups components, imports, providers
@Injectable Class Marks a class as injectable (service). {providedIn: 'root'} = singleton
@Input() Property Receives data from parent component
@Output() Property Emits events to parent component (with EventEmitter)
Component decorator example 
@Component({
  selector: 'app-booking-detail',      // HTML tag: <app-booking-detail>
  templateUrl: './booking-detail.component.html',
  styleUrls: ['./booking-detail.component.css']
})
export class BookingDetailComponent implements OnInit {
  @Input() bookingId!: number;          // Received from parent
  @Output() statusChanged = new EventEmitter<string>();  // Emits to parent

  booking: Booking | null = null;

  constructor(private bookingService: BookingService) {}

  ngOnInit() {
    this.bookingService.getById(this.bookingId).subscribe(
      data => this.booking = data
    );
  }
}

Postman Tips

  • Variables: Set {{baseUrl}} to http://localhost:8080 in environment. Use {{baseUrl}}/api/bookings in requests. Switch between dev/prod by changing the environment.
  • Tests tab: Add pm.test("Status is 200", () => pm.response.to.have.status(200)); to auto-verify responses.
  • Runner: Run an entire collection in sequence. Great for testing the full workflow: register → login → create → get → update.
  • Pre-request Scripts: Set a dynamic timestamp: pm.variables.set("now", new Date().toISOString());
