Section 01

What You Need to Download

Only 2 things. That's it. No Tomcat. No MySQL. No extra servers. Everything else is built into the tools or handled by the project itself.

1. JDK 17 (Java Development Kit)

This is the Java compiler + runtime. Without it, Eclipse can't compile or run your code.

Download from: https://adoptium.net/temurin/releases/

Choose: Windows x64 → .msi installer. Run the installer, click Next on everything. Done.

Why JDK 17?

Spring Boot 3 (which we use from Sprint 2) requires Java 17 minimum. Java 8 won't work. Make sure you download JDK 17, not 8 or 11.

2. Eclipse IDE for Enterprise Java Developers

This is your code editor. It has Maven, Java, server support all built in.

Download from: https://www.eclipse.org/downloads/packages/

Choose: "Eclipse IDE for Enterprise Java and Web Developers" → Windows x86_64. Download the ZIP file. Extract it anywhere (e.g., C:\eclipse). Run eclipse.exe.

MUST be Enterprise Edition

Do NOT download "Eclipse IDE for Java Developers" (the basic one). It's missing Maven support, servlet tools, and Spring Boot features you need for Sprints 2-4. Always pick the Enterprise version.

What You Do NOT Need

Don't DownloadWhy
Apache TomcatSpring Boot has Tomcat built in. When you run your app, Tomcat starts automatically inside it.
MySQL / PostgreSQLWe use H2 Database which runs inside your Java app. No install needed. No server to start.
Maven (standalone)Eclipse Enterprise has Maven built in. No separate download.
Node.js / npmNot needed. Our frontend is plain HTML/CSS/JS (no React, no Angular build step).
Section 02

Install Checklist

Verify everything is ready before moving on.

Verify in Command Prompt (Win+R → cmd) 
java -version

You should see something like: openjdk version "17.0.x". If it says "not recognized" or shows version 8/11, your JDK 17 isn't installed correctly or isn't in PATH.

Fix: java not recognized

After installing JDK 17, you may need to add it to PATH. Go to: System Properties → Environment Variables → System Variables → Path → Edit → New → add C:\Program Files\Eclipse Adoptium\jdk-17.x.x\bin. Restart Command Prompt.

Eclipse check: Just open Eclipse. If it launches without errors and you see "Welcome" tab, you're good. Close the Welcome tab.

Workspace: When Eclipse asks for a workspace folder, use C:\eclipse-workspace. This is where all your projects live. Don't use Desktop.

Section 03

Sprint 1: Create a Java Project

Sprint 1 is a plain Java console app. No Maven, no database, no server. Just Java classes using ArrayList to store data in memory.

What Sprint 1 Does

A console app where users type commands in the terminal. Register, login, book parcels, track deliveries — all stored in ArrayList (data disappears when you close the app). This is to learn Java basics, OOP, and Collections.

Steps in Eclipse

  1. File → New → Java Project
  2. Project name: nimbus-sprint1
  3. JRE: Make sure "Use an execution environment JRE" shows JavaSE-17. If not, click "Configure JREs" and add your JDK 17 path.
  4. Click Finish.

You now have a project in Package Explorer on the left with a src folder.

Alternative: Online GDB

If Eclipse isn't available (e.g., at TCS before setup), you can run Sprint 1 on https://www.onlinegdb.com/online_java_compiler. Select Language: Java. Paste all code into one file or use their multi-file feature.

What Changes in Sprint 2

Same features as Sprint 1 (register, login, book, track) but now: data saves to a real database file, DAOs replace direct ArrayList access, Maven manages your libraries, and Spring Boot sets up the database automatically.

Section 08

Sprint 2: pom.xml

This file tells Maven what libraries to download. Replace the entire contents of pom.xml with this:

pom.xml 
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>

  <!-- Spring Boot Parent: gives you default configs for everything -->
  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.5</version>
  </parent>

  <groupId>com.nimbus</groupId>
  <artifactId>nimbus-parcel</artifactId>
  <version>0.0.1-SNAPSHOT</version>
  <packaging>jar</packaging>

  <properties>
    <java.version>17</java.version>
  </properties>

  <dependencies>
    <!-- Spring Boot core -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter</artifactId>
    </dependency>

    <!-- JDBC support (for writing SQL queries) -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-jdbc</artifactId>
    </dependency>

    <!-- H2 Database (embedded, no install needed) -->
    <dependency>
      <groupId>com.h2database</groupId>
      <artifactId>h2</artifactId>
      <scope>runtime</scope>
    </dependency>
  </dependencies>

  <build>
    <plugins>
      <plugin>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-maven-plugin</artifactId>
      </plugin>
    </plugins>
  </build>
</project>

After pasting: Save (Ctrl+S), then right-click the project → Maven → Update Project (or Alt+F5). Wait for the progress bar at bottom-right to finish downloading.

What did that just do?

Maven downloaded Spring Boot, H2, and JDBC libraries into your project. You can see them under "Maven Dependencies" in Package Explorer. These are the same as manually downloading JAR files — but Maven does it automatically and handles version compatibility.

    phone VARCHAR(15),
    role VARCHAR(20) NOT NULL,

    booking_id VARCHAR(20) PRIMARY KEY,

dashboard.jspUser dashboard after login
booking.jspBook a parcel form
tracking.jspTrack parcel status
Why WEB-INF?

Files inside WEB-INF/ cannot be accessed directly by typing a URL. Users can't type localhost:8080/WEB-INF/jsp/dashboard.jsp — they get a 404. Only your Servlets can forward requests to these files. This is a security feature.

Section 20

Sprint 4: CORS Config

CORS (Cross-Origin Resource Sharing) allows your frontend JavaScript to call your API. Without it, browsers block the request.

Create this file in com.nimbus.config:

src/main/java/com/nimbus/config/CorsConfig.java 
package com.nimbus.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.CorsRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class CorsConfig {

    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/api/**")
                        .allowedOrigins("*")
                        .allowedMethods("GET", "POST", "PUT", "DELETE");
            }
        };
    }
}
No new pom.xml changes needed

Sprint 3 already added spring-boot-starter-web which includes everything for REST APIs. No new dependencies for Sprint 4.

storage-api.js — The Sprint 4 Swap

This file has the exact same function names as storage.js, but instead of using LocalStorage, it calls your REST APIs. One file swap = frontend is now connected to the real backend.

How to Swap

In every HTML file that has this line:

Find this in your HTML files 
<script src="js/storage.js"></script>

Replace with:

Replace with this 
<script src="js/storage-api.js"></script>

The storage-api.js File

Create this file at src/main/resources/static/js/storage-api.js:

src/main/resources/static/js/storage-api.js 
/**
 * storage-api.js — REST API version of storage.js
 * Same function names, same return values.
 * Instead of localStorage, calls the Spring Boot REST API.
 */

const API_BASE = 'http://localhost:8080/api';

const NimbusDB = {

  // ===== AUTH =====

  async register(userData) {
    const response = await fetch(`${API_BASE}/auth/register`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(userData)
    });
    return await response.json();
  },

  async login(userId, password) {
    const response = await fetch(`${API_BASE}/auth/login`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ userId, password })
    });
    const data = await response.json();
    if (data.success) {
      // Store session locally (same as storage.js did)
      sessionStorage.setItem('currentUser', JSON.stringify(data.user));
    }
    return data;
  },

  getCurrentUser() {
    const user = sessionStorage.getItem('currentUser');
    return user ? JSON.parse(user) : null;
  },

  logout() {
    sessionStorage.removeItem('currentUser');
  },

  // ===== BOOKINGS =====

  async createBooking(bookingData) {
    const response = await fetch(`${API_BASE}/bookings`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(bookingData)
    });
    return await response.json();
  },

  async getBooking(bookingId) {
    const response = await fetch(`${API_BASE}/bookings/${bookingId}`);
    return await response.json();
  },

  async getBookingsByUser(userId) {
    const response = await fetch(`${API_BASE}/bookings/user/${userId}`);
    return await response.json();
  },

  // ===== TRACKING =====

  async trackParcel(bookingId) {
    const response = await fetch(`${API_BASE}/tracking/${bookingId}`);
    return await response.json();
  },

  // ===== ISSUES =====

  async createIssue(issueData) {
    const response = await fetch(`${API_BASE}/issues`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(issueData)
    });
    return await response.json();
  },

  async getIssuesByUser(userId) {
    const response = await fetch(`${API_BASE}/issues/user/${userId}`);
    return await response.json();
  }
};

// Make it globally available (same as storage.js)
window.NimbusDB = NimbusDB;
Important: async/await

The API version uses async/await because HTTP calls take time (network). If your HTML files call NimbusDB functions without await, you'll need to add it. Example: const user = await NimbusDB.login(id, pass); and the calling function must be async.

Why this approach works

The frontend doesn't care WHERE data comes from. It just calls NimbusDB.login(). In Sprint 3, that reads from localStorage. In Sprint 4, it calls POST /api/auth/login. Same interface, different backend. This is called the adapter pattern.

Section 27

Troubleshooting

Common errors and how to fix them.

ProblemFix
Red X on project / "Cannot resolve symbol" Right-click project → Maven → Update Project (Alt+F5). Check "Force Update". Click OK. Wait for downloads.
"Port 8080 already in use" Another app is using port 8080. Either: (a) close that app, or (b) change server.port=8081 in application.properties.
"No main class found" Make sure NimbusApplication.java has @SpringBootApplication annotation and a public static void main method.
"Table not found" / H2 error Check that schema.sql is in src/main/resources/ (not somewhere else). Check filename spelling. Delete nimbusdb.mv.db file from project root and restart.
Maven dependencies not downloading Check internet connection. Try: Right-click project → Maven → Update Project → check "Force Update of Snapshots/Releases".
"src/main/resources" folder missing Right-click src/main → New → Folder → name: resources. Then Alt+F5 to tell Maven about it.
JSP pages show 404 Check: (a) spring.mvc.view.prefix=/WEB-INF/jsp/ in properties, (b) JSP files are in src/main/webapp/WEB-INF/jsp/, (c) tomcat-embed-jasper dependency in pom.xml.
Console app can't connect to database Run NimbusApplication.java FIRST (creates the database file), stop it, THEN run ConsoleApp.java. Both use the same nimbusdb.mv.db file.
Eclipse shows wrong Java version Right-click project → Properties → Java Build Path → Libraries → double-click JRE System Library → select JavaSE-17.
Universal Fix

When in doubt: Right-click project → Maven → Update Project → Force Update → OK. This resolves 80% of Eclipse/Maven issues.
