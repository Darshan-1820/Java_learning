The Big Picture

From Console App to Web App

In Sprint 2, you built a console app — you type commands in the terminal. But users don't use terminals. They use browsers. Sprint 3 converts your console app into a web app that runs in a browser.

Think About It

Your Sprint 2 app works, but only YOU can use it on YOUR computer. What if 100 customers want to book parcels at the same time? They can't all sit at your terminal. They need a website they can open in Chrome.

Concept

What is a Web Server?

When you type a URL in your browser, something has to RECEIVE that request and send back HTML. That something is a web server.

Analogy

Think of a web server like a restaurant host. When you (the browser) walk in and say "I want table 5" (a URL), the host (web server) takes you to the right table (routes you to the right code). The chef (your Java code) prepares the food (HTML), and the waiter (HTTP response) brings it back to you.

In our case, the web server is Tomcat — and Spring Boot runs Tomcat inside your Java app automatically. You don't install Tomcat separately. When you run NimbusApplication.java, Spring Boot starts Tomcat on port 8080, and it's ready to receive HTTP requests.

In Nimbus

When you run the app and open http://localhost:8080/servlet/login, here's what happens:

  1. Your browser sends a GET request to localhost:8080
  2. Tomcat (running inside Spring Boot) receives it
  3. Tomcat sees the URL /servlet/login and routes it to LoginServlet
  4. LoginServlet.doGet() forwards to login.jsp
  5. The JSP generates HTML and sends it back to the browser
Concept

What is a Servlet?

A Servlet is a Java class that handles HTTP requests. When someone visits a URL, Tomcat routes that request to the right Servlet.

Think of it like this: In your Phase 1 frontend, you had login.html with a form. When the user clicks "Login", the form submitted to... nowhere (it used JavaScript and LocalStorage). Now, the form submits to a Servlet, which runs Java code on the server.

The Two Main Methods

Every Servlet has two methods that handle requests:

Think About It

GET = "Show me something" (display a page). POST = "Do something with this data" (submit a form). When you open a website, that's a GET. When you fill a form and click submit, that's a POST.

Concept

What is JSP?

JSP = JavaServer Pages. It's HTML with Java code embedded inside it. Instead of returning plain text from a Servlet, you forward to a JSP page that generates beautiful HTML.

In Sprint 2, your output was System.out.println("Welcome, Amit!") — just text in a terminal. With JSP, you generate a full HTML page with styling, buttons, and links.

Sprint 2 Output (Console)
System.out.println("Welcome, " + user.getFullName());

Shows text in terminal. No colors, no buttons, no links.

Sprint 3 Output (JSP)
<h2>Welcome, ${sessionScope.fullName}!</h2>

Shows a styled webpage in Chrome. Colors, buttons, navigation.

Key JSP Syntax

SyntaxWhat It DoesExample

<c:if>Conditional: show HTML only if condition is trueShow "Book Parcel" only for customers
<c:forEach>Loop: repeat HTML for each item in a listShow each booking in a table row

${param.userId}Reads URL parameters?userId=260401-AK01 → shows the ID
Why WEB-INF?

JSP files go inside src/main/webapp/WEB-INF/jsp/. The WEB-INF folder is special — users CANNOT directly access files inside it by typing a URL. They can only be reached through a Servlet's forward(). This is a security feature. If JSPs were outside WEB-INF, anyone could open them directly and bypass your login check.

Concept

What is Spring?

Spring is a framework that reduces boilerplate code. Without Spring, you write hundreds of lines of configuration. With Spring, you add annotations and Spring wires everything together.

The Core Idea: Dependency Injection

In Sprint 2, YOU created every object manually:

Sprint 2 — Manual object creation 
// YOU create objects manually, YOU manage them
UserDAO userDAO = new UserDAO();
BookingDAO bookingDAO = new BookingDAO();
IssueDAO issueDAO = new IssueDAO();

With Spring, you never write new UserRepository(). Spring creates it and passes it in automatically. This is called Dependency Injection (DI).

Sprint 3 — Spring injects dependencies 
@Service
public class UserService {
    private final UserRepository userRepository;  // Spring creates this

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;  // Spring passes it automatically
    }
}
Analogy

Imagine building a house. In Sprint 2, you personally go to the market, buy bricks, cement, and pipes, and carry them to the site. With Spring (DI), you just tell the builder "I need plumbing" and the builder delivers a plumber to your door. You don't care WHERE the plumber came from — you just use them.

Concept

What is Spring Data JPA?

Remember your DAOs from Sprint 2? You wrote 50+ lines of SQL for each one. Spring Data JPA eliminates all of that. You write an INTERFACE (not a class) with method NAMES, and Spring generates the SQL automatically.

Sprint 2: Manual SQL (UserDAO)
public User findByEmail(String email) {
    String sql = "SELECT * FROM users WHERE email = ?";
    PreparedStatement ps = conn.prepareStatement(sql);
    ps.setString(1, email);
    ResultSet rs = ps.executeQuery();
    if (rs.next()) {
        User user = new User();
        user.setUserId(rs.getString("user_id"));
        user.setEmail(rs.getString("email"));
        user.setFirstName(rs.getString("first_name"));
        // ... 10 more fields
        return user;
    }
    return null;
}

~20 lines of repetitive code for ONE query

Sprint 3: Spring Data JPA
public interface UserRepository
    extends JpaRepository<User, String> {

    Optional<User> findByEmail(String email);
    // That's it. Spring writes the SQL.
}

1 line. Spring generates the SQL from the method name.

How Method Name Queries Work

Spring reads the method name and generates SQL. The pattern is: findBy + FieldName + optional operators.

Method NameGenerated SQL
findByEmail(String email)SELECT * FROM users WHERE email = ?
findByUserIdAndPassword(String id, String pw)SELECT * FROM users WHERE user_id = ? AND password = ?
findByRole(String role)SELECT * FROM users WHERE role = ?
existsByEmail(String email)SELECT COUNT(*) > 0 FROM users WHERE email = ?
countByUserId(String userId)SELECT COUNT(*) FROM bookings WHERE user_id = ?
findByUserIdOrderByCreatedAtDesc(String id)SELECT * FROM bookings WHERE user_id = ? ORDER BY created_at DESC
Think About It

Spring reads findByUserIdAndPassword and breaks it down: findBy = SELECT, UserId = WHERE user_id = ?, And = AND, Password = password = ?. It's like a naming convention that becomes real SQL.

DAO vs Repository — What Changed?

Both DAOs and Repositories do the same job: talk to the database. The difference is HOW MUCH code YOU write.

AspectSprint 2 (DAO)Sprint 3 (Repository)
TypeRegular Java classJava interface
SQLYou write every SQL query manuallySpring generates SQL from method names
ConnectionYou manage Connection, PreparedStatement, ResultSetSpring handles all of it
Error handlingYou write try-catch for every querySpring handles exceptions

In Nimbus

The Sprint 2 DAOs (UserDAO.java, BookingDAO.java, IssueDAO.java) still exist in the project for learning and reference. Sprint 3 Repositories REPLACE them — same job, 90% less code. The Services now use Repositories, not DAOs. You can compare both approaches side by side.

NimbusApplication.java

The main class. Starts the embedded Tomcat server, loads all Spring beans, and serves everything.

Think About It

This is the entry point — like ConsoleApp.main() in Sprint 2. But instead of starting a menu loop, it starts a web server that listens for HTTP requests forever (until you stop it).

src/main/java/com/nimbus/NimbusApplication.java 
package com.nimbus;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.web.servlet.ServletComponentScan;

/**
 * NimbusApplication — Spring Boot main entry point.
 *
 * This starts the embedded Tomcat server with:
 *   - REST API controllers (Phase 4)
 *   - Servlet endpoints (Phase 3)
 *   - JSP pages (Phase 3)
 *   - Static frontend files (Phase 1 HTML/CSS/JS)
 *   - H2 database console
 *
 * HOW TO RUN IN ECLIPSE:
 *   Right-click this file > Run As > Spring Boot App
 *   (or Run As > Java Application)
 *
 * ENDPOINTS:
 *   http://localhost:8080/             → Static frontend (login.html)
 *   http://localhost:8080/h2-console   → Database browser
 *   http://localhost:8080/servlet/*    → Servlet/JSP pages (Phase 3)
 *   http://localhost:8080/api/*        → REST APIs (Phase 4)
 */
@SpringBootApplication
@ServletComponentScan
public class NimbusApplication {

    public static void main(String[] args) {
        SpringApplication.run(NimbusApplication.class, args);
        System.out.println("\n========================================");
        System.out.println("  NIMBUS is running!");
        System.out.println("  Frontend:    http://localhost:8080/login.html");
        System.out.println("  H2 Console:  http://localhost:8080/h2-console");
        System.out.println("  REST API:    http://localhost:8080/api/");
        System.out.println("  Servlets:    http://localhost:8080/servlet/login");
        System.out.println("========================================\n");
    }
}
Line-by-line explanation
1
@SpringBootApplication
This one annotation does THREE things: enables auto-configuration, component scanning (finds all @Service, @Repository, etc.), and marks this as a Spring config class.
2
@ServletComponentScan
Tells Spring Boot to scan for traditional Servlet classes (our Phase 3 servlets). Without this, Spring wouldn't register them.
3
SpringApplication.run(NimbusApplication.class, args);
Starts the entire application: creates Spring container, starts embedded Tomcat, loads all beans, opens port 8080. This one line does what would take 50+ lines of XML configuration.

ServletConfig.java

Registers our Servlets with Spring Boot. Maps URLs to Servlet classes.

Think About It

In a traditional Servlet project (no Spring Boot), you would configure URL mappings in a file called web.xml. Spring Boot replaces that with Java code — each @Bean method registers one Servlet and maps it to a URL pattern.

package com.nimbus.config;

import com.nimbus.servlet.BookingServlet;
import com.nimbus.servlet.DashboardServlet;
import com.nimbus.servlet.LoginServlet;
import com.nimbus.servlet.RegisterServlet;
import com.nimbus.servlet.TrackingServlet;
import org.springframework.boot.web.servlet.ServletRegistrationBean;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

/**
 * ServletConfig — Registers traditional Servlets inside Spring Boot.
 *
 * This lets us demonstrate Phase 3 Servlets alongside Phase 4 REST controllers.
 * Each Servlet is mapped to /servlet/* URLs.
 *
 * In a pure Servlet project (no Spring Boot), you'd configure these in web.xml.
 * Spring Boot's ServletRegistrationBean is the Java-config equivalent.

Spring Data JPA interface for the users table. Replaces UserDAO from Sprint 2.

src/main/java/com/nimbus/repository/UserRepository.java 
package com.nimbus.repository;

import com.nimbus.model.User;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

import java.util.List;
import java.util.Optional;

/**
 * UserRepository — Spring Data JPA interface for the users table.
 *
 * This replaces UserDAO from Phase 2. Instead of writing SQL manually,
 * Spring generates the SQL from method names automatically.
 *
 * Phase 2: UserDAO.findByEmail("amit@email.com")  → manual SQL + PreparedStatement
 * Phase 3: userRepository.findByEmail("amit@email.com") → Spring generates the SQL
 *
 * JpaRepository provides: save(), findById(), findAll(), deleteById(), count(), etc.
 * We just add custom query methods below.
 */
@Repository
public interface UserRepository extends JpaRepository<User, String> {

    /** Find a user by email address (for duplicate check). */
    Optional<User> findByEmail(String email);

    /** Validate login: find user by both userId and password. */
    Optional<User> findByUserIdAndPassword(String userId, String password);

    /** Get all users with a specific role (customer, officer, support). */
    List<User> findByRole(String role);

    /** Check if an email already exists. */
    boolean existsByEmail(String email);
}
Line-by-line explanation
1
public interface UserRepository
Notice: this is an INTERFACE, not a class. You never write the implementation. Spring creates it at runtime.
2
extends JpaRepository<User, String>
JpaRepository<Entity, IdType>. User is the entity class, String is the type of its primary key (userId). This gives you save(), findById(), findAll(), delete(), count() for FREE.
3
Optional<User> findByEmail(String email);
Spring reads "findByEmail" and generates: SELECT * FROM users WHERE email = ?. Returns Optional because the user might not exist.
4
boolean existsByEmail(String email);
Spring generates: SELECT COUNT(*) > 0 FROM users WHERE email = ?. Returns true/false. Used during registration to check duplicates.

package com.nimbus.repository;

import com.nimbus.model.Booking;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;
import org.springframework.stereotype.Repository;

import java.time.LocalDateTime;
import java.util.List;

/**
 * BookingRepository — Spring Data JPA interface for the bookings table.
 * Replaces BookingDAO from Phase 2.
 */
@Repository
public interface BookingRepository extends JpaRepository<Booking, String> {

    /** Get all bookings for a user, sorted newest first. */
    List<Booking> findByUserIdOrderByCreatedAtDesc(String userId);

    /** Get bookings by status. */
    List<Booking> findByParStatus(String parStatus);

    /** Get bookings by user and status. */
    List<Booking> findByUserIdAndParStatus(String userId, String parStatus);

    /** Get bookings within a date range (for history filtering). */
    @Query("SELECT b FROM Booking b WHERE " +
           "(:userId IS NULL OR b.userId = :userId) AND " +
           "(:startDate IS NULL OR b.createdAt >= :startDate) AND " +
           "(:endDate IS NULL OR b.createdAt <= :endDate) " +
           "ORDER BY b.createdAt DESC")
    List<Booking> findByDateRange(
            @Param("userId") String userId,
            @Param("startDate") LocalDateTime startDate,
            @Param("endDate") LocalDateTime endDate);

    /** Count bookings for a specific user. */
    long countByUserId(String userId);
}
Think About It

Notice the @Query annotation on findByDateRange(). When the method name gets too complex (multiple optional filters), you can write JPQL (Java Persistence Query Language) directly. JPQL looks like SQL but uses Java class names instead of table names. Booking b instead of bookings b.

UserService.java

Business logic for user operations: registration, login, validation. Uses UserRepository instead of UserDAO.

Why a Service Layer?

Why not call the Repository directly from the Servlet? Because the Service layer contains BUSINESS RULES: "validate the email format", "check for duplicate accounts", "generate a user ID". If you put business logic in the Servlet, and later build a REST API (Sprint 4), you'd have to duplicate all that logic. With a Service, both Servlets and Controllers share the same business rules.

src/main/java/com/nimbus/service/UserService.java 
package com.nimbus.service;

import com.nimbus.model.User;
import com.nimbus.repository.UserRepository;
import com.nimbus.util.IdGenerator;
import com.nimbus.util.InputValidator;
import org.springframework.stereotype.Service;

import java.time.LocalDateTime;
import java.util.List;
import java.util.Optional;
import java.util.stream.Collectors;

/**
 * UserService — Business logic for user operations.
 * Used by both Servlets (Phase 3) and REST Controllers (Phase 4).
 *
 * This layer sits between the Controller/Servlet and the Repository.
 * It contains validation, business rules, and data transformation.
 */
@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    /**
     * Register a new customer.
     * Validates input, checks for duplicate email, generates user ID, and saves.
     */
    public User register(String firstName, String middleName, String lastName,
                         String email, String countryCode, String mobile,
                         String street, String zipCode, String city, String state, String country,
                         String password, String confirmPassword, String preferences) {

        // Validate input
        List<String> errors = InputValidator.validateRegistration(
                firstName, lastName, email, mobile, street, zipCode, city, state, password, confirmPassword);

        if (!errors.isEmpty()) {
            throw new IllegalArgumentException("Validation failed: " + String.join("; ", errors));
        }

        // Check duplicate email
        String normalizedEmail = email.trim().toLowerCase();
        if (userRepository.existsByEmail(normalizedEmail)) {
            throw new IllegalArgumentException("This email is already registered.");
        }

        // Generate User ID
        List<String> existingIds = userRepository.findAll().stream()
                .map(User::getUserId)
                .collect(Collectors.toList());
        String userId = IdGenerator.generateUserId(firstName.trim(), lastName.trim(), existingIds);

        // Create User
        User user = new User(userId, firstName.trim(),
                middleName != null ? middleName.trim() : "",
                lastName.trim(), normalizedEmail,
                countryCode != null ? countryCode : "+91",
                mobile.trim(), street.trim(), zipCode.trim(),
                city.trim(), state.trim(),
                country != null && !country.isEmpty() ? country.trim() : "India",
                password,
                "customer",
                preferences != null ? preferences : "Email notifications");

        return userRepository.save(user);
    }

    /**
     * Register an admin/officer user (for registerAdmin endpoint).
     */
    public User registerAdmin(String firstName, String lastName, String email,
                              String mobile, String password, String role) {

        String normalizedEmail = email.trim().toLowerCase();
        if (userRepository.existsByEmail(normalizedEmail)) {
            throw new IllegalArgumentException("This email is already registered.");
        }

        List<String> existingIds = userRepository.findAll().stream()
                .map(User::getUserId).collect(Collectors.toList());

        // Officers and support get different ID prefix
        String userId;
        if ("officer".equals(role)) {
            long count = userRepository.findByRole("officer").size();
            userId = "OFF-" + String.format("%03d", count + 1);
        } else if ("support".equals(role)) {
            long count = userRepository.findByRole("support").size();
            userId = "SUP-" + String.format("%03d", count + 1);
        } else {
            userId = IdGenerator.generateUserId(firstName.trim(), lastName.trim(), existingIds);
        }

        User user = new User(userId, firstName.trim(), "", lastName.trim(),
                normalizedEmail, "+91", mobile.trim(),
                "Office", "000000", "Office", "Office", "India",
                password, role, "");

        return userRepository.save(user);
    }

    /**
     * Validate login credentials.
     * Returns the user if valid, throws exception if invalid.
     */
    public User validateLogin(String userId, String password) {
        if (InputValidator.isBlank(userId) || InputValidator.isBlank(password)) {
            throw new IllegalArgumentException("User ID and password are required.");
        }

        Optional<User> user = userRepository.findByUserIdAndPassword(userId.trim(), password);
        if (user.isEmpty()) {
            throw new IllegalArgumentException("Invalid credentials.");
        }

        return user.get();
    }

    /** Get a user by ID. */
    public User getUserById(String userId) {
        return userRepository.findById(userId)
                .orElseThrow(() -> new IllegalArgumentException("User not found: " + userId));
    }

    /** Get all users. */
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    /** Get users by role. */
    public List<User> getUsersByRole(String role) {
        return userRepository.findByRole(role);
    }
}

                                  String recPin, String recMobile, int parWeightGram,
                                  String parContentsDescription, String parDeliveryType,
                                  String parPackingPreference, String paymentMethod) {

        // Validate required fields
        if (recName == null || recName.trim().isEmpty()) {
            throw new IllegalArgumentException("Receiver name is required.");
        }
        if (recAddress == null || recAddress.trim().isEmpty()) {
            throw new IllegalArgumentException("Receiver address is required.");
        }
        if (parWeightGram <= 0) {
            throw new IllegalArgumentException("Weight must be greater than 0.");
        }

        // Generate Booking ID
        int totalBookings = (int) bookingRepository.count();
        String bookingId = IdGenerator.generateBookingId(totalBookings);

        // Calculate cost
        double cost = PricingCalculator.calculateSimple(
                parWeightGram,
                parDeliveryType != null ? parDeliveryType : "Standard",
                parPackingPreference != null ? parPackingPreference : "Standard Packaging");

        // Calculate estimated pickup and dropoff times
        LocalDateTime now = LocalDateTime.now();
        LocalDateTime pickupTime = now.plusDays(1);
        int deliveryDays = switch (parDeliveryType != null ? parDeliveryType : "Standard") {
            case "Express" -> 3;
            case "Overnight" -> 1;
            default -> 7;
        };
        LocalDateTime dropoffTime = pickupTime.plusDays(deliveryDays);

        // Create booking
        Booking booking = new Booking();
        booking.setBookingId(bookingId);
        booking.setUserId(userId);
        booking.setRecName(recName.trim());
        booking.setRecAddress(recAddress.trim());
        booking.setRecPin(recPin != null ? recPin.trim() : "");
        booking.setRecMobile(recMobile != null ? recMobile.trim() : "");
        booking.setParWeightGram(parWeightGram);
        booking.setParContentsDescription(parContentsDescription != null ? parContentsDescription.trim() : "");
        booking.setParDeliveryType(parDeliveryType != null ? parDeliveryType : "Standard");
        booking.setParPackingPreference(parPackingPreference != null ? parPackingPreference : "Standard Packaging");
        booking.setParPickupTime(pickupTime);
        booking.setParDropoffTime(dropoffTime);
        booking.setParServiceCost(cost);
        booking.setParPaymentTime(now);
        booking.setPaymentMethod(paymentMethod != null ? paymentMethod : "card");
        booking.setParStatus("Booked");
        booking.setCreatedAt(now);

        return bookingRepository.save(booking);
    }

    /**
     * Track a booking by ID.
     * Customers can only see their own bookings; officers can see any.
     */
    public Booking trackBooking(String bookingId, String userId, String role) {
        Booking booking = bookingRepository.findById(bookingId)
                .orElseThrow(() -> new IllegalArgumentException("Booking not found: " + bookingId));

        // Security: customers can only track their own bookings
        if ("customer".equals(role) && !booking.getUserId().equals(userId)) {
            throw new SecurityException("Access denied. This booking belongs to another user.");
        }

        return booking;
    }

    /**
     * Update delivery status (officer operation).
     */
    public Booking updateDeliveryStatus(String bookingId, String newStatus) {
        Booking booking = bookingRepository.findById(bookingId)
                .orElseThrow(() -> new IllegalArgumentException("Booking not found: " + bookingId));

        // Validate status value
        List<String> validStatuses = List.of("Booked", "Picked Up", "In Transit", "Delivered", "Returned");
        if (!validStatuses.contains(newStatus)) {
            throw new IllegalArgumentException("Invalid status: " + newStatus);
        }

        booking.setParStatus(newStatus);
        return bookingRepository.save(booking);
    }

    /**
     * Update pickup and dropoff times (officer scheduling).
     */
    public Booking updatePickupDropoff(String bookingId, LocalDateTime pickupTime, LocalDateTime dropoffTime) {
        Booking booking = bookingRepository.findById(bookingId)
                .orElseThrow(() -> new IllegalArgumentException("Booking not found: " + bookingId));

        booking.setParPickupTime(pickupTime);
        booking.setParDropoffTime(dropoffTime);
        return bookingRepository.save(booking);
    }

    /**
     * Get booking history with optional filters.
     */
    public List<Booking> getHistory(String userId, String role, String startDate, String endDate) {
        if ("customer".equals(role)) {
            // Customers see only their own bookings
            if (startDate != null || endDate != null) {
                LocalDateTime start = startDate != null ?
                        LocalDateTime.parse(startDate + "T00:00:00") : null;
                LocalDateTime end = endDate != null ?
                        LocalDateTime.parse(endDate + "T23:59:59") : null;
                return bookingRepository.findByDateRange(userId, start, end);
            }
            return bookingRepository.findByUserIdOrderByCreatedAtDesc(userId);
        } else {
            // Officers see all bookings (optionally filtered)
            if (userId != null || startDate != null || endDate != null) {
                LocalDateTime start = startDate != null ?
                        LocalDateTime.parse(startDate + "T00:00:00") : null;
                LocalDateTime end = endDate != null ?
                        LocalDateTime.parse(endDate + "T23:59:59") : null;
                return bookingRepository.findByDateRange(userId, start, end);
            }
            return bookingRepository.findAll();
        }
    }

    /** Get a single booking by ID. */
    public Booking getBookingById(String bookingId) {
        return bookingRepository.findById(bookingId)
                .orElseThrow(() -> new IllegalArgumentException("Booking not found: " + bookingId));
    }

LoginServlet.java

Handles login. GET shows the login page, POST processes the login form.

How Login Works

This is the most important Servlet to understand. Here's the full flow:

  1. User opens /servlet/login → doGet() runs → forwards to login.jsp → user sees the login form
  2. User types ID + password and clicks "Login" → form sends POST to /servlet/login
  3. doPost() runs → extracts userId and password from the form → queries the database
  4. If credentials match: creates an HTTP session (stores user data on server) → redirects to dashboard
  5. If credentials don't match: sets error message → forwards back to login.jsp → user sees "Invalid User ID or Password"
src/main/java/com/nimbus/servlet/LoginServlet.java 
package com.nimbus.servlet;

import com.nimbus.model.User;
import com.nimbus.repository.UserRepository;
import jakarta.servlet.ServletException;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;
import org.springframework.web.context.WebApplicationContext;
import org.springframework.web.context.support.WebApplicationContextUtils;

import java.io.IOException;
import java.util.Optional;

/**
 * LoginServlet — Phase 3 traditional Servlet for login.
 *
 * GET  /servlet/login  → Shows the login JSP page
 * POST /servlet/login  → Processes login form, creates session, redirects to dashboard
 *
 * This demonstrates the Servlet lifecycle:
 *   1. Client sends HTTP request
 *   2. Tomcat routes to this Servlet based on URL mapping
 *   3. doGet/doPost handles the request
 *   4. Forward to JSP (view) or redirect to another URL
 */
public class LoginServlet extends HttpServlet {

    /**
     * GET — Display the login page.
     */
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {

        // Forward to the JSP view
        request.getRequestDispatcher("/WEB-INF/jsp/login.jsp").forward(request, response);
    }

    /**
     * POST — Process login form submission.
     */
    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {

        String userId = request.getParameter("userId");
        String password = request.getParameter("password");

        // Get Spring's UserRepository from the application context
        WebApplicationContext ctx = WebApplicationContextUtils
                .getWebApplicationContext(getServletContext());
        UserRepository userRepository = ctx.getBean(UserRepository.class);

        // Validate login
        Optional<User> userOpt = userRepository.findByUserIdAndPassword(userId, password);

        if (userOpt.isPresent()) {
            User user = userOpt.get();

            // Create HTTP session (stores user info on the server)
            HttpSession session = request.getSession();
            session.setAttribute("user", user);
            session.setAttribute("userId", user.getUserId());
            session.setAttribute("fullName", user.getFullName());
            session.setAttribute("role", user.getRole());

            // Redirect to dashboard JSP
            response.sendRedirect(request.getContextPath() + "/servlet/dashboard?role=" + user.getRole());
        } else {
            // Login failed — show error on login page
            request.setAttribute("error", "Invalid User ID or Password.");
            request.getRequestDispatcher("/WEB-INF/jsp/login.jsp").forward(request, response);
        }
    }
}
Key concepts explained
1
request.getParameter("userId")
Gets the value the user typed in the form's "userId" input field. Like reading what someone wrote on a paper form.
2
WebApplicationContextUtils.getWebApplicationContext(getServletContext())
Since Servlets are NOT Spring-managed beans, they can't use @Autowired. This line manually gets Spring's container so we can retrieve the UserRepository bean.
3
HttpSession session = request.getSession();
Creates a server-side session for this user. Think of it as a locker at the server — you store user data there, and it persists across multiple page visits until they log out or close the browser.
4
response.sendRedirect(...) vs request.getRequestDispatcher(...).forward(...)
Redirect: tells the browser "go to this new URL" (browser makes a NEW request). Forward: server internally sends the request to another page (browser URL doesn't change). Use redirect after success (POST-Redirect-GET pattern), forward for showing errors on the same page.

Role-Based Access Control (RBAC) in JSP

Look at the three <c:if> blocks. Each one checks ${sessionScope.role} — the role stored in the HTTP session when the user logged in. Customers see "Book Parcel" and "Track Parcel". Officers see those PLUS "Update Delivery" and "Manage Issues". Support agents see only issue-related cards. Same JSP file, different content per role.

How to Run

Get the web app running in your browser.

  1. Open the project in Eclipse. Navigate to src/main/java/com/nimbus/NimbusApplication.java
  2. Right-click NimbusApplication.java → Run As → Java Application (or Spring Boot App if available)
  3. Wait for the console to show:
    NIMBUS is running!
    This means Tomcat started on port 8080.
  4. Open your browser (Chrome) and go to: http://localhost:8080/servlet/login
  5. Login with: 260401-AK01 / Amit@123
  6. You should see the Customer Dashboard with "Book Parcel", "Track Parcel", etc.
Common Issues

Port already in use: If you see "Address already in use: bind", another app is using port 8080. Stop the previous run first (red square in Eclipse console).

Can't find NimbusApplication: Make sure you're running the file in com.nimbus package, not in a test folder.

Dependencies not found: Right-click project → Maven → Update Project (Alt+F5).

Test H2 Console

H2 is the in-memory database. You can browse its tables and run SQL queries directly in the browser.

  1. Make sure the app is running (NimbusApplication)
  2. Open browser → http://localhost:8080/h2-console
  3. Set the JDBC URL to: jdbc:h2:file:./nimbusdb;MODE=MySQL
  4. Username: sa, Password: (leave empty)
  5. Click Connect
  6. On the left panel, you'll see all tables: USERS, BOOKINGS, ISSUES
  7. Try running: SELECT * FROM USERS; — you should see the seeded demo users
Why This Matters

The H2 Console is your window into the database. Every time you register a user through the JSP form, you can immediately verify it was saved by running SELECT * FROM USERS in the H2 Console. Every booking, every issue — you can see all the data your Servlets are creating.

What's Next

Moving to Sprint 4: REST APIs

Your web app works in the browser! Users can login, book parcels, and track them through JSP pages. But there's a problem...

Think About It

Right now, your server generates the HTML (JSP pages). But what if you want a mobile app? Or a React frontend? They can't use JSP pages. They need RAW DATA (JSON), not HTML. That's what REST APIs do — they return data, and the frontend decides how to display it.

In Sprint 4, you'll build REST APIs — the modern way to connect frontend and backend. Your existing HTML/CSS/JS frontend from Phase 1 will call these APIs instead of using LocalStorage. The same APIs can serve a web app, mobile app, or any client.

Traditional approach

Decoupled (backend = API, frontend = separate)

Works with any client (web, mobile, CLI)

Modern approach (industry standard)
