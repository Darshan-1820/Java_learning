The 5 Templates That Solve Every PRA

Memorize these. Type them without thinking. They cover 95% of what PRA asks.

How to use this page: Don't just read — TYPE each template into the IDE from memory. Do this 3 times. By the third time, your fingers should type them automatically.
1
The POJO Class
When you see: "Create a class X with the following attributes..."
class ClassName {
    private Type field1;
    private Type field2;
    private Type field3;

    public ClassName(Type field1, Type field2, Type field3) {
        this.field1 = field1;
        this.field2 = field2;
        this.field3 = field3;
    }

    public Type getField1() { return field1; }
    public void setField1(Type field1) { this.field1 = field1; }
    // ... repeat for each field
}
What changes: class name, field names, field types. Structure is ALWAYS the same.

2
Read Input
When you see: "Read N objects from input..."
Scanner sc = new Scanner(System.in);
int n = Integer.parseInt(sc.nextLine().trim());
List<ClassName> list = new ArrayList<>();

for (int i = 0; i < n; i++) {
    int id = Integer.parseInt(sc.nextLine().trim());
    String name = sc.nextLine().trim();
    double value = Double.parseDouble(sc.nextLine().trim());
    list.add(new ClassName(id, name, value));
}
// Read method parameters AFTER the loop
String param1 = sc.nextLine().trim();
int param2 = Integer.parseInt(sc.nextLine().trim());
Golden rule: ALWAYS use Integer.parseInt(sc.nextLine().trim()). NEVER use sc.nextInt(). The nextInt() leaves a newline in the buffer and your next read will break.
Nested Reading (Brand with Mobiles)
for (int i = 0; i < n; i++) {
    int brandId = Integer.parseInt(sc.nextLine().trim());
    String brandName = sc.nextLine().trim();
    int mobileCount = Integer.parseInt(sc.nextLine().trim());

    List<Mobile> mList = new ArrayList<>();
    for (int j = 0; j < mobileCount; j++) {
        // read mobile fields
        mList.add(new Mobile(...));
    }
    list.add(new Brand(brandId, brandName, mList));
}
3
Filter Method
When you see: "return all items that match..." or "find items where..."
public static List<ClassName> filterBy(List<ClassName> list, String value) {
    List<ClassName> result = new ArrayList<>();
    for (ClassName item : list) {
        if (item.getField().equalsIgnoreCase(value)) {
            result.add(item);
        }
    }
    return result;
}
What changes: the condition inside the if(). Structure is ALWAYS the same.
Filter by number
if (item.getGains() < 0)         // negative values
if (item.getStorage() >= min && item.getStorage() <= max)  // range
Filter through nested list
for (Brand b : brands) {
    for (Mobile m : b.getMobileList()) {
        if (m.getStorage() >= min) {
            result.add(m);
        }
    }
}
4
Aggregate Method
When you see: "calculate average/sum/count of..."
public static double calculateX(List<ClassName> list) {
    double sum = 0;
    int count = 0;
    for (ClassName item : list) {
        if (/* condition */) {
            sum += item.getValue();
            count++;
        }
    }
    return count > 0 ? sum / count : 0;
}
Return Map (average per group)
public static Map<String, Double> avgByGroup(List<Brand> brands) {
    Map<String, Double> result = new LinkedHashMap<>();
    for (Brand b : brands) {
        double sum = 0;
        for (Mobile m : b.getMobileList()) {
            sum += m.getPrice();
        }
        result.put(b.getBrandName(), sum / b.getMobileList().size());
    }
    return result;
}
5
Main with Try-Catch
When you see: "throw exception if..."
public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);
    // Template 2: Read input
    // Read method params

    // Method 1 — OWN try-catch
    try {
        ResultType r1 = method1(list, param);
        // print r1
    } catch (CustomException1 e) {
        System.out.println("CustomException1: " + e.getMessage());
    }

    // Method 2 — SEPARATE try-catch
    try {
        ResultType r2 = method2(list, param);
        // print r2
    } catch (CustomException2 e) {
        System.out.println("CustomException2: " + e.getMessage());
    }
}
THE RULE: Each method gets its OWN try-catch. Never put both in one try block. If method1 throws, method2 must still run.
