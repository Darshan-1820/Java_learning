Mock 4 — Question 1

Mobile Brand Service

Major 55–75 min

Question Statement

Create the below classes and attributes:

Brand:

  • brandId: Integer
  • brandName: String
  • mobileList: List<Mobile>

Mobile:

  • modelNumber: Integer
  • modelName: String
  • price: Double
  • storageCapacity: Integer

Implement two methods in a Solution class:

Method 1: calculateAveragePrice(List<Brand> brands)

  • Calculate the average price of mobiles for each brand.
  • Return a Map<String, Double> of brand names to their average mobile price.
  • If average cannot be determined (brand has no mobiles), throw custom exception AverageCannotBeCalculatedException with message "Average cannot be calculated".

Method 2: findMobilesInStorageRange(List<Brand> brands, int minStorage, int maxStorage)

  • Returns a list of mobiles whose storage capacity falls within the given range (inclusive).
  • If no mobile found, throw custom exception NoMobileDetailsFoundException with message "No mobile found".

Output format for Method 1:

Average Price of each brand:
<BrandName>: <Average>

Output format for Method 2:

Mobiles in the storage range (<min>GB - <max>GB):
Brand: <BrandName>, Model Number: <ModelNumber>, Model Name: <ModelName>, Price: <Price>, Storage: <Storage>GB

Notes: Maintain insertion order. Case insensitive comparisons. Catch exceptions in main and print: AverageCannotBeCalculatedException: Average cannot be calculated or NoMobileDetailsFoundException: No mobile found.

Analysis

What This Question Tests

This is a heavy PRA major question. It combines composition, Maps, custom exceptions, and nested loops — all in one.

Concepts Tested
  • Composition — Brand HAS a List<Mobile> (one-to-many relationship)
  • Map return type — Method 1 returns a Map, not void. Main prints it.
  • TWO custom exceptions — Each method throws a different one
  • LinkedHashMap — "Insertion order" means you need LinkedHashMap, not HashMap
  • Nested loops — Loop brands, then loop each brand's mobiles

Composition: Brand contains a List of Mobile objects
Key Insight: "return Map, not void"

The method returns data to main. Main then prints it. This is different from previous questions where methods printed directly. If you make it void and print inside the method, you lose control over exception handling in main.

Method 1

calculateAveragePrice

Returns a Map<String, Double> — brand name mapped to average price. Uses LinkedHashMap to maintain insertion order.

Solution.java — Method 1 
public static Map<String, Double> calculateAveragePrice(List<Brand> brands)
        throws AverageCannotBeCalculatedException {
    Map<String, Double> result = new LinkedHashMap<>();
    for (Brand b : brands) {
        List<Mobile> mobiles = b.getMobileList();
        if (mobiles == null || mobiles.isEmpty()) {
            throw new AverageCannotBeCalculatedException("Average cannot be calculated");
        }
        double sum = 0;
        for (Mobile m : mobiles) {
            sum += m.getPrice();
        }
        result.put(b.getBrandName(), sum / mobiles.size());
    }
    return result;
}
Line-by-line explanation
1
Map<String, Double> result = new LinkedHashMap<>();
LinkedHashMap (not HashMap) because the question says "insertion order." HashMap does NOT guarantee order.
2
if (mobiles == null || mobiles.isEmpty())
If a brand has zero mobiles, we can't calculate average. Throw the exception immediately.
3
sum += m.getPrice();
Sum all mobile prices for this brand.
4
result.put(b.getBrandName(), sum / mobiles.size());
Average = total / count. Put into map with brand name as key.
LinkedHashMap, NOT HashMap

If the question mentions "insertion order" or "order should be maintained," always use LinkedHashMap. A plain HashMap randomizes the order and your output will not match expected output — even if the values are correct.

Method 2

findMobilesInStorageRange

Returns a List<Mobile> of mobiles whose storage capacity is within the given range (inclusive). The tricky part: main needs the brand name for printing, but the method only returns mobiles.

Solution.java — Method 2 
public static List<Mobile> findMobilesInStorageRange(List<Brand> brands,
        int minStorage, int maxStorage) throws NoMobileDetailsFoundException {
    List<Mobile> result = new ArrayList<>();
    for (Brand b : brands) {
        for (Mobile m : b.getMobileList()) {
            if (m.getStorageCapacity() >= minStorage && m.getStorageCapacity() <= maxStorage) {
                result.add(m);
            }
        }
    }
    if (result.isEmpty()) {
        throw new NoMobileDetailsFoundException("No mobile found");
    }
    return result;
}
How Main Gets the Brand Name

The method returns List<Mobile> — but the output format needs the brand name too. In main, loop through brands and their mobiles again, checking if each mobile is in the result list. This way you have access to both the brand and the mobile for printing.

Main

Main Method — Reading Input & Printing

The main method reads brands with their mobiles, reads the storage range, calls both methods, and handles exceptions with try-catch.

Solution.java — main method 
public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);
    int brandCount = sc.nextInt();
    List<Brand> brands = new ArrayList<>();

    for (int i = 0; i < brandCount; i++) {
        int brandId = sc.nextInt();
        sc.nextLine(); // consume newline
        String brandName = sc.nextLine();
        int mobileCount = sc.nextInt();
        List<Mobile> mobiles = new ArrayList<>();

        for (int j = 0; j < mobileCount; j++) {
            int modelNumber = sc.nextInt();
            sc.nextLine(); // consume newline
            String modelName = sc.nextLine();
            double price = sc.nextDouble();
            int storage = sc.nextInt();
            mobiles.add(new Mobile(modelNumber, modelName, price, storage));
        }

        brands.add(new Brand(brandId, brandName, mobiles));
    }

    int minStorage = sc.nextInt();
    int maxStorage = sc.nextInt();

    // Method 1: Calculate average price per brand
    try {
        Map<String, Double> avgMap = calculateAveragePrice(brands);
        System.out.println("Average Price of each brand:");
        for (Map.Entry<String, Double> entry : avgMap.entrySet()) {
            System.out.println(entry.getKey() + ": " + entry.getValue());
        }
    } catch (AverageCannotBeCalculatedException e) {
        System.out.println("AverageCannotBeCalculatedException: " + e.getMessage());
    }

    // Method 2: Find mobiles in storage range
    try {
        List<Mobile> rangeResult = findMobilesInStorageRange(brands, minStorage, maxStorage);
        System.out.println("Mobiles in the storage range (" + minStorage + "GB - " + maxStorage + "GB):");
        for (Brand b : brands) {
            for (Mobile m : b.getMobileList()) {
                if (rangeResult.contains(m)) {
                    System.out.println("Brand: " + b.getBrandName()
                        + ", Model Number: " + m.getModelNumber()
                        + ", Model Name: " + m.getModelName()
                        + ", Price: " + m.getPrice()
                        + ", Storage: " + m.getStorageCapacity() + "GB");
                }
            }
        }
    } catch (NoMobileDetailsFoundException e) {
        System.out.println("NoMobileDetailsFoundException: " + e.getMessage());
    }
}
The "Printing After Exception" Pattern

Look at hidden test case 1: the brand has 0 mobiles. Method 1 throws the exception. But Method 2 still runs because each method call is in its own try-catch. If you put both methods in the same try block, Method 2 would get skipped when Method 1 throws. Always use separate try-catch blocks for each method.

public class Solution {

    public static Map<String, Double> calculateAveragePrice(List<Brand> brands)
            throws AverageCannotBeCalculatedException {
        Map<String, Double> result = new LinkedHashMap<>();
        for (Brand b : brands) {
            List<Mobile> mobiles = b.getMobileList();
            if (mobiles == null || mobiles.isEmpty()) {
                throw new AverageCannotBeCalculatedException("Average cannot be calculated");
            }
            double sum = 0;
            for (Mobile m : mobiles) {
                sum += m.getPrice();
            }
            result.put(b.getBrandName(), sum / mobiles.size());
        }
        return result;
    }

    public static List<Mobile> findMobilesInStorageRange(List<Brand> brands,
            int minStorage, int maxStorage) throws NoMobileDetailsFoundException {
        List<Mobile> result = new ArrayList<>();
        for (Brand b : brands) {
            for (Mobile m : b.getMobileList()) {
                if (m.getStorageCapacity() >= minStorage && m.getStorageCapacity() <= maxStorage) {
                    result.add(m);
                }
            }
        }
        if (result.isEmpty()) {
            throw new NoMobileDetailsFoundException("No mobile found");
        }
        return result;
    }

Watch Out

Common Mistakes

Wrong: void return type

public static void calculateAveragePrice(...)

If the method returns void, main cannot get the Map to print it. You end up printing inside the method, which makes exception handling messy.

Correct: Map return type

public static Map<String, Double> calculateAveragePrice(...)

Method returns data. Main prints it. Exception handling stays clean in main's try-catch.

Mistake What Goes Wrong Fix
Using HashMap instead of LinkedHashMap Output order is random — brands print in wrong order Use LinkedHashMap when insertion order matters
Making method void instead of return type Cannot get the result in main, cannot print after header text Read the return type carefully: Map<String, Double>
Both methods in single try-catch If Method 1 throws, Method 2 never executes Use separate try-catch blocks for each method call
Throwing exception in main instead of method The method signature says throws — the method must throw throw inside method, catch inside main
Printing "Mobiles in the storage range..." before try Header prints even when exception is thrown Print the header inside the try block, after successful method call
"throw" vs "catch" Confusion

The method throws the exception (creates it and sends it up). The main method catches the exception (receives it and prints the message). Never catch inside the method itself — that defeats the purpose. The whole point is that main decides what to do when something goes wrong.
