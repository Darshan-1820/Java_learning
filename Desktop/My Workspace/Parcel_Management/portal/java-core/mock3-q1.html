Mock 3 — Question 1

Author-Book BookStore

Major 45–55 min

Question Statement

Create the below Classes and attributes.

Author:

  • authorId: Integer
  • authorName: String

Book:

  • bookId: Integer
  • title: String
  • genre: String
  • price: double
  • author: Author

Create a business class and implement the following functionalities:

getBooksBelongingToAGenre:

  • Takes a list of Book and genre as input parameters.
  • Find all books belonging to the input genre and return the list of books matching the genre.
  • If no matching genre is found, print error message "Genre is not available in the list"

calculateDiscountedPrice:

  • Takes a list of Book, genre and discount percentage as input parameters.
  • Find books matching the genre and apply the discount to the book.
  • Return list of books matching the genre after applying the discount. Output should be printed followed by: "Discounted <genre> Books:"
  • If no matching genre is found, print error message "Discounted books are unavailable in the given genre."
  • Discounted Price = Price - (Price * discount%)

Notes:

  • All string comparisons need to be case-insensitive.
  • Assume an author has written only one book.

Input order:

  1. Read number of Author objects
  2. Read Author attributes (authorId, authorName)
  3. Read Book attributes (bookId, title, genre, price) — author is the one just read
  4. Repeat for all
  5. Read parameters for methods (genre for method 1, genre + discount for method 2)

Method 1

getBooksBelongingToAGenre

Classic filter pattern: loop through all books, check if genre matches, collect matches into a result list.

Solution.java — Method 1 
public static List<Book> getBooksBelongingToAGenre(List<Book> books, String genre) {
    List<Book> result = new ArrayList<>();
    for (Book book : books) {
        if (book.getGenre().equalsIgnoreCase(genre)) {
            result.add(book);
        }
    }
    if (result.isEmpty()) {
        System.out.println("Genre is not available in the list");
    }
    return result;
}
Line-by-line explanation
1
List<Book> result = new ArrayList<>();
Empty result list. We'll fill it with books that match the genre.
2
for (Book book : books)
Loop through every Book in the input list. Enhanced for-loop — cleanest way.
3
book.getGenre().equalsIgnoreCase(genre)
Case-insensitive comparison. "Crime" matches "crime", "CRIME", etc. The question explicitly requires this.
4
if (result.isEmpty()) { ... }
After the loop, if no books matched, print the error message. We still return the empty list.
Common Mistake

Using .equals() instead of .equalsIgnoreCase(). The question says "case-insensitive." If the input genre is "crime" but the book has "Crime", .equals() returns false. You lose test cases silently.

Method 2

calculateDiscountedPrice

Same filter pattern as Method 1, but with an extra step: apply the discount formula to matching books before returning them.

Discount Formula

Discounted Price = Price - (Price * discount / 100.0)

Example: Book costs 30.0, discount is 20% → 30.0 - (30.0 * 20 / 100.0) = 30.0 - 6.0 = 24.0

Solution.java — Method 2 
public static List<Book> calculateDiscountedPrice(List<Book> books, String genre, int discount) {
    List<Book> result = new ArrayList<>();
    for (Book book : books) {
        if (book.getGenre().equalsIgnoreCase(genre)) {
            double newPrice = book.getPrice() - (book.getPrice() * discount / 100.0);
            book.setPrice(newPrice);
            result.add(book);
        }
    }
    if (result.isEmpty()) {
        System.out.println("Discounted books are unavailable in the given genre.");
    } else {
        System.out.println("Discounted " + genre + " Books:");
    }
    return result;
}
Line-by-line explanation
1
double newPrice = book.getPrice() - (book.getPrice() * discount / 100.0);
Calculate the discounted price. Note 100.0 (not 100) — this forces double division so we don't lose decimal places.
2
book.setPrice(newPrice);
Update the book's price in-place. This modifies the original object — important because the question says "apply the discount to the book."
3
System.out.println("Discounted " + genre + " Books:");
Print this header ONLY if matches were found. The genre variable in the print comes from the method parameter — it prints whatever case the user typed.
Critical: 100.0, not 100

If you write discount / 100 (integer division), then 20 / 100 = 0 in Java. The discount becomes zero. Always use 100.0 to force floating-point division.

Main

Main Method — Reading Input

The trickiest part of this question is reading the input correctly. Author and Book are read together in pairs.

Solution.java — main method 
public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);
    int n = sc.nextInt();
    List<Book> books = new ArrayList<>();

    for (int i = 0; i < n; i++) {
        int authorId = sc.nextInt();
        sc.nextLine(); // consume leftover newline
        String authorName = sc.nextLine();

        int bookId = sc.nextInt();
        sc.nextLine(); // consume leftover newline
        String title = sc.nextLine();
        String genre = sc.nextLine();
        double price = sc.nextDouble();

        Author author = new Author(authorId, authorName);
        Book book = new Book(bookId, title, genre, price, author);
        books.add(book);
    }

    sc.nextLine(); // consume leftover newline
    String searchGenre = sc.nextLine();
    String discountGenre = sc.nextLine();
    int discount = sc.nextInt();

    // Method 1
    List<Book> genreBooks = getBooksBelongingToAGenre(books, searchGenre);
    for (Book b : genreBooks) {
        System.out.println("AuthorName: " + b.getAuthor().getAuthorName()
            + ", Title: " + b.getTitle());
    }

    // Method 2
    List<Book> discountedBooks = calculateDiscountedPrice(books, discountGenre, discount);
    for (Book b : discountedBooks) {
        System.out.println("AuthorName: " + b.getAuthor().getAuthorName()
            + ", Title: " + b.getTitle()
            + ", Updated Price: " + b.getPrice());
    }
}
The sc.nextLine() Trap

After sc.nextInt() or sc.nextDouble(), the newline character \n stays in the buffer. The next sc.nextLine() reads that empty newline instead of the actual text. Fix: always call sc.nextLine() after nextInt()/nextDouble() to consume the leftover newline.

Input Reading Flow

For each of n iterations:
  authorId     ← nextInt()
  consume \n   ← nextLine()
  authorName   ← nextLine()
  bookId       ← nextInt()
  consume \n   ← nextLine()
  title        ← nextLine()
  genre        ← nextLine()
  price        ← nextDouble()

After the loop:
  consume \n   ← nextLine()
  searchGenre  ← nextLine()
  discountGenre ← nextLine()
  discount     ← nextInt()
Every nextInt()/nextDouble() leaves a \n that the next nextLine() must consume

public class Solution {

    public static List<Book> getBooksBelongingToAGenre(List<Book> books, String genre) {
        List<Book> result = new ArrayList<>();
        for (Book book : books) {
            if (book.getGenre().equalsIgnoreCase(genre)) {
                result.add(book);
            }
        }
        if (result.isEmpty()) {
            System.out.println("Genre is not available in the list");
        }
        return result;
    }

    public static List<Book> calculateDiscountedPrice(List<Book> books, String genre, int discount) {
        List<Book> result = new ArrayList<>();
        for (Book book : books) {
            if (book.getGenre().equalsIgnoreCase(genre)) {
                double newPrice = book.getPrice() - (book.getPrice() * discount / 100.0);
                book.setPrice(newPrice);
                result.add(book);
            }
        }
        if (result.isEmpty()) {
            System.out.println("Discounted books are unavailable in the given genre.");
        } else {
            System.out.println("Discounted " + genre + " Books:");
        }
        return result;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        List<Book> books = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            int authorId = sc.nextInt();
            sc.nextLine();
            String authorName = sc.nextLine();

            int bookId = sc.nextInt();
            sc.nextLine();
            String title = sc.nextLine();
            String genre = sc.nextLine();
            double price = sc.nextDouble();

            Author author = new Author(authorId, authorName);
            Book book = new Book(bookId, title, genre, price, author);
            books.add(book);
        }

        sc.nextLine();
        String searchGenre = sc.nextLine();
        String discountGenre = sc.nextLine();
        int discount = sc.nextInt();

        // Method 1: Filter by genre
        List<Book> genreBooks = getBooksBelongingToAGenre(books, searchGenre);
        for (Book b : genreBooks) {
