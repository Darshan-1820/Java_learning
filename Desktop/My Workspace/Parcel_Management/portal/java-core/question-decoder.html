Question Decoder

Read Any PRA Question in 3 Minutes

Stop re-reading the question 5 times. Here's how to extract exactly what you need.

Step 1: Find the Classes

Scan for "Create a class X with the following attributes". Write down:

  • Class name
  • Each field: name + type
  • Is any field a List<OtherClass>? → That's composition
Example from question:
"Create a class Brand with attributes: brandId (Integer), brandName (String), mobileList (List<Mobile>)"
→ You write: Brand: Integer brandId, String brandName, List<Mobile> mobileList

Step 2: Find the Methods

Scan for method names. For each method, answer 3 questions:

QuestionWhere to LookWhat It Tells You
What's the name?Bold text or code formattingYour method name
What does it take?"takes a list/collection of X"Parameter type
What does it give back?KEY WORDS below ↓Return type

The Return Type Decision (most important part):

Question uses these wordsReturn typeYour code does
"returns" a list/collection List<X> return result;
"returns" a map Map<String, Double> return map;
"returns" the average/total/count double or Double return value;
"returns" null if not found Double (wrapper, nullable) return null;
"print" the details or "display" void System.out.println() inside method
"throw" exception Add throws XException throw new XException("msg")

Step 3: Find the Output Format

Scroll to "Sample Output". Copy the EXACT format. Character by character.

Format Trap #1
Samsung: 27150.0
colon + space + decimal (.0 at end)
Format Trap #2
Brand: Samsung, Model Number: 1001, Model Name: Galaxy S21, Price: 20300.0, Storage: 128GB
exact commas, spaces, GB suffix — every character matters
Format Trap #3
"No such Investors found" ≠ "No Investors found"
different messages for different conditions! Read EXACTLY.

Step 4: Find the Input Order

Scroll to "Sample Input Explanation". It tells you EXACTLY what to read and in what order.

Step 5: Find the Exception Pattern

Scan for "throw exception" or "custom exception". If present:

  1. Create the exception class (4 lines — extends Exception, constructor with super(msg))
  2. Throw in the method when condition is met
  3. Catch in main, print "ExceptionName: " + e.getMessage()
  4. EACH method gets its OWN try-catch in main
Template:
class BrandNotFoundException extends Exception {
    BrandNotFoundException(String msg) { super(msg); }
}

Practice: Decode This Question

Create a class Investor with the following attributes:
investorId (Integer), investorName (String), stockList (List<Stock>)

Create a class Stock with the following attributes:
stockId (Integer), stockName (String), purchasePrice (Double), currentPrice (Double)

Create a method getStocksByInvestorName that takes a list of Investor objects and a String (investorName). The method returns a list of Stock objects belonging to the investor with the given name. If no investor is found, throw InvestorNotFoundException with the message "No such Investors found".

Create a method getAveragePurchasePrice that takes a list of Stock objects and returns the average of purchasePrice of all stocks. If the list is empty, return 0.0.

Identify: Classes & fields, method names, return types, exception name, output format clues.

Classes:
• Investor: Integer investorId, String investorName, List<Stock> stockList (composition!)
• Stock: Integer stockId, String stockName, Double purchasePrice, Double currentPrice

Method 1 — getStocksByInvestorName:
• Takes: List<Investor>, String
• Returns: List<Stock> (keyword: "returns a list")
• Exception: InvestorNotFoundException ("No such Investors found")

Method 2 — getAveragePurchasePrice:
• Takes: List<Stock>
• Returns: double (keyword: "returns the average")
• No exception — returns 0.0 if empty

Custom Exception:
• InvestorNotFoundException extends Exception
• Message: "No such Investors found" (exact text!)

In main:
• SEPARATE try-catch for each method call
• Method 1 in one try-catch, method 2 in another
