Section 01

What is Composition?

Composition is when one class HAS another class as a field. It is one of the most important OOP concepts in PRA — about 50% of PRA Major questions use it.

Two Types in PRA

PRA uses two forms of composition. Knowing which one you are dealing with changes how you write the Scanner input and the method logic:

Simple Composition

One object inside another.

Example: A Book HAS one Author

Field: private Author author;

Access: book.getAuthor().getAuthorName()

Mock: Mock 3 Q1 (Author-Book)

List Composition

A list of objects inside another.

Example: A Cricketer HAS many Statistics

Field: private List<Statistics> statistics;

Access: Loop through cricketer.getStatistics()

Mock: Mock 2 Q1 (Cricketer-Statistics)

Analogy — Composition vs Inheritance

A car HAS an engine. That is composition — the engine is a PART of the car. You do not say a car IS an engine (that would be inheritance).

Similarly, a Book HAS an Author. The Author is a part of the Book's data. A Cricketer HAS Statistics (possibly many of them). The Statistics are data that belong to that Cricketer.

Inheritance = IS-A relationship (a Cat IS an Animal)
Composition = HAS-A relationship (a Car HAS an Engine)

How to Spot Composition in a PRA Question

Look for these signals:

  • The question describes two separate classes with their own fields
  • One class has a field whose type is the other class
  • The question says something like "Book contains Author information" or "Each Cricketer has a list of Statistics"
  • The input format shows nested data — some fields belong to the inner class
PRA Reading Tip

When you see two classes in the question, immediately ask: "Does one class contain the other?" If yes, that is composition. Then ask: "Is it ONE object or a LIST of objects?" That tells you simple vs list composition.

Section 02

Simple Composition — Author Inside Book

This is the easier form. One object contains exactly one instance of another object. From Mock 3 Q1.

The Classes

Author.java 
class Author {
    private int authorId;
    private String authorName;

    public Author(int authorId, String authorName) {
        this.authorId = authorId;
        this.authorName = authorName;
    }

    public int getAuthorId() { return authorId; }
    public void setAuthorId(int authorId) { this.authorId = authorId; }

    public String getAuthorName() { return authorName; }
    public void setAuthorName(String authorName) { this.authorName = authorName; }
}
Book.java 
class Book {
    private int bookId;
    private String title;
    private String genre;
    private double price;
    private Author author;  // <-- COMPOSITION: Book HAS an Author

    public Book(int bookId, String title, String genre, double price, Author author) {
        this.bookId = bookId;
        this.title = title;
        this.genre = genre;
        this.price = price;
        this.author = author;
    }

    public int getBookId() { return bookId; }
    public void setBookId(int bookId) { this.bookId = bookId; }

    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }

    public String getGenre() { return genre; }
    public void setGenre(String genre) { this.genre = genre; }

    public double getPrice() { return price; }
    public void setPrice(double price) { this.price = price; }

    public Author getAuthor() { return author; }
    public void setAuthor(Author author) { this.author = author; }
}

List Composition — Statistics Inside Cricketer

This is the harder form. One object contains a LIST of other objects. More complex to read from Scanner and more complex to process. From Mock 2 Q1.

The Classes

Statistics.java 
class Statistics {
    private int Rank;
    private String LeagueName;
    private double PerformanceAverage;
    private String CountryName;

    public Statistics(int Rank, String LeagueName, double PerformanceAverage,
                      String CountryName) {
        this.Rank = Rank;
        this.LeagueName = LeagueName;
        this.PerformanceAverage = PerformanceAverage;
        this.CountryName = CountryName;

    public int getRank() { return Rank; }

    public void setLeagueName(String LeagueName) { this.LeagueName = LeagueName; }

Section 04

Reading Composed Objects from Scanner

This is the tricky part of composition. The input format has nested data, and you must read it in the correct order. One wrong nextLine() and everything shifts.

Simple Composition — Author-Book Input

Input format: For each book, the Author fields come first, then the Book fields.

ReadAuthorBook.java 
Scanner sc = new Scanner(System.in);
int n = sc.nextInt(); // number of books
ArrayList<Book> books = new ArrayList<>();

for (int i = 0; i < n; i++) {
    // ===== Read Author fields FIRST =====
    int authorId = sc.nextInt();
    sc.nextLine(); // consume newline after nextInt
    String authorName = sc.nextLine();

    // ===== Read Book fields =====
    int bookId = sc.nextInt();
    sc.nextLine(); // consume newline after nextInt
    String title = sc.nextLine();
    String genre = sc.nextLine();
    double price = sc.nextDouble();

    // ===== Create Author object, then Book with Author inside =====
    Author author = new Author(authorId, authorName);
    Book book = new Book(bookId, title, genre, price, author);
    books.add(book);
}
Step-by-step for Author-Book input
1
Read Author fields first
The question's input format tells you which fields come first. In this case, authorId and authorName are read BEFORE bookId.
2
Create Author author = new Author(authorId, authorName);
Create the INNER object first. You need this object to pass into the Book constructor.
3
Book book = new Book(bookId, title, genre, price, author);
Create the OUTER object, passing the Author as the last parameter. The Book now "contains" the Author.
Warning — Read the Input Format Carefully

Different PRA questions may order the fields differently. Some put Author fields first, some put Book fields first. ALWAYS follow the input format in the question, not what "makes sense" to you.

List Composition — Cricketer-Statistics Input (Nested Loop)

This is more complex because each Cricketer has a VARIABLE number of Statistics. You need a nested loop — an outer loop for Cricketers and an inner loop for Statistics.

ReadCricketerStats.java 
Scanner sc = new Scanner(System.in);
int n = sc.nextInt(); // number of cricketers
ArrayList<Cricketer> cricketers = new ArrayList<>();

for (int i = 0; i < n; i++) {
    // ===== Read Cricketer's own fields =====
    int id = sc.nextInt();
    sc.nextLine(); // consume newline
    String name = sc.nextLine();
    int age = sc.nextInt();
    sc.nextLine(); // consume newline
    String gender = sc.nextLine();

    // ===== Read how many Statistics this cricketer has =====
    int numStats = sc.nextInt();

    // ===== INNER LOOP: read each Statistics object =====
    List<Statistics> statsList = new ArrayList<>();
    for (int j = 0; j < numStats; j++) {
        int rank = sc.nextInt();
        sc.nextLine(); // consume newline
        String league = sc.nextLine();
        double avg = sc.nextDouble();
        sc.nextLine(); // consume newline
        String country = sc.nextLine();

        statsList.add(new Statistics(rank, league, avg, country));
    }

    // ===== Create Cricketer with the list of Statistics =====
    Cricketer c = new Cricketer(id, name, age, gender, statsList);
    cricketers.add(c);
}
The nested loop pattern explained
1
Outer loop: for (int i = 0; i < n; i++)
Runs once for each Cricketer. Reads the cricketer's own fields (id, name, age, gender).
2
int numStats = sc.nextInt();
BEFORE the inner loop, read HOW MANY statistics this cricketer has. This number drives the inner loop.
3
Inner loop: for (int j = 0; j < numStats; j++)
Runs once for each Statistics object. Creates Statistics and adds to a temporary list.
4
new Cricketer(id, name, age, gender, statsList)
After the inner loop finishes, create the Cricketer with the completed statistics list. The list is now "inside" the Cricketer.
Warning — The sc.nextLine() After sc.nextDouble()

In the inner loop, after reading avg with sc.nextDouble(), you MUST add sc.nextLine() before reading country with sc.nextLine(). This is the same newline trap from Page 01, but it happens more often with nested loops because there are more transitions between nextInt()/nextDouble() and nextLine().

Section 05

Accessing Nested Data

Once objects are composed, you need to "drill down" through getters to access the inner object's data. This is where many students get confused.

Simple Composition — Chained Getters

ChainedGetters.java 
// To get the author's name from a Book object:
String authorName = book.getAuthor().getAuthorName();
//                  │               │
//                  │               └── Step 2: on the Author, get the name
//                  └── Step 1: get the Author object from the Book

// You can also break it into two lines (same result):
Author author = book.getAuthor();        // Step 1: get the Author
String name = author.getAuthorName();     // Step 2: get the name

// In a loop — get author name for every book:
for (Book book : books) {
    System.out.println(book.getTitle() + " by " + book.getAuthor().getAuthorName());
}

List Composition — Loop Through Inner List

InnerListLoop.java 
// To find if a cricketer played in a specific league:
// You MUST loop through their statistics list

for (Cricketer cricketer : cricketers) {
    // Get the list of statistics for THIS cricketer
    for (Statistics stat : cricketer.getStatistics()) {
        // Check each statistics entry
        if (stat.getLeagueName().equalsIgnoreCase(searchLeague)) {
            System.out.println(cricketer.getCricketerName());
            break; // found a match, no need to check more stats
        }
    }
}

// To calculate total performance average for a country:
double total = 0;
boolean found = false;
for (Cricketer cricketer : cricketers) {
    for (Statistics stat : cricketer.getStatistics()) {
        if (stat.getCountryName().equalsIgnoreCase(searchCountry)) {
            total += stat.getPerformanceAverage();
            found = true;
        }
    }
}
return found ? total : null;
The Double Loop Pattern

With list composition, you always end up with a double loop:

  • Outer loop: iterates through the main collection (cricketers)
  • Inner loop: iterates through the nested list (statistics for each cricketer)

This is the standard pattern. Every PRA method that involves list composition uses this double loop.

Cheat Sheet — Accessing Data

What You Want How to Get It Type
Book's own title book.getTitle() Simple — one getter
Book's author name book.getAuthor().getAuthorName() Chained — two getters
Cricketer's own name cricketer.getCricketerName() Simple — one getter
A specific stat's league stat.getLeagueName() (inside inner loop) Simple — but needs inner loop first
All leagues a cricketer played in Loop: for (Statistics s : cricketer.getStatistics()) Inner loop required

PRA Exam Tip

When you see two classes in the question with one being a field of the other, that is composition. Read the question carefully to determine if it is a single object (Author author) or a list of objects (List<Statistics> statistics). This decision affects:

  • Scanner reading: Single object = read fields sequentially. List = nested loop with count.
  • Method logic: Single object = chained getters. List = inner loop through getList().
