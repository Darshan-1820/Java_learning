Section 01

What Is a Filter?

A filter sits between the client and your servlet. Every request/response passes through it — like a security guard at a building entrance.

Security Guard Analogy

Imagine a building with a security guard at the door. Before anyone enters (request), the guard checks their ID. Before anyone leaves (response), the guard logs the exit time. The guard doesn't do the actual work inside — that's the servlet's job. The guard just intercepts and decides: let them pass, or block them.

Filters work exactly the same way. They intercept HTTP requests before they reach the servlet, and responses after the servlet is done.

Browser HTTP Request
→
Filter 1 Auth Check
→
Filter 2 Logging
→
Servlet Business Logic
Filters form a chain. Request flows left-to-right, response flows right-to-left.

Common Uses

  • Authentication — Is the user logged in? If not, redirect to login page.
  • Logging — Log every request's URL, method, and time taken.
  • Encoding — Set request.setCharacterEncoding("UTF-8") globally.
  • CORS — Add cross-origin headers to every response.
  • Compression — Compress response before sending.
Section 02

The Filter Interface

Every filter implements javax.servlet.Filter with three lifecycle methods.

MethodWhen CalledPurpose
init(FilterConfig)Once, at server startupRead config parameters, set up resources
doFilter(request, response, chain)On every matching requestYour logic goes here. Call chain.doFilter() to pass to next.
destroy()Once, at server shutdownRelease resources, cleanup

Minimal Filter

MyFilter.java 
import javax.servlet.*;
import java.io.IOException;

public class MyFilter implements Filter {

    @Override
    public void init(FilterConfig config) throws ServletException {
        // runs once at startup
        System.out.println("Filter initialized");
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                          FilterChain chain) throws IOException, ServletException {
        // BEFORE servlet
        System.out.println("Request intercepted");

        chain.doFilter(request, response);  // pass to next filter or servlet

        // AFTER servlet (response coming back)
        System.out.println("Response leaving");
    }

    @Override
    public void destroy() {
        // runs once at shutdown
        System.out.println("Filter destroyed");
    }
}
Critical Rule

If you forget to call chain.doFilter(request, response), the request stops dead. The servlet never executes. The browser gets a blank page or timeout. This is actually useful for auth filters — block unauthorized requests on purpose.

Section 03

Authentication Filter

Check if the user has a valid session. If not, redirect to login. This protects every page behind login.

AuthFilter.java 
import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.*;
import java.io.IOException;

@WebFilter(urlPatterns = {"/dashboard/*", "/bookings/*", "/admin/*"})
public class AuthFilter implements Filter {

    @Override
    public void doFilter(ServletRequest req, ServletResponse res,
                          FilterChain chain) throws IOException, ServletException {

        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;

        // Check for existing session (false = don't create new one)
        HttpSession session = request.getSession(false);

        if (session != null && session.getAttribute("user") != null) {
            // User is logged in, let them through
            chain.doFilter(request, response);
        } else {
            // Not logged in, redirect to login page
            response.sendRedirect(request.getContextPath() + "/login");
        }
    }
}
In Nimbus

The AuthFilter protects /dashboard/*, /bookings/*, and /admin/*. When a user logs in via LoginServlet, it sets session.setAttribute("user", userObject). The filter checks for that attribute on every protected request.

Section 04

Logging Filter

Log every request's HTTP method, URL, and time taken. Essential for debugging.

LoggingFilter.java 
import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.HttpServletRequest;
import java.io.IOException;

@WebFilter("/*")  // intercepts ALL requests
public class LoggingFilter implements Filter {

    @Override
    public void doFilter(ServletRequest req, ServletResponse res,
                          FilterChain chain) throws IOException, ServletException {

        HttpServletRequest request = (HttpServletRequest) req;

        String method = request.getMethod();        // GET, POST, etc.
        String url = request.getRequestURI();       // /nimbus/bookings
        long startTime = System.currentTimeMillis();

        System.out.println("[REQUEST] " + method + " " + url);

        chain.doFilter(req, res);  // let the request proceed

        long duration = System.currentTimeMillis() - startTime;
        System.out.println("[RESPONSE] " + method + " " + url + " | " + duration + "ms");
    }
}
Section 05

@WebFilter Annotation

Configure filters with annotations instead of web.xml.

@WebFilter Examples 
// Match all requests
@WebFilter("/*")

// Match specific paths
@WebFilter(urlPatterns = {"/dashboard/*", "/admin/*"})

// Match specific servlet by name
@WebFilter(servletNames = {"BookingServlet"})

// With init parameters
@WebFilter(
    urlPatterns = "/*",
    initParams = {
        @WebInitParam(name = "encoding", value = "UTF-8")
    }
)

// web.xml equivalent (if you need ordering):
// <filter>
//   <filter-name>AuthFilter</filter-name>
//   <filter-class>com.nimbus.filters.AuthFilter</filter-class>
// </filter>
// <filter-mapping>
//   <filter-name>AuthFilter</filter-name>
//   <url-pattern>/dashboard/*</url-pattern>
// </filter-mapping>
Section 06

Filter Chain Ordering

When multiple filters match the same URL, the order matters.

How Order is Determined

  • web.xml filters: Execute in the order they appear in the file. First declared = first executed.
  • @WebFilter filters: Order is not guaranteed by the spec. It depends on the server's classloading order.
  • Best practice: If order matters (it usually does), define filters in web.xml and remove @WebFilter.
Request
→
LoggingFilter #1 in web.xml
→
AuthFilter #2 in web.xml
→
Servlet
↓ response flows back ↓
Servlet
→
AuthFilter
→
LoggingFilter
→
Browser
Request: Filter1 → Filter2 → Servlet. Response: Servlet → Filter2 → Filter1.
Why Logging First?

Put LoggingFilter before AuthFilter so you log ALL requests — including the ones that get blocked by auth. If auth was first, you'd never see failed login attempts in your logs.

Section 07

Listeners

Listeners react to lifecycle events: app startup/shutdown, sessions created/destroyed.

ServletContextListener

Runs code when the app starts and stops. Use it for one-time setup like loading config or initializing a connection pool.

AppStartupListener.java 
import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;
import javax.servlet.annotation.WebListener;

@WebListener
public class AppStartupListener implements ServletContextListener {

    @Override
    public void contextInitialized(ServletContextEvent event) {
        System.out.println("Nimbus app started!");
        // Load DB driver, create connection pool, read config
        event.getServletContext().setAttribute("appName", "Nimbus Parcels");
    }

    @Override
    public void contextDestroyed(ServletContextEvent event) {
        System.out.println("Nimbus app shutting down!");
        // Close connection pool, flush logs
    }
}

HttpSessionListener

Track active sessions. Useful for showing "X users online" or monitoring for session leaks.

SessionTracker.java 
import javax.servlet.annotation.WebListener;
import javax.servlet.http.HttpSessionEvent;
import javax.servlet.http.HttpSessionListener;
import java.util.concurrent.atomic.AtomicInteger;

@WebListener
public class SessionTracker implements HttpSessionListener {

    private static final AtomicInteger activeSessions = new AtomicInteger(0);

    @Override
    public void sessionCreated(HttpSessionEvent event) {
        int count = activeSessions.incrementAndGet();
        System.out.println("Session created. Active: " + count);
    }

    @Override
    public void sessionDestroyed(HttpSessionEvent event) {
        int count = activeSessions.decrementAndGet();
        System.out.println("Session destroyed. Active: " + count);
    }

    public static int getActiveCount() {
        return activeSessions.get();
    }
}

Section 08

Full Nimbus Filter Code

Complete AuthFilter and LoggingFilter for the Nimbus Parcel Management System.

In Nimbus

Two filters protect the system: LoggingFilter (logs all requests) runs first, then AuthFilter (blocks unauthenticated access to protected pages). The AppStartupListener initializes the DB connection pool when Tomcat starts.

com/nimbus/filters/AuthFilter.java 
package com.nimbus.filters;

import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.*;
import java.io.IOException;
import java.util.Arrays;
import java.util.List;

@WebFilter(urlPatterns = {"/dashboard/*", "/bookings/*", "/admin/*", "/track/*"})
public class AuthFilter implements Filter {

    // Pages that don't need login
    private static final List<String> PUBLIC_PATHS = Arrays.asList(
        "/login", "/register", "/css/", "/js/", "/images/"
    );

    @Override
    public void doFilter(ServletRequest req, ServletResponse res,
                          FilterChain chain) throws IOException, ServletException {

        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        String path = request.getRequestURI().substring(request.getContextPath().length());

        // Allow public paths
        for (String publicPath : PUBLIC_PATHS) {
            if (path.startsWith(publicPath)) {
                chain.doFilter(req, res);
                return;
            }
        }

        // Check session
        HttpSession session = request.getSession(false);
        if (session != null && session.getAttribute("user") != null) {
            chain.doFilter(req, res);
        } else {
            // Save the page they wanted so we can redirect after login
            session = request.getSession(true);
            session.setAttribute("redirectAfterLogin", request.getRequestURI());
            response.sendRedirect(request.getContextPath() + "/login");
        }
    }
}
com/nimbus/filters/LoggingFilter.java 
package com.nimbus.filters;

import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.HttpServletRequest;
import java.io.IOException;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

@WebFilter("/*")
public class LoggingFilter implements Filter {

    private static final DateTimeFormatter FMT =
        DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");

    @Override
    public void doFilter(ServletRequest req, ServletResponse res,
                          FilterChain chain) throws IOException, ServletException {

        HttpServletRequest request = (HttpServletRequest) req;

        String method = request.getMethod();
        String url = request.getRequestURI();
        String clientIP = request.getRemoteAddr();
        String timestamp = LocalDateTime.now().format(FMT);

        long start = System.currentTimeMillis();

        // Let request proceed
        chain.doFilter(req, res);

        long duration = System.currentTimeMillis() - start;

        System.out.printf("[%s] %s %s | IP: %s | %dms%n",
            timestamp, method, url, clientIP, duration);
    }
}
Section 09

Exercises

Build these to solidify filters and listeners.

Exercise 1 — Encoding Filter

Create a filter that sets request.setCharacterEncoding("UTF-8") and response.setContentType("text/html; charset=UTF-8") on every request. Map it to /*. This prevents garbled text from form submissions.

Exercise 2 — Admin-Only Filter

Create a filter on /admin/* that checks not just if the user is logged in, but if their role is "ADMIN". If not, redirect to /dashboard with an error message in the session: "Access denied. Admin only."

Exercise 3 — Request Counter Listener

Create a ServletRequestListener that counts total requests since app startup. Store the count in ServletContext as an attribute. Display it on a JSP page: "Total requests served: 1,247".

Exercise 4 — Session Timeout Warning

Create an HttpSessionListener that logs a warning when sessions are being created faster than destroyed (possible memory leak). If activeSessions > 100, print a warning to the console.
