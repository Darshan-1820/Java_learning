Section 01

Question Statement

Mock Test 1, Question 2. This one tests custom exceptions combined with odd/even arithmetic logic.

PRA Major 30-40 min

OddEven Exception

Create a OddEvenException class.

Create Solution class with main method. In the main read a number of integers to be stored into collection.

The main method will add up odd integers and multiplies the even numbers. And if in case sum of odd integers is greater than multiplied even integers, raise exception with message as "Sum is greater", else print multiplied value.

Section 02

Analysis

This question looks short but has several moving parts. Let us break down what is being tested.

What Concepts Are Being Tested?
  • Custom Exception: Create a class that extends Exception with a constructor that takes a message
  • Collections: Storing integers in an ArrayList
  • Odd/Even logic: Using the modulo operator % to detect odd vs even numbers
  • Exception handling: throw + try-catch block
  • Arithmetic: Sum for odd numbers, product for even numbers
Section 03

Step-by-Step Approach

Four simple steps to solve this.

  1. Create OddEvenException class that extends Exception. It needs one constructor that takes a String message and passes it to the parent using super(message).
  2. Read n integers from Scanner and store them in an ArrayList.
  3. Separate the math: loop through the list. For odd numbers, add to sumOdd. For even numbers, multiply into productEven.
  4. Compare and decide: if sumOdd > productEven, throw the exception. Otherwise, print the product.
Section 04

The Exception Class

Custom exceptions in Java are just classes that extend Exception. That is it — 4 lines of code.

OddEvenException.java 
class OddEvenException extends Exception {
    public OddEvenException(String message) {
        super(message);
    }
}
Line-by-line explanation
1
class OddEvenException extends Exception
extends Exception makes this a checked exception. Java now treats it like any built-in exception (can be thrown and caught).
2-4
public OddEvenException(String message) { super(message); }
super(message) passes the error message to the parent Exception class. When you later call e.getMessage(), it returns this message.
PRA Exam Tip

Every custom exception in PRA follows this exact same pattern: extends Exception, one constructor, call super(message). Memorize these 4 lines — you will use them in almost every PRA question that involves exceptions.

Section 05

The Logic — Odd vs Even

Understanding how to detect odd/even and the critical initialization trap.

Odd/Even Detection

The modulo operator % gives the remainder after division:

  • num % 2 != 0 → the number is odd (dividing by 2 leaves a remainder)
  • num % 2 == 0 → the number is even (dividing by 2 leaves no remainder)
Critical: Initialize Product to 1, Not 0

When multiplying numbers, you must start with productEven = 1. If you start with productEven = 0, any number multiplied by 0 is still 0, and your product will always be 0 no matter what the inputs are.

For sum, starting at 0 is fine because 0 + anything = anything. But for product, 0 * anything = 0.

Odd/Even Logic 
int sumOdd = 0;       // sum starts at 0 (adding)
int productEven = 1;  // product starts at 1 (multiplying)

for (int num : numbers) {
    if (num % 2 != 0) {
        sumOdd += num;        // odd: add to sum
    } else {
        productEven *= num;   // even: multiply into product
    }
}

Worked Example

Given numbers: [5, 4, 2, 13, 8]

Number Odd or Even? sumOdd productEven
5 5 % 2 = 1 → Odd 0 + 5 = 5 1
4 4 % 2 = 0 → Even 5 1 * 4 = 4
2 2 % 2 = 0 → Even 5 4 * 2 = 8
13 13 % 2 = 1 → Odd 5 + 13 = 18 8
8 8 % 2 = 0 → Even 18 8 * 8 = 64

Result: sumOdd = 18, productEven = 64. Since 18 < 64, we print 64.

Section 06

Complete Solution

Copy-paste ready. Paste into HackerRank and it compiles.

Copy-Paste Ready

Full program with import, exception class, and Solution class with main method.

Solution.java (Complete) 
import java.util.*;

class OddEvenException extends Exception {
    public OddEvenException(String message) {
        super(message);
    }
}

public class Solution {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        ArrayList<Integer> numbers = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            numbers.add(sc.nextInt());
        }

        int sumOdd = 0;
        int productEven = 1;

        for (int num : numbers) {
            if (num % 2 != 0) {
                sumOdd += num;
            } else {
                productEven *= num;
            }
        }

        try {
            if (sumOdd > productEven) {
                throw new OddEvenException("Sum is greater");
            } else {
                System.out.println(productEven);
            }
        } catch (OddEvenException e) {
            System.out.println("OddEvenException: " + e.getMessage());
        }
    }
}
Line-by-line explanation
1
import java.util.*;
Imports ArrayList, Scanner, and everything else from java.util.
3-7
class OddEvenException extends Exception { ... }
Custom exception class. Extends Exception, takes a message string, passes it to parent via super().
11-13
Scanner + read n + create ArrayList
Standard input setup. Read how many numbers to expect, create an empty list to store them.
15-17
for loop: numbers.add(sc.nextInt())
Read n integers and add each to the ArrayList. No nextLine() trap here because we are reading int after int (no String reads in between).
19-20
int sumOdd = 0; int productEven = 1;
Initialize accumulators. Sum starts at 0 (neutral for addition). Product starts at 1 (neutral for multiplication).
22-28
for loop: odd/even check and accumulate
For each number: if odd (% 2 != 0), add to sumOdd. If even (% 2 == 0), multiply into productEven.
30-37
try { if (sumOdd > productEven) throw ... } catch ...
The comparison happens inside a try block. If sum is greater, we throw our custom exception. The catch block immediately catches it and prints the formatted message. If sum is not greater, we just print the product.

Section 08

Common Mistakes

Watch out for these traps specific to this question.

Mistake 1: Initializing productEven to 0

If you write int productEven = 0;, every multiplication will give 0. The product will always be 0, and the sum will almost always be greater, causing false exceptions.

Fix: Initialize to 1. The identity element for multiplication is 1, not 0.

Mistake 2: Wrong Output Format for Exception

The expected output is OddEvenException: Sum is greater (with the class name, colon, space, then message). If you just print e.getMessage(), you get Sum is greater without the class name prefix.

Fix: Use "OddEvenException: " + e.getMessage() in the catch block.

Mistake 3: Forgetting try-catch

Since OddEvenException extends Exception (a checked exception), Java requires you to either catch it or declare it with throws. If you throw it outside a try block without a catch, the code will not compile.

Fix: Always wrap the throw statement in a try-catch block.

Mistake 4: Using == 1 for Odd Check

Some students write num % 2 == 1 to check for odd. This fails for negative numbers because -5 % 2 gives -1, not 1. Using != 0 handles both positive and negative numbers.

Fix: Use num % 2 != 0 instead of num % 2 == 1.
