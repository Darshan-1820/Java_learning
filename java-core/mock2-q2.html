Sum of Digits

Minor 10–15 min

Question Statement

Write a Java program to find the sum of digits of each element in an array of integers. The solution class should be implemented with the main method.

  • The program should read the size of the integer array.
  • It should then calculate the sum of digits of each element in the array.
  • Assume there are valid values in the array.
Analysis

What This Question Tests

This is a MINOR question — simple, no OOP needed. Just arrays and math.

Key Concepts
  • Arrays (not Collections) — plain int[]
  • Loop + digit extraction — peel off digits one by one
  • Math: num % 10 gets the last digit, num / 10 removes the last digit

No classes, no getters/setters, no ArrayList. Just read numbers, crunch digits, print results. This is the kind of question you should finish in under 15 minutes and move on to the major question.

Core Logic

How Digit Sum Works

The trick is simple: keep dividing by 10 and collecting remainders. Let's walk through the number 101.

Step 1:  101 % 10 = 1    (last digit)
         101 / 10 = 10   (remove last digit)

Step 2:   10 % 10 = 0    (last digit)
          10 / 10 = 1    (remove last digit)

Step 3:    1 % 10 = 1    (last digit)
           1 / 10 = 0    (STOP — number is 0)

Sum = 1 + 0 + 1 = 2
% 10 extracts the last digit, / 10 chops it off. Repeat until nothing left.
Think of it like this

Imagine you have a stack of coins labeled 1, 0, 1 (from bottom to top). % 10 picks up the top coin. / 10 removes it from the stack. Keep picking until the stack is empty.

The Two Operators

Operator What It Does Example (num = 253)
num % 10 Gets the last digit (remainder when divided by 10) 253 % 10 = 3
num / 10 Removes the last digit (integer division drops the remainder) 253 / 10 = 25
Method

The digitSum Method

If you want to write it as a separate method (cleaner code, but not required for this question):

digitSum helper method 
public static int digitSum(int num) {
    num = Math.abs(num); // handle negative numbers
    int sum = 0;
    while (num > 0) {
        sum += num % 10;  // add last digit to sum
        num /= 10;        // remove last digit
    }
    return sum;
}
Line-by-line explanation
1
num = Math.abs(num);
If the number is negative (like -42), make it positive first. We only care about the digits, not the sign.
2
int sum = 0;
Our running total. Start at zero, then keep adding digits to it.
3
while (num > 0)
Keep looping as long as there are digits left. When num becomes 0, we've processed every digit.
4
sum += num % 10;
Get the last digit using modulo 10, add it to our running sum.
5
num /= 10;
Chop off the last digit. Integer division by 10 drops the remainder. 253 becomes 25.
Full Code

Complete Solution

Copy-paste ready. This handles all cases including zero and negative numbers.

Solution.java — Complete 
import java.util.Scanner;

public class Solution {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        int[] arr = new int[n];

        for (int i = 0; i < n; i++) {
            arr[i] = sc.nextInt();
        }

        for (int i = 0; i < n; i++) {
            int num = Math.abs(arr[i]);
            int sum = 0;
            if (num == 0) {
                System.out.println(0);
                continue;
            }
            while (num > 0) {
                sum += num % 10;
                num /= 10;
            }
            System.out.println(sum);
        }
    }
}
Line-by-line explanation
1
int n = sc.nextInt();
Read the array size from input. How many numbers will follow.
2
int[] arr = new int[n];
Create an integer array of size n. Plain array, not ArrayList — no need for Collections here.
3
for (int i = 0; i < n; i++) arr[i] = sc.nextInt();
Read all n numbers into the array, one by one.
4
if (num == 0) { System.out.println(0); continue; }
Special case: if the number is 0, the while loop won't execute (0 is not > 0). So we handle it explicitly and skip to the next number.
5
while (num > 0) { sum += num % 10; num /= 10; }
The core digit-sum loop. Extract last digit, add to sum, remove last digit. Repeat until all digits are processed.

Watch Out

Edge Cases

The Zero Trap

If num is 0, the while (num > 0) loop never executes. The sum stays at 0, which is correct — but you need to print it! Without the if (num == 0) check, the program would print nothing for that element.

Edge Case What Happens How We Handle It
0 While loop skips (0 is not > 0) if (num == 0) check prints 0 and continues
000 Java reads this as just 0 Same as above — no special handling needed
Negative numbers -42 % 10 = -2 in Java (negative remainder) Math.abs(num) makes it positive first
PRA Exam Tip

The question says "assume valid values" so negatives probably won't appear. But adding Math.abs() costs you 5 seconds and protects against hidden test cases. Always add it.
