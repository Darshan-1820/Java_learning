Mock 5 — Question 2

Car Management System

Major 60–75 min

Question Statement

Design and implement a Car Management System using inheritance and polymorphism.

Base Class — Car: brand, model, price, type (Petrol, Diesel, or Hybrid)

Subclass — PetrolOrDieselCar extends Car: tankCapacity, mileage

Subclass — SmartHybridCar extends Car: batteryCapacity, tankCapacity, mileage

Methods in CarService:

  1. getAveragesByType(List<Car>, String type) — Filter cars by type, then print averages:
    • Hybrid: avgTankCapacity + avgBatteryCapacity
    • Petrol/Diesel: avgTankCapacity only
    • Not found: "No cars available for the given type or the required criteria."
  2. getMinMileageByBrand(List<Car>) — Find the car with minimum mileage per brand (across all types), print results.

Input: Read numCars, then for each car: brand, model, price, type. If Petrol/Diesel: tankCapacity, mileage. If Hybrid: batteryCapacity, tankCapacity, mileage. Then read the search type.

Analysis

What This Question Tests

This is a full inheritance + polymorphism problem. It combines class hierarchies, type-checking at runtime, downcasting, and aggregation logic — all the heavy OOP concepts in one question.

Core Concepts Being Tested
  • Inheritance: Base class Car with two subclasses that add their own fields
  • instanceof: Runtime type checking to determine if a Car is actually a SmartHybridCar or PetrolOrDieselCar
  • Downcasting: Casting a Car reference to its subclass to access subclass-specific methods like getMileage()
  • LinkedHashMap: Maintaining insertion order while grouping cars by brand
  • Conditional input parsing: Reading different fields depending on the car type
Walkthrough

Tracing Through Sample Data

Let's trace through Test Case 1: 3 cars, search type = "Hybrid"

Input cars:
1. Toyota | HyRyder | 1990000 | Hybrid  | battery=4.3 | tank=40 | mileage=28.0
2. Toyota | Glanza  | 650000  | Petrol  | tank=44     | mileage=20.0
3. Honda  | Amaze   | 800000  | Hybrid  | battery=6.8 | tank=47 | mileage=23.0

Step 1: getAveragesByType(cars, "Hybrid")
  Filter by type "Hybrid" → [HyRyder, Amaze]
  avgTankCapacity  = (40 + 47) / 2 = 43.5
  avgBatteryCapacity = (4.3 + 6.8) / 2 = 5.55

Step 2: getMinMileageByBrand(cars)
  Brand "Toyota": HyRyder=28.0, Glanza=20.0 → min = 20.0
  Brand "Honda":  Amaze=23.0 → min = 23.0
Both methods work across the entire list, but getAveragesByType only looks at the filtered subset
Key Insight: getMinMileageByBrand uses ALL cars

The min mileage method does NOT filter by type. It looks at every car regardless of whether it's Petrol, Diesel, or Hybrid. You need instanceof checks to extract mileage from the correct subclass type.

Algorithm

The Algorithm Step by Step

  • Step 1: Read n cars. For each, read brand, model, price, type. Based on type, read extra fields and create the correct subclass object.
  • Step 2: Read the search type string.
  • Step 3 (getAveragesByType): Filter the list by matching type (case-insensitive). If empty, print error message. If Hybrid, compute avgTankCapacity and avgBatteryCapacity. Otherwise, compute avgTankCapacity only.
  • Step 4 (getMinMileageByBrand): Iterate all cars. Use instanceof to get mileage from each car. Use a LinkedHashMap<String, Double> to track minimum mileage per brand.
  • Step 5: Print the map entries — LinkedHashMap preserves insertion order.
Why LinkedHashMap, not HashMap?

HashMap does not guarantee order. The output needs brands in the order they first appeared in input. LinkedHashMap preserves insertion order, so the first brand added stays first in the output. This is a common PRA requirement — if output order matters, always use LinkedHashMap.

Full Code

Complete Solution

Three classes + one main class. The key is the inheritance hierarchy and knowing when to cast.

Car.java — Base Class 
class Car {
    private String brand, model, type;
    private double price;

    public Car(String brand, String model, double price, String type) {
        this.brand = brand;
        this.model = model;
        this.price = price;
        this.type = type;
    }

    public String getBrand() { return brand; }
    public String getModel() { return model; }
    public double getPrice() { return price; }
    public String getType() { return type; }
}
PetrolOrDieselCar.java — Subclass 
class PetrolOrDieselCar extends Car {
    private double tankCapacity, mileage;

    public PetrolOrDieselCar(String brand, String model, double price, String type,
                             double tankCapacity, double mileage) {
        super(brand, model, price, type);
        this.tankCapacity = tankCapacity;
        this.mileage = mileage;
    }

    public double getTankCapacity() { return tankCapacity; }
    public double getMileage() { return mileage; }
}
SmartHybridCar.java — Subclass 
class SmartHybridCar extends Car {
    private double batteryCapacity, tankCapacity, mileage;

    public SmartHybridCar(String brand, String model, double price, String type,
                          double batteryCapacity, double tankCapacity, double mileage) {
        super(brand, model, price, type);
        this.batteryCapacity = batteryCapacity;
        this.tankCapacity = tankCapacity;
        this.mileage = mileage;
    }

    public double getBatteryCapacity() { return batteryCapacity; }
    public double getTankCapacity() { return tankCapacity; }
    public double getMileage() { return mileage; }
}
Solution.java — Main + Service Methods 
import java.util.*;

public class Solution {

    public static void getAveragesByType(List<Car> cars, String type) {
        List<Car> filtered = new ArrayList<>();
        for (Car c : cars) {
            if (c.getType().equalsIgnoreCase(type)) filtered.add(c);
        }
        if (filtered.isEmpty()) {
            System.out.println("No cars available for the given type or the required criteria.");
            return;
        }
        if (type.equalsIgnoreCase("Hybrid")) {
            double avgTank = 0, avgBattery = 0;
            for (Car c : filtered) {
                SmartHybridCar h = (SmartHybridCar) c;
                avgTank += h.getTankCapacity();
                avgBattery += h.getBatteryCapacity();
            }
            avgTank /= filtered.size();
            avgBattery /= filtered.size();
            System.out.println("Averages for type: Hybrid");
            System.out.println("avgTankCapacity: " + avgTank);
            System.out.println("avgBatteryCapacity: " + avgBattery);
        } else {
            double avgTank = 0;
            for (Car c : filtered) {
                PetrolOrDieselCar p = (PetrolOrDieselCar) c;
                avgTank += p.getTankCapacity();
            }
            avgTank /= filtered.size();
            System.out.println("Averages for type: " + type);
            System.out.println("avgTankCapacity: " + avgTank);
        }
    }

    public static void getMinMileageByBrand(List<Car> cars) {
        Map<String, Double> minMap = new LinkedHashMap<>();
        for (Car c : cars) {
            double mileage = 0;
            if (c instanceof SmartHybridCar)
                mileage = ((SmartHybridCar) c).getMileage();
            else if (c instanceof PetrolOrDieselCar)
                mileage = ((PetrolOrDieselCar) c).getMileage();

            if (!minMap.containsKey(c.getBrand()) || mileage < minMap.get(c.getBrand())) {
                minMap.put(c.getBrand(), mileage);
            }
        }
        System.out.println("Minimum Mileage by Brand");
        for (Map.Entry<String, Double> entry : minMap.entrySet()) {
            System.out.println("Brand: " + entry.getKey() + ", Min Mileage: " + entry.getValue());
        }
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = Integer.parseInt(sc.nextLine().trim());
        List<Car> cars = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            String brand = sc.nextLine().trim();
            String model = sc.nextLine().trim();
            double price = Double.parseDouble(sc.nextLine().trim());
            String type = sc.nextLine().trim();

            if (type.equalsIgnoreCase("Hybrid")) {
                double battery = Double.parseDouble(sc.nextLine().trim());
                double tank = Double.parseDouble(sc.nextLine().trim());
                double mileage = Double.parseDouble(sc.nextLine().trim());
                cars.add(new SmartHybridCar(brand, model, price, type, battery, tank, mileage));
            } else {
                double tank = Double.parseDouble(sc.nextLine().trim());
                double mileage = Double.parseDouble(sc.nextLine().trim());
                cars.add(new PetrolOrDieselCar(brand, model, price, type, tank, mileage));
            }
        }

        String searchType = sc.nextLine().trim();
        getAveragesByType(cars, searchType);
        getMinMileageByBrand(cars);
    }
}
Key sections explained
1
class Car { private String brand, model, type; private double price; }
Base class with 4 fields. All fields are private with getters — standard encapsulation. The type field determines which subclass to create.
2
super(brand, model, price, type);
Both subclasses call super() to pass common fields to the parent constructor. This is mandatory — Java won't compile without it if the parent has no default constructor.
3
SmartHybridCar h = (SmartHybridCar) c;
Downcasting. The variable c is declared as Car, but we know it's actually a SmartHybridCar (we filtered by type). We cast it to access getBatteryCapacity() and getTankCapacity().
4
if (c instanceof SmartHybridCar) mileage = ((SmartHybridCar) c).getMileage();
instanceof checks the actual runtime type. Since both subclasses have getMileage() but the base class does not, we must check type and cast before calling it. Always check instanceof before casting.
5
Map<String, Double> minMap = new LinkedHashMap<>();
LinkedHashMap preserves insertion order. As we iterate cars, the first brand seen stays first in output. If we used HashMap, brand order would be random.
6
if (!minMap.containsKey(c.getBrand()) || mileage < minMap.get(c.getBrand()))
Two conditions: (1) brand not seen yet — add it, or (2) this car's mileage is lower than the current minimum for that brand — update it. This is the standard "find minimum per group" pattern.

Watch Out

Common Mistakes

Wrong: Forgetting instanceof before cast

Directly casting (SmartHybridCar) c without checking type first will throw ClassCastException at runtime if the car is actually a PetrolOrDieselCar.

In getAveragesByType, you're safe because you filtered by type first. But in getMinMileageByBrand, you're iterating ALL cars — you MUST check instanceof.

ClassCastException = instant 0 marks on that test case.

Correct: Always check before casting

Use the instanceof → cast pattern:

if (c instanceof SmartHybridCar)
  mileage = ((SmartHybridCar) c).getMileage();

else if (c instanceof PetrolOrDieselCar)
  mileage = ((PetrolOrDieselCar) c).getMileage();

Safe. No runtime exceptions. Works for any combination of car types.

Mistake What Goes Wrong Fix
Using HashMap instead of LinkedHashMap Brand output order is random — fails test cases that check order Use LinkedHashMap to preserve insertion order
Not reading input conditionally Hybrid has 3 extra fields, Petrol/Diesel has 2. Reading wrong number corrupts all subsequent input Check type before deciding how many lines to read
Using == for String comparison type == "Hybrid" compares references, not values. Always false for Scanner input. Use type.equalsIgnoreCase("Hybrid")
Forgetting return after error message If filtered list is empty and you don't return, the code continues to divide by zero Print error message AND return; immediately
Putting mileage in the base Car class Seems cleaner but violates the question's class design. PRA expects exact class structure. Keep mileage in subclasses only. Use instanceof + cast to access it.
Integer division for averages sum / count where both are int gives truncated result (43 instead of 43.5) Use double for sum and count, or cast: (double) sum / count
Exam Strategy: Build the Classes First

Start by writing all 3 classes (Car, PetrolOrDieselCar, SmartHybridCar) with constructors and getters. Then write main() to read input and create objects. Test that the objects are created correctly. Only then write the service methods. This way, even if you run out of time on the methods, you get marks for correct class structure and input parsing.
