Section 01

Question Statement

This is the actual HackerRank question from Mock Test 1, Question 1. Read it carefully before looking at the solution.

PRA Major 45-55 min

Investors

Create a class named Investor with the following attributes:

  • id - int: to capture investor ID
  • name - String: to capture investor name
  • gains - double: to capture gains accumulated

Define required getters, setters and parameterized constructor.

Create a class named Solution, and define the following two static methods:

1. averageOfPositiveGains
This method takes Collection of Investors and returns the average of positive gains to the main method.

2. negativeGainsInvestorNames
This method takes collection of Investor objects and returns Collection of Investor objects with negative gains to the main method.

The above two static methods should be called from main method.

For averageOfPositiveGains:
The main method will print returned value as it is, if the returned value is greater than zero; else, the main method should print "No such Investors found".

For negativeGainsInvestorNames:
The main method should print the id, name, and gains of the Investor object with negative gains. If no investor found with negative gains then the main method should print "No Investors found".

Note:

  1. Collection should be in insertion order.
  2. Refer sample testcases for Input and Output.

Read Note:

  1. Read number of Investor objects to be created.
  2. Read id, name and gains for each Investor object.
Section 02

Analysis

Before writing any code, let us understand what concepts this question is testing and what approach to take.

What Concepts Are Being Tested?
  • OOP: Class with private fields, parameterized constructor, getters and setters
  • Collections: ArrayList (insertion order required), Collection<T> as method parameter type
  • Static methods: Two methods in Solution class that process a collection of objects
  • Conditional output: Different messages printed depending on whether results are found or not

This is a classic PRA pattern: one POJO class, two static methods (one filter, one aggregate), and a main method that reads input, calls both methods, and prints results with fallback messages.

Section 03

Step-by-Step Approach

Here is the plan of attack before writing a single line of code.

  1. Create the Investor class with 3 private fields (id, name, gains), a parameterized constructor that takes all 3, and getters/setters for each field.
  2. Write averageOfPositiveGains() — loop through all investors, check if gains > 0, sum those gains, count how many, return sum / count. Return 0 if none found.
  3. Write negativeGainsInvestorNames() — loop through all investors, check if gains < 0, add matching investors to a new ArrayList, return the list.
  4. Write main method — read n, loop to create n Investor objects from Scanner input, call both methods, handle the output with the correct messages.

Section 05

Method 1 — averageOfPositiveGains

This method loops through all investors, picks the ones with positive gains, and returns the average of those gains.

averageOfPositiveGains 
public static double averageOfPositiveGains(Collection<Investor> investors) {
    double sum = 0;
    int count = 0;
    for (Investor inv : investors) {
        if (inv.getGains() > 0) {
            sum += inv.getGains();
            count++;
        }
    }
    if (count > 0) {
        return sum / count;
    }
    return 0;
}
Line-by-line explanation
1
public static double averageOfPositiveGains(Collection<Investor> investors)
static = belongs to the class, not an object. Takes a Collection<Investor> (not ArrayList — the question says Collection). Returns double.
2-3
double sum = 0; int count = 0;
Two tracking variables. sum accumulates the total of positive gains. count tracks how many investors had positive gains.
4-9
for (Investor inv : investors) { if (inv.getGains() > 0) ... }
Enhanced for-loop. For each investor, check if gains are positive (> 0). If yes, add gains to sum and increment count.
10-13
if (count > 0) return sum / count; return 0;
Safety check: only divide if we found at least one positive-gains investor. Otherwise return 0 (main method checks for this).
Watch Out

The parameter type is Collection<Investor>, not ArrayList<Investor>. The question specifically says "Collection." Using ArrayList here will still work because ArrayList implements Collection, but match the question exactly.

public static Collection<Investor> negativeGainsInvestorNames(...)
Returns Collection<Investor> — a list of Investor objects (not just names, despite the method name).
2
ArrayList<Investor> result = new ArrayList<>();
Create a new empty ArrayList to hold the filtered results. ArrayList maintains insertion order (as the question requires).
3-6

Section 07

The Main Method

The main method reads input, creates Investor objects, calls both methods, and prints results with the correct messages.

main method 
public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);
    int n = sc.nextInt();
    ArrayList<Investor> investors = new ArrayList<>();

    for (int i = 0; i < n; i++) {
        int id = sc.nextInt();
        sc.nextLine();           // consume leftover newline after nextInt
        String name = sc.nextLine();
        double gains = sc.nextDouble();
        investors.add(new Investor(id, name, gains));
    }

    // Call method 1
    double avg = averageOfPositiveGains(investors);
    if (avg > 0) {
        System.out.println(avg);
    } else {
        System.out.println("No such Investors found");
    }

    // Call method 2
    Collection<Investor> negativeInvestors = negativeGainsInvestorNames(investors);
    if (!negativeInvestors.isEmpty()) {
        for (Investor inv : negativeInvestors) {
            System.out.println(inv.getId());
            System.out.println(inv.getName());
            System.out.println(inv.getGains());
        }
    } else {
        System.out.println("No Investors found");
    }
}
Line-by-line explanation
1-3
Scanner sc = new Scanner(System.in); int n = sc.nextInt();
Create Scanner for reading input. Read the first integer — the number of investors to create.
4
ArrayList<Investor> investors = new ArrayList<>();
Create an empty ArrayList to hold all investor objects. We use ArrayList because the question says "insertion order."
6-12
for loop: read id, name, gains for each investor
Critical: after sc.nextInt() on line 7, we call sc.nextLine() on line 8 to consume the leftover newline. Without this, name would read an empty string.
15-20
Call averageOfPositiveGains, check result
If the returned average is greater than 0, print it. Otherwise print the "not found" message. The message must match EXACTLY.
23-32
Call negativeGainsInvestorNames, check result
If the returned collection is not empty, loop and print each investor's id, name, gains (each on its own line). Otherwise print the "not found" message.
PRA Exam Tip

The sc.nextLine() after sc.nextInt() on line 8 is the #1 bug in PRA. If you forget it, the name will be empty and your entire output will be wrong. Memorize this pattern: every nextInt() or nextDouble() followed by nextLine() needs a buffer-clearing nextLine() in between.

Complete Solution

Solution.java (Complete) 
import java.util.*;

class Investor {
    private int id;
    private String name;
    private double gains;

    public Investor(int id, String name, double gains) {
        this.id = id;
        this.name = name;
        this.gains = gains;
    }

    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public double getGains() { return gains; }
    public void setGains(double gains) { this.gains = gains; }
}

public class Solution {

    public static double averageOfPositiveGains(Collection<Investor> investors) {
        double sum = 0;
        int count = 0;
        for (Investor inv : investors) {
            if (inv.getGains() > 0) {
                sum += inv.getGains();
                count++;
            }
        }
        if (count > 0) {
            return sum / count;
        }
        return 0;
    }

    public static Collection<Investor> negativeGainsInvestorNames(Collection<Investor> investors) {
        ArrayList<Investor> result = new ArrayList<>();
        for (Investor inv : investors) {
            if (inv.getGains() < 0) {
                result.add(inv);
            }
        }
        return result;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        ArrayList<Investor> investors = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            int id = sc.nextInt();
            sc.nextLine();
            String name = sc.nextLine();
            double gains = sc.nextDouble();
            investors.add(new Investor(id, name, gains));
        }

        double avg = averageOfPositiveGains(investors);
        if (avg > 0) {
            System.out.println(avg);
        } else {
            System.out.println("No such Investors found");
        }

        Collection<Investor> negativeInvestors = negativeGainsInvestorNames(investors);
        if (!negativeInvestors.isEmpty()) {
            for (Investor inv : negativeInvestors) {
                System.out.println(inv.getId());
                System.out.println(inv.getName());
                System.out.println(inv.getGains());
            }
        } else {
            System.out.println("No Investors found");
        }
    }
}
Section 09

Test Cases

Walk through these two test cases to verify you understand the input/output flow.

Test Case 1 — All Negative Gains
Input
4 101 Simon -50000 102 Danny -30000 103 Kiran -40000 104 Danny -50000
Output
No such Investors found 101 Simon -50000.0 102 Danny -30000.0 103 Kiran -40000.0 104 Danny -50000.0
Why This Output?

All 4 investors have negative gains. So averageOfPositiveGains finds zero positive-gains investors, returns 0, and main prints "No such Investors found." Then negativeGainsInvestorNames returns all 4, and main prints each one's id, name, and gains (note: gains printed as -50000.0 with the decimal because it is a double).

Test Case 2 — All Positive Gains
Input
3 101 Kiran 40000 102 Samuel 40000 103 Joe 40000
Output
40000.0 No Investors found
Why This Output?

All 3 investors have positive gains of 40000. Average = (40000 + 40000 + 40000) / 3 = 40000.0. No investors have negative gains, so the second method returns an empty list and main prints "No Investors found."

Section 10

Common Mistakes

These are the mistakes that will cost you marks on this specific question. Memorize them.

Mistake 1: Forgetting sc.nextLine() After sc.nextInt()

After reading id with sc.nextInt(), the newline character stays in the buffer. The next sc.nextLine() for reading name will read that empty newline instead of the actual name. Every investor's name will be blank.

Fix: Add sc.nextLine(); immediately after sc.nextInt(); to consume the leftover newline.

Mistake 2: Using > Instead of > 0 for Positive Check

Some students write if (inv.getGains() > 0) correctly but then forget to check avg > 0 in main. If you just check avg > 0 without the method also handling it, edge cases break.

Fix: The method returns 0 when no positive gains exist. Main checks if (avg > 0) — both checks must be present.

Mistake 3: Printing Gains as int Instead of double

The gains field is a double. When you print -50000, Java will output -50000.0 (with the decimal point). If you somehow cast it to int and print -50000, the output will not match.

Fix: Just use System.out.println(inv.getGains()) — Java handles the formatting automatically for doubles.

Mistake 4: Wrong Error Message Text

The two error messages are different:

  • For no positive gains: "No such Investors found"
  • For no negative gains: "No Investors found"

They look similar but are NOT the same. HackerRank matches output exactly — character by character. If you swap them or add/remove a word, the test case fails.
