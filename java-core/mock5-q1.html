Mock 5 — Question 1

Subscription Management System

Major 60–75 min

Question Statement

Develop a Subscription Management System to track subscription details for multiple customers. Implement two methods:

  1. calculateTotalAmount — calculates the total subscription amount after a 5% discount for each customer.
  2. getSubscriptionDetails — finds all subscriptions for a specific customer by their ID.

Classes required:

  • Customer — customerId (int), customerName (String)
  • Subscription — subscriptionName (String), amount (double), validityInMonth (int), subscribedCustomers (List<Customer>)
  • Solution — calculateTotalAmount(Map<Integer, List<Subscription>>), getSubscriptionDetails(int, Map<Integer, List<Subscription>>)

Discount formula: TotalAmountAfterDiscount = TotalAmount - (TotalAmount * 5 / 100)

If the search customer ID is not found, print: "No customer found with ID: <id>"

Analysis

What This Question Tests

This is a heavy OOP + Collections question. It combines class composition, Map operations, list traversal, and a simple calculation — all wired together through Scanner input parsing.

Core Concepts Being Tested
  • Composition: A Subscription holds a List<Customer> — objects inside objects
  • Map<Integer, List<Subscription>>: Grouping subscriptions by customer ID
  • computeIfAbsent: Building a map where each key maps to a list
  • LinkedHashMap: Preserving insertion order so output matches expected order
  • Discount calculation: Simple math but easy to mess up the formula placement
  • Scanner input parsing: Multiple nested loops reading different data types

Data flow: Input → Objects → Map → Calculation
Read the Input Carefully

The biggest trap in this question is mis-reading which customers belong to which subscription. The customer lists are different for each subscription. Don't assume all customers subscribe to everything.

Algorithm

The Algorithm Step by Step

  • Step 1: Read number of subscriptions.
  • Step 2: For each subscription, read: name, amount, validity, number of customers, then each customer's ID and name. Create Subscription and Customer objects.
  • Step 3: Build a Map<Integer, List<Subscription>> (LinkedHashMap). For each subscription, loop through its customers. For each customer ID, add the subscription to that customer's list in the map.
  • Step 4: calculateTotalAmount: For each entry in the map, sum all subscription amounts for that customer. Apply 5% discount: total - (total * 5 / 100). Store in a result map.
  • Step 5: Print all customer totals.
  • Step 6: Read search customer ID.
  • Step 7: getSubscriptionDetails: Check if the map contains the ID. If yes, return the list of subscriptions. If no, return null.
  • Step 8: Print subscription names or "No customer found" message.
Why LinkedHashMap?

A regular HashMap doesn't guarantee insertion order. The expected output prints customers in the order they first appeared in the input. LinkedHashMap preserves insertion order, so your output matches exactly.

Full Code

Complete Solution

Three classes + main method with input parsing, map building, and both service methods.

Solution.java — Complete 
import java.util.*;

class Customer {
    private int customerId;
    private String customerName;

    public Customer(int customerId, String customerName) {
        this.customerId = customerId;
        this.customerName = customerName;
    }

    public int getCustomerId() { return customerId; }
    public String getCustomerName() { return customerName; }
}

class Subscription {
    private String subscriptionName;
    private double amount;
    private int validityInMonth;
    private List<Customer> subscribedCustomers;

    public Subscription(String subscriptionName, double amount, int validityInMonth, List<Customer> subscribedCustomers) {
        this.subscriptionName = subscriptionName;
        this.amount = amount;
        this.validityInMonth = validityInMonth;
        this.subscribedCustomers = subscribedCustomers;
    }

    public String getSubscriptionName() { return subscriptionName; }
    public double getAmount() { return amount; }
    public int getValidityInMonth() { return validityInMonth; }
    public List<Customer> getSubscribedCustomers() { return subscribedCustomers; }
}

public class Solution {

    public static Map<Integer, Double> calculateTotalAmount(Map<Integer, List<Subscription>> customerSubscriptions) {
        Map<Integer, Double> result = new LinkedHashMap<>();
        for (Map.Entry<Integer, List<Subscription>> entry : customerSubscriptions.entrySet()) {
            double total = 0;
            for (Subscription sub : entry.getValue()) {
                total += sub.getAmount();
            }
            double discounted = total - (total * 5.0 / 100);
            result.put(entry.getKey(), discounted);
        }
        return result;
    }

    public static List<Subscription> getSubscriptionDetails(int customerId, Map<Integer, List<Subscription>> customerSubscriptions) {
        if (customerSubscriptions.containsKey(customerId)) {
            return customerSubscriptions.get(customerId);
        }
        return null;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int numSubs = Integer.parseInt(sc.nextLine().trim());
        List<Subscription> allSubs = new ArrayList<>();

        for (int i = 0; i < numSubs; i++) {
            String name = sc.nextLine().trim();
            double amount = Double.parseDouble(sc.nextLine().trim());
            int validity = Integer.parseInt(sc.nextLine().trim());
            int numCust = Integer.parseInt(sc.nextLine().trim());
            List<Customer> customers = new ArrayList<>();
            for (int j = 0; j < numCust; j++) {
                int cId = Integer.parseInt(sc.nextLine().trim());
                String cName = sc.nextLine().trim();
                customers.add(new Customer(cId, cName));
            }
            allSubs.add(new Subscription(name, amount, validity, customers));
        }

        // Build customerSubscriptions map (LinkedHashMap for insertion order)
        Map<Integer, List<Subscription>> customerSubscriptions = new LinkedHashMap<>();
        for (Subscription sub : allSubs) {
            for (Customer c : sub.getSubscribedCustomers()) {
                customerSubscriptions.computeIfAbsent(c.getCustomerId(), k -> new ArrayList<>()).add(sub);
            }
        }

        // Method 1: Calculate totals
        Map<Integer, Double> totals = calculateTotalAmount(customerSubscriptions);
        System.out.println("Total amount after discount for each customer:");
        for (Map.Entry<Integer, Double> entry : totals.entrySet()) {
            System.out.println("Customer ID: " + entry.getKey() + ", Total Amount: " + entry.getValue());
        }

        // Method 2: Search by customer ID
        int searchId = Integer.parseInt(sc.nextLine().trim());
        List<Subscription> details = getSubscriptionDetails(searchId, customerSubscriptions);
        if (details == null) {
            System.out.println("No customer found with ID: " + searchId);
        } else {
            System.out.println("Subscriptions for Customer ID " + searchId + ":");
            for (Subscription sub : details) {
                System.out.println("- " + sub.getSubscriptionName());
            }
        }
    }
}
Line-by-line explanation
1
class Customer { private int customerId; private String customerName; }
Simple POJO with two private fields. The constructor sets them, getters expose them. This is the inner object that Subscription will hold a list of.
2
class Subscription { ... private List<Customer> subscribedCustomers; }
Composition in action. A Subscription doesn't just have primitive fields — it holds a List of Customer objects. This is the "has-a" relationship.
3
Map<Integer, List<Subscription>> customerSubscriptions = new LinkedHashMap<>();
The key data structure. Maps each customer ID (Integer) to a list of their subscriptions. LinkedHashMap keeps insertion order so output matches expected order.
4
customerSubscriptions.computeIfAbsent(c.getCustomerId(), k -> new ArrayList<>()).add(sub);
The most important line. computeIfAbsent says: "if this key doesn't exist yet, create a new empty list for it." Then .add(sub) adds the subscription to that list. This one line replaces a 5-line if-else block.
5
double discounted = total - (total * 5.0 / 100);
5% discount formula. Note 5.0 not 5 — using a double ensures we get decimal precision. 5/100 with integers would give 0 due to integer division.
6
if (customerSubscriptions.containsKey(customerId)) { return ... } return null;
containsKey() checks if the customer ID exists in the map. If yes, return their subscription list. If no, return null — the caller checks for null and prints the "not found" message.
7
int numSubs = Integer.parseInt(sc.nextLine().trim());
Read using nextLine().trim() and parse. This pattern avoids the classic nextInt() + nextLine() newline bug. Always use nextLine() and parse manually in PRA questions.

Watch Out

Common Mistakes

Wrong: Using HashMap

Map<Integer, List<Subscription>> map = new HashMap<>();

HashMap doesn't guarantee order. Your output might print Customer 3423 before 2345.

Output order mismatch = test case failure even if logic is correct.

Correct: Using LinkedHashMap

Map<Integer, List<Subscription>> map = new LinkedHashMap<>();

LinkedHashMap preserves insertion order. Customers appear in the order they were first encountered in the input.

Output matches expected order every time.

Mistake What Goes Wrong Fix
Using HashMap instead of LinkedHashMap Output order is random, test cases fail Always use LinkedHashMap when output order matters
Integer division: total * 5 / 100 If total is an int, 5/100 = 0 — no discount applied Use 5.0 / 100 or declare total as double
Mixing nextInt() and nextLine() nextInt() leaves a newline in the buffer. Next nextLine() reads an empty string. Use Integer.parseInt(sc.nextLine().trim()) for everything
Building the map wrong — subscription as key The map needs customer ID as key, not subscription name Loop through each subscription's customer list and flip the relationship
Forgetting computeIfAbsent NullPointerException when adding to a list that doesn't exist yet computeIfAbsent(id, k -> new ArrayList<>()).add(sub) handles both cases
Applying discount per subscription instead of per customer total Different rounding, wrong totals Sum ALL subscription amounts first, then apply 5% discount once on the total
Exam Strategy

This is a major question — it tests composition, Maps, and method design all at once. Start by writing the Customer and Subscription classes first (5 min), then the input parsing (10 min), then the map-building logic (10 min), then the two methods (10 min each). Test with TC1 before submitting. The map-building step is where most people get stuck.
