Mock 6 — Question 2

TrainTicket

Major 60–75 min

Question Statement

Create a class TrainTicket with the following attributes:

  • ticketId (int)
  • travellerName (String)
  • source (String)
  • destination (String)
  • price (double)
  • tierClass (String)

Write a parameterized constructor and generate getters and setters.

Method 1: findTicketWithMinPriceForGivenSource(TrainTicket[] tickets, String source) — Find the ticket with the minimum price for the given source. Return null if no ticket matches. Case-insensitive comparison.

Method 2: findAveragePriceOfTicketsForTierClass(TrainTicket[] tickets, String tierClass) — Return the average price of tickets for the given tier class. Return 0 if no ticket matches. Case-insensitive comparison.

Main method output:

  • Method 1: If result is not null, print ticketId, source, price (each on a new line). If null: "No such Ticket found with mentioned attribute"
  • Method 2: If result is not 0, print the average. If 0: "No such tier class found with mentioned attribute"
Analysis

What This Question Tests

Standard OOP + array traversal pattern. Class with 6 fields, two static search/aggregate methods. The twist is combining filtering (by source/tierClass) with aggregation (min price, average price) — and case-insensitive matching.

Two Patterns to Recognize
  • Method 1 — Filter + Find Min: Loop through array, filter by source (case-insensitive), track the ticket with the lowest price. Classic "min tracker" pattern.
  • Method 2 — Filter + Average: Loop through array, filter by tierClass (case-insensitive), accumulate total and count, then divide. Classic "running average" pattern.
  • Both methods return a sentinel value (null / 0) when nothing matches — main method checks and prints accordingly.

The pattern: filter by attribute, then aggregate (min / average)
Case-Insensitive is Critical

Both methods use equalsIgnoreCase(). If the input is "Chennai" and stored data is "chennai", a regular equals() would miss the match. Always use equalsIgnoreCase() for string comparisons in PRA questions unless told otherwise.

Algorithm

The Algorithm Step by Step

  • Step 1: Read N (number of tickets). Create a TrainTicket[] array of size N.
  • Step 2: For each ticket, read 6 fields (id, name, source, destination, price, tierClass) and construct the object.
  • Step 3: Read the search parameters: searchSource and searchTierClass.
  • Step 4 (Method 1): Loop through all tickets. For each ticket whose source matches (case-insensitive), compare its price with the current minimum. Track the ticket with the lowest price.
  • Step 5 (Method 2): Loop through all tickets. For each ticket whose tierClass matches (case-insensitive), add its price to a running total and increment count. Divide total by count at the end.
  • Step 6: Print results based on null/0 checks.
Min-Tracker Pattern

Initialize minTicket = null. On the first match, set it directly. On subsequent matches, compare: if (minTicket == null || t.getPrice() < minTicket.getPrice()). This handles both "first assignment" and "better candidate found" in one condition.

Full Code

Complete Solution

Two classes: TrainTicket (model) and Solution (logic + main). Standard PRA structure.

Solution.java — Complete 
import java.util.*;

class TrainTicket {
    private int ticketId;
    private String travellerName;
    private String source;
    private String destination;
    private double price;
    private String tierClass;

    public TrainTicket(int ticketId, String travellerName, String source, String destination, double price, String tierClass) {
        this.ticketId = ticketId;
        this.travellerName = travellerName;
        this.source = source;
        this.destination = destination;
        this.price = price;
        this.tierClass = tierClass;
    }

    public int getTicketId() { return ticketId; }
    public String getTravellerName() { return travellerName; }
    public String getSource() { return source; }
    public String getDestination() { return destination; }
    public double getPrice() { return price; }
    public String getTierClass() { return tierClass; }

    public void setTicketId(int ticketId) { this.ticketId = ticketId; }
    public void setTravellerName(String travellerName) { this.travellerName = travellerName; }
    public void setSource(String source) { this.source = source; }
    public void setDestination(String destination) { this.destination = destination; }
    public void setPrice(double price) { this.price = price; }
    public void setTierClass(String tierClass) { this.tierClass = tierClass; }
}

public class Solution {

    public static TrainTicket findTicketWithMinPriceForGivenSource(TrainTicket[] tickets, String source) {
        TrainTicket minTicket = null;
        for (TrainTicket t : tickets) {
            if (t.getSource().equalsIgnoreCase(source)) {
                if (minTicket == null || t.getPrice() < minTicket.getPrice()) {
                    minTicket = t;
                }
            }
        }
        return minTicket;
    }

    public static double findAveragePriceOfTicketsForTierClass(TrainTicket[] tickets, String tierClass) {
        double total = 0;
        int count = 0;
        for (TrainTicket t : tickets) {
            if (t.getTierClass().equalsIgnoreCase(tierClass)) {
                total += t.getPrice();
                count++;
            }
        }
        return count == 0 ? 0 : total / count;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = Integer.parseInt(sc.nextLine().trim());
        TrainTicket[] tickets = new TrainTicket[n];

        for (int i = 0; i < n; i++) {
            int id = Integer.parseInt(sc.nextLine().trim());
            String name = sc.nextLine().trim();
            String source = sc.nextLine().trim();
            String dest = sc.nextLine().trim();
            double price = Double.parseDouble(sc.nextLine().trim());
            String tier = sc.nextLine().trim();
            tickets[i] = new TrainTicket(id, name, source, dest, price, tier);
        }

        String searchSource = sc.nextLine().trim();
        String searchTier = sc.nextLine().trim();

        TrainTicket minTicket = findTicketWithMinPriceForGivenSource(tickets, searchSource);
        if (minTicket != null) {
            System.out.println(minTicket.getTicketId());
            System.out.println(minTicket.getSource());
            System.out.println(minTicket.getPrice());
        } else {
            System.out.println("No such Ticket found with mentioned attribute");
        }

        double avg = findAveragePriceOfTicketsForTierClass(tickets, searchTier);
        if (avg != 0) {
            System.out.println(avg);
        } else {
            System.out.println("No such tier class found with mentioned attribute");
        }
    }
}
Line-by-line explanation
1
class TrainTicket { private int ticketId; ... }
Model class with 6 private fields. This is the data container — each object represents one train ticket. Fields are private (encapsulation), accessed through getters/setters.
2
public TrainTicket(int ticketId, String travellerName, ...) { this.ticketId = ticketId; ... }
Parameterized constructor. Takes all 6 fields and assigns them using this. to distinguish parameter names from field names. Called when creating each ticket in the loop.
3
TrainTicket minTicket = null;
Initialize the tracker as null. This means "no matching ticket found yet." The null check later (minTicket == null) handles the first match automatically.
4
if (t.getSource().equalsIgnoreCase(source))
equalsIgnoreCase() compares strings ignoring upper/lower case. "Chennai" matches "chennai" matches "CHENNAI". Always use this in PRA unless told otherwise.
5
if (minTicket == null || t.getPrice() < minTicket.getPrice())
Two conditions combined with OR. First match: minTicket == null is true, so we assign it. Later matches: we compare prices. If current is cheaper, update. This is the min-tracker pattern.
6
return count == 0 ? 0 : total / count;
Ternary operator. If no tickets matched the tierClass (count is 0), return 0 to avoid division by zero. Otherwise, return the average (total / count).
7
int n = Integer.parseInt(sc.nextLine().trim());
Read the first line as the number of tickets. nextLine() reads the whole line, trim() removes whitespace, parseInt() converts to int. Using nextLine() instead of nextInt() avoids the newline-leftover bug.
8
double price = Double.parseDouble(sc.nextLine().trim());
Same pattern as parseInt but for doubles. Each field is on its own line, so we read line-by-line. This is the safest input approach for PRA — avoids Scanner quirks.

Watch Out

Common Mistakes

Wrong: Using equals() instead of equalsIgnoreCase()

if (t.getSource().equals(source))

If user inputs "Chennai" but data has "chennai", this fails silently — returns null even though there IS a matching ticket.

Half the PRA test cases are designed to catch exactly this mistake.

Correct: Always equalsIgnoreCase()

if (t.getSource().equalsIgnoreCase(source))

"Chennai", "chennai", "CHENNAI" all match. Safe regardless of input casing.

Unless the question explicitly says "case-sensitive", always use equalsIgnoreCase().

Mistake What Goes Wrong Fix
Using nextInt() / nextDouble() instead of nextLine() Scanner leaves a newline in the buffer, next nextLine() reads an empty string Always use Integer.parseInt(sc.nextLine().trim()) and Double.parseDouble(sc.nextLine().trim())
Forgetting to handle "not found" case NullPointerException when calling methods on null, or printing 0.0 instead of the message Always check: if (minTicket != null) and if (avg != 0) before printing
Initializing minPrice to 0 No ticket's price will be less than 0, so the tracker never updates Initialize minTicket = null and use the null-check pattern: if (minTicket == null || ...)
Integer division in average If total and count are both int, Java does integer division (truncates decimals) Declare total as double so the division produces a decimal result
Missing setters Question says "generate getters and setters" — missing setters may lose marks Include all 6 setters even if the solution doesn't use them
Multi-word input (e.g., "gandhi nagar") Using sc.next() only reads "gandhi", leaving "nagar" for the next read Always use sc.nextLine() — it reads the entire line including spaces
Exam Strategy: This is a Template Question

Almost every major PRA question follows this exact pattern: model class + 2 static methods (one returns object, one returns number) + main with null/0 checks. Once you memorize this structure, you just change the field names and method logic. Write the class first, then the two methods, then main. Keep the Scanner input block as a template you can reuse every time.
