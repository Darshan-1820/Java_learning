PRA Cheat Sheet

Everything you need on one page. Print it, keep it next to you.

1. POJO Template

class X {
  private Type f1;
  private Type f2;

  // Parameterized constructor
  public X(Type f1, Type f2) {
    this.f1 = f1;
    this.f2 = f2;
  }

  // Getters
  public Type getF1() { return f1; }
  public Type getF2() { return f2; }

  // Setters
  public void setF1(Type f1) { this.f1 = f1; }
  public void setF2(Type f2) { this.f2 = f2; }
}

For List<X> field: getter returns List<X>, setter takes List<X>.

2. Input Reading

Scanner sc = new Scanner(System.in);

int n = Integer.parseInt(sc.nextLine().trim());
String s = sc.nextLine().trim();
double d = Double.parseDouble(sc.nextLine().trim());
boolean b = Boolean.parseBoolean(sc.nextLine().trim());
NEVER use sc.nextInt() or sc.nextDouble() in PRA!

Always use nextLine().trim() then parse. Avoids the newline bug that fails test cases.

3. Filter Template

public List<X> filterMethod(List<X> list, Type param) {
  List<X> result = new ArrayList<>();
  for (X item : list) {
    if (item.getField().equals(param)) {
      result.add(item);
    }
  }
  return result;
}

Use .equals() for String/Object. Use == or .equals() for Integer. Use > < for numeric comparisons.

4. Aggregate Template (Avg/Sum)

public double getAverage(List<X> list) {
  double sum = 0;
  int count = 0;
  for (X item : list) {
    if (condition) {
      sum += item.getVal();
      count++;
    }
  }
  return count > 0 ? sum / count : 0;
}

If question says "returns null if not found" use Double return type and return null;.

5. Try-Catch (SEPARATE for each method)

// Method 1 - its own try-catch
try {
  List<X> r = obj.method1(list, param);
  for (X x : r) {
    System.out.println(x.toString());
  }
} catch (Exception1 e) {
  System.out.println(e.getMessage());
}

// Method 2 - its own try-catch
try {
  double r = obj.method2(list, param);
  System.out.println(r);
} catch (Exception2 e) {
  System.out.println(e.getMessage());
}
NEVER put two methods in the same try block!

10. Custom Exception

class MyException extends Exception {
  MyException(String msg) {
    super(msg);
  }
}

In the method:

public X method(...) throws MyException {
  if (condition) {
    throw new MyException("Exact message");
  }
  // normal logic
}

In main:

try {
  result = obj.method(params);
} catch (MyException e) {
  System.out.println(e.getMessage());
}

11. Integer vs int

UseWhen
IntegerWhen question says Integer, in Collections, nullable values
intLoops, math, local variables
DoubleWhen method must return null
doubleMath, averages, prices

They auto-convert (autoboxing). Integer x = 5; works. int y = Integer.valueOf(5); works.

Key: Collections (List<>, Map<>) MUST use wrappers. Cannot use int, double in generics.
