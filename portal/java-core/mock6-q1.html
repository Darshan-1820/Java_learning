Mock 6 — Question 1

Music-Fest

Major 60–75 min

Question Statement

In the ABC Music Festival, musicians perform songs. You need to model this with two classes and a service class.

Song has: songId (int), songName (String), duration (int), genre (String).

Musician has: musicianId (int), musicianName (String), songs (List<Song>).

MusicService methods:

  • findSongsByGenre(List<Musician>, String genre) — returns a List<String> of song names matching the genre across all musicians. If none found, print: "Songs in <genre> genre are not present."
  • findAverageDurationByMusician(List<Musician>, String musicianName) — returns the average duration (double) of all songs by that musician.

Input: Read numMusicians. For each musician: musicianId, musicianName, numSongs. For each song: songId, songName, duration, genre. Then read searchGenre and searchMusicianName.

Output format:

Songs in Pop genre :
Twinkling Stars
Living
Average duration of songs composed by Alice Green : 190.0 seconds

Note the space before the colon in both output lines.

Analysis

What This Question Tests

This is a classic composition + list traversal problem. A Musician HAS-A list of Songs. You iterate through nested structures, filter by a field, and compute an aggregate (average).

Core Concepts Tested
  • Composition: Musician contains a List<Song> — nested object structure
  • Nested iteration: Loop through musicians, then loop through each musician's songs
  • Filtering: Match songs by genre using equalsIgnoreCase()
  • Aggregation: Sum durations and divide by count for average
  • Scanner input: Multi-level nested input parsing (musicians → songs)
Watch the Output Format

The output has a space before the colon: "Songs in Pop genre :" not "Songs in Pop genre:". Same for the average line. Missing this space will fail hidden test cases.

Algorithm

The Algorithm Step by Step

  • Step 1: Create Song class with 4 fields (songId, songName, duration, genre) + constructor + getters.
  • Step 2: Create Musician class with 3 fields (musicianId, musicianName, songs list) + constructor + getters.
  • Step 3: In main, read numMusicians. For each musician, read their details and their songs in a nested loop.
  • Step 4: Read searchGenre and searchMusicianName.
  • Step 5: findSongsByGenre — Iterate all musicians → all songs. If song's genre matches (case-insensitive), add songName to result list. Return the list.
  • Step 6: If result list is empty, print the "not present" message. Otherwise print the header line + each song name on its own line.
  • Step 7: findAverageDurationByMusician — Find the musician by name (case-insensitive). Sum all song durations. Divide by song count. Return as double.
  • Step 8: Print the average line with the exact format.
Why double, not int?

The average must be a double. If you use int total, the division total / size does integer division (truncates decimals). Use double total = 0 so the division automatically returns a double. E.g., 380 / 2 = 190 (int), but 380.0 / 2 = 190.0 (double).

Full Code

Complete Solution

Three classes (Song, Musician, Solution) + two service methods + main with nested input parsing.

Solution.java — Complete 
import java.util.*;

class Song {
    private int songId;
    private String songName;
    private int duration;
    private String genre;

    public Song(int songId, String songName, int duration, String genre) {
        this.songId = songId;
        this.songName = songName;
        this.duration = duration;
        this.genre = genre;
    }

    public int getSongId() { return songId; }
    public String getSongName() { return songName; }
    public int getDuration() { return duration; }
    public String getGenre() { return genre; }
}

class Musician {
    private int musicianId;
    private String musicianName;
    private List<Song> songs;

    public Musician(int musicianId, String musicianName, List<Song> songs) {
        this.musicianId = musicianId;
        this.musicianName = musicianName;
        this.songs = songs;
    }

    public int getMusicianId() { return musicianId; }
    public String getMusicianName() { return musicianName; }
    public List<Song> getSongs() { return songs; }
}

public class Solution {

    public static List<String> findSongsByGenre(List<Musician> musicians, String genre) {
        List<String> result = new ArrayList<>();
        for (Musician m : musicians) {
            for (Song s : m.getSongs()) {
                if (s.getGenre().equalsIgnoreCase(genre)) {
                    result.add(s.getSongName());
                }
            }
        }
        return result;
    }

    public static double findAverageDurationByMusician(List<Musician> musicians, String musicianName) {
        for (Musician m : musicians) {
            if (m.getMusicianName().equalsIgnoreCase(musicianName)) {
                double total = 0;
                for (Song s : m.getSongs()) {
                    total += s.getDuration();
                }
                return total / m.getSongs().size();
            }
        }
        return 0;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int numMusicians = Integer.parseInt(sc.nextLine().trim());
        List<Musician> musicians = new ArrayList<>();

        for (int i = 0; i < numMusicians; i++) {
            int mId = Integer.parseInt(sc.nextLine().trim());
            String mName = sc.nextLine().trim();
            int numSongs = Integer.parseInt(sc.nextLine().trim());
            List<Song> songs = new ArrayList<>();
            for (int j = 0; j < numSongs; j++) {
                int sId = Integer.parseInt(sc.nextLine().trim());
                String sName = sc.nextLine().trim();
                int duration = Integer.parseInt(sc.nextLine().trim());
                String genre = sc.nextLine().trim();
                songs.add(new Song(sId, sName, duration, genre));
            }
            musicians.add(new Musician(mId, mName, songs));
        }

        String searchGenre = sc.nextLine().trim();
        String searchMusician = sc.nextLine().trim();

        List<String> songsByGenre = findSongsByGenre(musicians, searchGenre);
        if (songsByGenre.isEmpty()) {
            System.out.println("Songs in " + searchGenre + " genre are not present.");
        } else {
            System.out.println("Songs in " + searchGenre + " genre :");
            for (String name : songsByGenre) {
                System.out.println(name);
            }
        }

        double avg = findAverageDurationByMusician(musicians, searchMusician);
        System.out.println("Average duration of songs composed by " + searchMusician + " : " + avg + " seconds");
    }
}
Line-by-line explanation
1
class Song { private int songId; ... }
The Song class encapsulates 4 fields: songId, songName, duration, genre. All fields are private with public getters — standard encapsulation pattern. The constructor takes all 4 values.
2
class Musician { ... private List<Song> songs; }
The Musician class uses composition: it HAS-A List<Song>. This is the core OOP concept being tested. A musician doesn't inherit from Song — they contain songs.
3
for (Musician m : musicians) { for (Song s : m.getSongs()) { ... } }
Nested for-each loop. The outer loop goes through each musician. The inner loop goes through that musician's song list. This is how you traverse a composed/nested data structure.
4
s.getGenre().equalsIgnoreCase(genre)
Case-insensitive comparison. equalsIgnoreCase() ensures "Pop" matches "pop", "POP", etc. Always use this instead of equals() when the problem doesn't specify exact case matching.
5
double total = 0; ... return total / m.getSongs().size();
Using double total instead of int total ensures the division produces a decimal result. 380.0 / 2 = 190.0. If total were int: 380 / 2 = 190 (no decimal).
6
int numSongs = Integer.parseInt(sc.nextLine().trim());
Each musician has a variable number of songs. We read this count first, then loop that many times to read each song's details. This is the nested input pattern — an outer loop (musicians) with an inner loop (songs per musician).
7
"Songs in " + searchGenre + " genre :"
The output format has a space before the colon. This is an exact format match requirement. Missing the space = failed test case. Always copy the output format exactly from the question.

Watch Out

Common Mistakes

Wrong: Integer division for average

Using int total and dividing by int size:

int total = 380; return total / 2; → returns 190 (not 190.0)

For values like 621 / 3, you'd get 207 instead of 207.0.

Even when the result is a whole number, the output expects a double (190.0, not 190).

Correct: Use double for total

Declare double total = 0;

double total = 380.0; return total / 2; → returns 190.0

This ensures the division always produces a double result, matching the expected output format.

Always use double for any calculation that involves averages or division.

Mistake What Goes Wrong Fix
Missing space before colon in output Output doesn't match expected format exactly — hidden test cases fail Use " genre :" and musicianName + " : " with spaces around colon
Using equals() instead of equalsIgnoreCase() "pop" won't match "Pop" — genre/name comparisons fail Always use equalsIgnoreCase() for string matching in these problems
Using sc.next() instead of sc.nextLine() Names with spaces like "Alice Green" get split — "Alice" goes to name, "Green" bleeds into next read Use sc.nextLine().trim() for all string inputs
Using int total for duration sum Average returns int instead of double — output shows 190 instead of 190.0 Declare as double total = 0; so division returns double
Forgetting the period in "not present" message "Songs in Rap genre are not present" without the period at the end Include the period: "Songs in " + genre + " genre are not present."
Printing songs on one line separated by commas Expected output has each song on its own line Loop through the result list and System.out.println() each song name separately
Exam Strategy: Tackle This Pattern

This is the most common major question pattern: two model classes (composition) + two service methods (filter + aggregate). If you can code this pattern from memory, you can solve 80% of major questions. Practice the input parsing loop — that's where most people get stuck under time pressure.
