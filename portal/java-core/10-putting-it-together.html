Master Template

The Universal PRA Template

This is the master template that fits 90% of PRA questions. It has all the pieces you've learned in previous chapters: POJO class, filter method, aggregate method, Scanner input, and output formatting. Copy this, modify the names and conditions, and you have a working solution.

Solution.java — Universal PRA Template 
import java.util.*;

// ============================================
// STEP 1: POJO CLASS
// ============================================
class ClassName {
    private int id;
    private String name;
    private double value;

    // Parameterized Constructor
    public ClassName(int id, String name, double value) {
        this.id = id;
        this.name = name;
        this.value = value;
    }

    // Getters
    public int getId() { return id; }
    public String getName() { return name; }
    public double getValue() { return value; }

    // Setters
    public void setId(int id) { this.id = id; }
    public void setName(String name) { this.name = name; }
    public void setValue(double value) { this.value = value; }
}

public class Solution {

    // ============================================
    // STEP 2: METHOD 1 — Filter or Search
    // ============================================
    public static ArrayList<ClassName> filterByCondition(
            Collection<ClassName> list, String filterParam) {
        ArrayList<ClassName> result = new ArrayList<>();
        for (ClassName item : list) {
            if (item.getName().equalsIgnoreCase(filterParam)) {
                result.add(item);
            }
        }
        return result;
    }

    // ============================================
    // STEP 3: METHOD 2 — Aggregate or Calculate
    // ============================================
    public static double calculateAverage(Collection<ClassName> list) {
        double sum = 0;
        int count = 0;
        for (ClassName item : list) {
            if (item.getValue() > 0) {   // condition (optional)
                sum += item.getValue();
                count++;
            }
        }
        return count > 0 ? sum / count : 0;  // SAFE division
    }

    // ============================================
    // STEP 4: MAIN METHOD
    // ============================================
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // Read number of objects
        int n = sc.nextInt();
        ArrayList<ClassName> list = new ArrayList<>();

        // Read each object
        for (int i = 0; i < n; i++) {
            int id = sc.nextInt();
            sc.nextLine();               // consume \n after int
            String name = sc.nextLine(); // read name
            double value = sc.nextDouble();
            list.add(new ClassName(id, name, value));
        }

        // Read method parameters
        sc.nextLine();                         // consume \n from loop
        String searchParam = sc.nextLine();    // for method 1

        // Call Method 1 and handle output
        ArrayList<ClassName> result1 = filterByCondition(list, searchParam);
        if (result1 != null && !result1.isEmpty()) {
            for (ClassName item : result1) {
                System.out.println(item.getId());
                System.out.println(item.getName());
                System.out.println(item.getValue());
            }
        } else {
            System.out.println("No results found");
        }

        // Call Method 2 and handle output
        double result2 = calculateAverage(list);
        if (result2 > 0) {
            System.out.println(result2);
        } else {
            System.out.println("No results found");
        }
    }
}

  3. Replace the fields (id, name, value) with the actual fields
  4. Modify the method conditions to match what the question asks
  5. Adjust the output format to match the expected output exactly

You now have a working skeleton. Just fill in the blanks.

Strategy

Step-by-Step Approach

You have 50 minutes for the major question. Here's how to spend them wisely. Don't jump into coding — spending 5 minutes reading saves 15 minutes debugging.

  1. Read the Question (5 min)
    Identify these 5 things before writing any code:
    • Class name — what POJO to create
    • Fields — what private variables with their types
    • Method signatures — method names, parameters, return types
    • Input format — exact order of inputs
    • Output format — exact expected output (copy it!)
  2. Write the POJO (5 min)
    Private fields, parameterized constructor, all getters and setters. This is mechanical — no thinking needed. Just map fields from the question.
  3. Write Method 1 (10 min)
    Usually a filter: loop through collection, check condition, add matches to result list. Use the filter template from Chapter 6.
  4. Write Method 2 (10 min)
    Usually an aggregate: loop, accumulate sum/count, return calculated value. Or a search with nested loops. Use the appropriate template.
  5. Write Main (15 min)
    Scanner input reading (watch the newline trap!), create objects in loop, call methods, format output exactly as specified.
  6. Test (5 min)
    Run with the sample input from the question. Compare output character-by-character. Check edge cases mentally.
🎯 Time Management

Don't spend more than 5 minutes stuck on any one part. If Method 2 is confusing, write a placeholder and move to Main. A solution with Method 1 working + correct input reading + correct output format can still score partial marks. A perfect Method 2 with broken input reading scores zero.

Watch Out

Edge Cases to Watch

HackerRank always includes edge case test inputs. These are the sneaky inputs designed to break your code if you didn't think about them. Here are the most common ones in PRA.

⚠ Edge Case 1: All Values Negative

Scenario: "Calculate average of positive gains" but ALL investors have negative gains.

What happens: count stays at 0. If you do sum / count, you get ArithmeticException (crash).

Fix: return count > 0 ? sum / count : 0;

⚠ Edge Case 2: Empty Collection (n = 0)

Scenario: The input starts with 0 — zero objects to read.

What happens: Your loop runs 0 times. If you then try to search or aggregate, the list is empty.

Fix: Your filter returns empty list, your aggregate returns 0 or "not found". Both should work naturally if you coded the patterns correctly.

⚠ Edge Case 3: No Matches Found

Scenario: "Find all books with genre 'Horror'" but no books have that genre.

What happens: Result list is empty. You need to print the error message, not iterate over an empty list.

Fix: Always check if (result.isEmpty()) before printing results.

⚠ Edge Case 4: Case Sensitivity

Scenario: Input has "crime" but your code checks for "Crime".

What happens: .equals("Crime") returns false for "crime". Filter finds nothing.

Fix: Always .equalsIgnoreCase(). This should be automatic by now.

⚠ Edge Case 5: double vs int Output

Scenario: Expected output is 50 sq.m but you used double, printing 50.0 sq.m.

What happens: HackerRank marks it wrong. "50 sq.m" is not equal to "50.0 sq.m".

Fix: Check expected output carefully. Use int when output shows no decimal.

Decode

Pattern Recognition

When you read the PRA question, specific keywords tell you exactly which code pattern to use. Learn to spot these keywords and you'll know the solution structure before you write a single line.

Question Says Pattern Template to Use
"find all...matching" Filter Loop + Condition + Add to result list
"return collection of..." Filter Return ArrayList<T>
"average of..." Aggregate Sum + Count + Divide (with zero check)
"sum of..." / "total of..." Aggregate Sum accumulator + return
"if no...found, print" Error handling Check empty result, print exact message
"case insensitive" String compare equalsIgnoreCase()
"insertion order" Collection choice Use ArrayList (preserves order)
"calculate discount" Filter + Modify Filter + setPrice(newPrice) + add to result
"create custom exception" Exception extends Exception + super(msg) + try-catch
"throw...if condition" Exception if (cond) throw new XException(msg)

Reading the Method Signature

The question always gives you the method signature. It tells you everything:

  public static ArrayList<Book> findBooksByGenre(Collection<Book> books, String genre)
  ~~~~~~ ~~~~~~ ~~~~~~~~~~~~~~~~ ~~~~~~~~~~~~~~~~ ~~~~~~~~~~~~~~~~~~~~~~  ~~~~~~~~~~~~
    |      |          |                |                   |                   |
  access static   return type     method name        input collection     filter param
    |      |          |                |                   |                   |
  always always  ArrayList = filter  descriptive name  what to loop over  what to match
  public static  double = aggregate
                 void = prints directly
The return type tells you which pattern: ArrayList = Filter, double/int = Aggregate, void = Print directly
🎯 Quick Decision Tree

Return type is ArrayList<T> or List<T>? → Use Filter Pattern

Return type is double, int, or Double? → Use Aggregate Pattern

Return type is void? → Prints results directly (search pattern)

Has String parameter? → Will use equalsIgnoreCase()

Has int discountPercent parameter? → Calculate price - (price * discount / 100.0)
