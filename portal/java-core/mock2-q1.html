Section 01

Question Statement

Mock Test 2, Question 1. This is the HARDEST PRA pattern — composition with nested lists and two search methods across nested data.

PRA Major 50-65 min

CountryLeagueSponsors

Cricketer:

  • CricketerId (int)
  • CricketerName (String)
  • CricketerAge (int)
  • CricketerGender (String)
  • List<Statistics>

Statistics:

  • Rank (int)
  • LeagueName (String)
  • PerformanceAverage (double)
  • CountryName (String)

Create a business class (Solution.java) and implement:

findCricketerByLeagueName method:
Takes a collection of Cricketer (Array or List), and a LeagueName as input. Searches the Cricketer by Statistics and print Cricketer name if the LeagueName is available, else print "No Cricketer Found".

findTotalOfPerformanceAverageByCountryName method:
Takes a list of Cricketer and Country name as input. Should return the TotalPerformance of all the Statistics with same CountryName else return null if not found and print "No Statistics are available With the Country : " and Country name.

Note: All String comparison needs to be case in-sensitive.

Input format:

  1. Read number of Cricketer objects
  2. Read attributes for each Cricketer
  3. Read number of Statistics for each Cricketer
  4. Read Statistics attributes
  5. Repeat for all cricketers
  6. Read parameters for methods (league name, country name)
Section 02

Analysis

This is the most complex PRA pattern. Two POJO classes, composition (one contains a list of the other), and nested loop searching.

What Makes This Hard?
  • Two POJO classes: Cricketer and Statistics — both need private fields, constructors, getters, setters
  • Composition: Cricketer has a List<Statistics> inside it — one class contains a list of another class
  • Nested loops: To search by league or country, you must loop through cricketers, then for each cricketer, loop through their statistics list
  • Case-insensitive: All string comparisons use equalsIgnoreCase()
  • Complex Scanner reading: Reading cricketers with nested statistics inside a loop is tricky

│                     COMPOSITION                       │

│   Cricketer                                           │
│   ├── cricketerId: int                                │
│   ├── cricketerName: String                           │
│   ├── cricketerAge: int                               │
│   ├── cricketerGender: String                         │
│   └── statistics: List<Statistics>                   │

│   To search, you must check EACH statistic of        │

PRA Exam Tip

The composition field List<Statistics> statistics is just another field. The constructor takes it as a parameter, and the getter returns it. Nothing special — treat it like any other field. The "magic" happens when you call cricketer.getStatistics() and get a list you can loop through.

Section 05

Method 1 — findCricketerByLeagueName

Search through each cricketer's statistics list for a matching league name. If found, print the cricketer's name.

findCricketerByLeagueName 
public static void findCricketerByLeagueName(List<Cricketer> cricketers, String leagueName) {
    boolean found = false;
    for (Cricketer c : cricketers) {
        for (Statistics stat : c.getStatistics()) {
            if (stat.getLeagueName().equalsIgnoreCase(leagueName)) {
                System.out.println(c.getCricketerName());
                found = true;
                break;  // found in this cricketer's stats, move to next cricketer
            }
        }
    }
    if (!found) {
        System.out.println("No Cricketer Found");
    }
}
Line-by-line explanation
1
public static void findCricketerByLeagueName(...)
Returns void — this method prints directly instead of returning a value. Takes the list and a search string.
2
boolean found = false;
Flag to track if ANY cricketer matched. If still false at the end, we print the "not found" message.
3-4
for (Cricketer c : cricketers) { for (Statistics stat : c.getStatistics()) {
NESTED LOOP: outer loop goes through each cricketer. Inner loop goes through that cricketer's statistics list. This is the key pattern for composition questions.
5
if (stat.getLeagueName().equalsIgnoreCase(leagueName))
equalsIgnoreCase is critical — the question says "case in-sensitive." If input is "bigbashleague" and data has "BigBashLeague", this still matches.
6-8
print name, set found = true, break
Print the cricketer's name (not the stat's data). break exits the inner loop only — so if a cricketer has multiple stats with the same league, their name prints only once.
Why the break Matters

Without break, if a cricketer has 3 statistics all in "BigBashLeague", the cricketer's name would print 3 times. The break ensures each cricketer's name prints at most once.

Section 06

Method 2 — findTotalOfPerformanceAverageByCountryName

Sum up all performanceAverage values from statistics that match the given country name.

findTotalOfPerformanceAverageByCountryName 
public static Double findTotalOfPerformanceAverageByCountryName(List<Cricketer> cricketers, String countryName) {
    double total = 0;
    boolean found = false;
    for (Cricketer c : cricketers) {
        for (Statistics stat : c.getStatistics()) {
            if (stat.getCountryName().equalsIgnoreCase(countryName)) {
                total += stat.getPerformanceAverage();
                found = true;
            }
        }
    }
    if (!found) {
        System.out.println("No Statistics are available With the Country : " + countryName);
        return null;
    }
    return total;
}
Line-by-line explanation
1
public static Double findTotalOf...
Returns Double (capital D = wrapper class), not double (lowercase = primitive). This is because the method needs to return null when not found, and primitive double cannot be null.
2-3
double total = 0; boolean found = false;
total accumulates the sum. found tracks whether any stat matched the country.
4-11
nested loop: check each stat's country, add to total
Same nested loop pattern as method 1. No break here because we want to sum ALL matching statistics, not stop at the first one.
12-15
if (!found) { print message, return null; }
If no statistics matched the country, print the error message AND return null. The main method checks for null to know whether to print the total.
Double vs double

double (lowercase) is a primitive — it holds a number and cannot be null. Double (uppercase) is a wrapper class — it can hold a number OR null. Since the question says "return null if not found," we must use the wrapper class Double as the return type.

Section 07

The Main Method — Complex Scanner Reading

The hardest part of this question is reading the nested input correctly. Each cricketer has their own statistics, and you must read the statistics INSIDE the cricketer reading loop.

main method 
public static void main(String[] args) {
    Scanner sc = new Scanner(System.in);
    int n = sc.nextInt();
    List<Cricketer> cricketers = new ArrayList<>();

    for (int i = 0; i < n; i++) {
        int id = sc.nextInt();
        sc.nextLine();
        String name = sc.nextLine();
        int age = sc.nextInt();
        sc.nextLine();
        String gender = sc.nextLine();

        int numStats = sc.nextInt();
        List<Statistics> statsList = new ArrayList<>();
        for (int j = 0; j < numStats; j++) {
            int rank = sc.nextInt();
            sc.nextLine();
            String league = sc.nextLine();
            double perfAvg = sc.nextDouble();
            sc.nextLine();
            String country = sc.nextLine();
            statsList.add(new Statistics(rank, league, perfAvg, country));
        }

        cricketers.add(new Cricketer(id, name, age, gender, statsList));
    }

    sc.nextLine();
    String searchLeague = sc.nextLine();
    String searchCountry = sc.nextLine();

    findCricketerByLeagueName(cricketers, searchLeague);

    Double total = findTotalOfPerformanceAverageByCountryName(cricketers, searchCountry);
    if (total != null) {
        System.out.println("Total PerformanceAverage " + searchCountry + " : " + total);
    }
}
Line-by-line explanation
1-4
Scanner, read n, create list
Standard setup. Read the number of cricketers to create.
6-12
Read cricketer fields: id, name, age, gender
The sc.nextLine() after sc.nextInt() appears TWICE here (after id and after age) because both are followed by a String read. This is the newline trap doubled.
14-24
Read numStats, then inner loop for statistics
NESTED READING: inside the cricketer loop, we read how many statistics this cricketer has, then loop AGAIN to read each statistic's fields. The same nextLine() pattern applies inside too.
26
cricketers.add(new Cricketer(id, name, age, gender, statsList));
Create the Cricketer with its completed statistics list and add to the main list. The statsList is built fresh for each cricketer.
29-31
Read search parameters
After all cricketers are read, read the league name and country name to search for. The sc.nextLine() on line 29 consumes the leftover newline from the last nextDouble() or nextInt().
33-37
Call both methods, handle output
Method 1 prints directly (void). Method 2 returns a Double — if not null, print the total with the exact format string. If null, the method already printed the error message.
The Nested Scanner Trap

This question has sc.nextLine() after sc.nextInt() in MULTIPLE places — after reading cricketer id, cricketer age, stat rank, and stat performanceAverage. Missing any single one will corrupt all subsequent reads. Count your nextInt/nextDouble calls and make sure each one that is followed by nextLine has a buffer-clearing nextLine in between.

public class Solution {

    public static void findCricketerByLeagueName(List<Cricketer> cricketers, String leagueName) {
        boolean found = false;
        for (Cricketer c : cricketers) {
            for (Statistics stat : c.getStatistics()) {
                if (stat.getLeagueName().equalsIgnoreCase(leagueName)) {
                    System.out.println(c.getCricketerName());
                    found = true;
                    break;
                }
            }
        }
        if (!found) {
            System.out.println("No Cricketer Found");
        }
    }

    public static Double findTotalOfPerformanceAverageByCountryName(List<Cricketer> cricketers, String countryName) {
        double total = 0;

        for (Cricketer c : cricketers) {
            for (Statistics stat : c.getStatistics()) {
                if (stat.getCountryName().equalsIgnoreCase(countryName)) {
                    total += stat.getPerformanceAverage();
                    found = true;
                }
            }
        }
        if (!found) {
            System.out.println("No Statistics are available With the Country : " + countryName);
            return null;
        }
        return total;
    }

            sc.nextLine();

                sc.nextLine();

            cricketers.add(new Cricketer(id, name, age, gender, statsList));

Section 10

Common Mistakes

This is the hardest question type. Here are the traps.

Mistake 1: Forgetting the Nested Loop

The most common mistake is looping only through cricketers but not through their statistics. You write for (Cricketer c : cricketers) but forget to add the inner for (Statistics stat : c.getStatistics()). Without the inner loop, you cannot access league names or country names at all.

Mistake 2: Not Breaking After First Match in Method 1

If a cricketer has 3 statistics all with the same league name, without the break statement, the cricketer's name prints 3 times. The break exits the inner loop after the first match, so each cricketer's name appears at most once.

Mistake 3: Wrong Output Format for Total

The exact output format is: Total PerformanceAverage Australia : 214.0. Notice the spacing around the colon and the fact that the country name appears in the output. If you print "Total: 214.0" or "Total PerformanceAverage: 214.0", it will not match.

Fix: Copy the format string exactly: "Total PerformanceAverage " + countryName + " : " + total

Mistake 4: Forgetting Case-Insensitive Comparison

The question explicitly says "All String comparison needs to be case in-sensitive." If you use .equals() instead of .equalsIgnoreCase(), hidden test cases with mixed-case input will fail.
