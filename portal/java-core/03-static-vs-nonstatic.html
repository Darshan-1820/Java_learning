Section 01

Static Methods in PRA

In PRA, the methods you write inside the Solution class are almost always static. This is the most important keyword to understand for your exam methods.

What Does static Mean?

A static method belongs to the class itself, not to any specific object. You can call it directly without creating an object of that class.

StaticCall.java 
// Static method — call directly on the class
double avg = Solution.averageOfPositiveGains(investors);
// No need to write: Solution s = new Solution(); s.averageOfPositiveGains(...)

// Non-static method — need an object first
Investor inv = new Investor(1, "Simon", 5000);
String name = inv.getInvestorName(); // called ON the object
Analogy — Static vs Non-Static

Think of a calculator app on your phone. The "calculate" function is static — you open the app and use it directly. You do not need to "create a calculator object" first.

Now think of a contact in your phone. To call someone, you need a specific contact (object) first — you cannot just "call" without knowing WHO. That is non-static — it needs an instance.

Why Are PRA Methods Static?

In PRA, your Solution class is a utility class — it does not hold any data of its own. It takes a collection of objects, processes them, and returns a result. Think of it as a tool: the tool does not own the data, it just works on whatever you give it.

Since the Solution class has no fields, there is no reason to create an object of it. So the methods are static — you call them directly from main.

Key Insight

The main method itself is static. A static method can only directly call other static methods in the same class. This is why your PRA solution methods MUST be static — otherwise main cannot call them without creating a Solution object.

Section 02

The PRA Method Signature — Anatomy

Every word in a PRA method signature has a specific purpose. Let us break down a real example piece by piece.

public static double averageOfPositiveGains(Collection<Investor> investors)
│       │      │       │                      │
│       │      │       │                      └── Parameter: the data you work with
│       │      │       │                          (a collection of Investor objects)
│       │      │       │
│       │      │       └── Method name: describes what it does
│       │      │           (camelCase, always descriptive)
│       │      │
│       │      └── Return type: what the method gives back
│       │          (double = a decimal number)
│       │
│       └── Static: belongs to the class, not an object
│           (can call without creating Solution object)
│
└── Public: accessible from anywhere
    (main method can call this)

Reading a Method Signature Like a Sentence

You can read any PRA method signature as an English sentence:

Reading Template

public static [RETURN] [NAME]([PARAMETER])

Reads as: "This method takes [PARAMETER], does something, and gives back [RETURN]."

Examples:

  • public static double averageOfPositiveGains(Collection<Investor> investors)
    "Takes a collection of investors, and gives back a double (the average of positive gains)."
  • public static Collection<String> negativeGainsInvestorNames(Collection<Investor> investors)
    "Takes a collection of investors, and gives back a collection of strings (names of investors with negative gains)."
  • public static Double findTotalOfPerformanceAverage(Collection<Cricketer> cricketers, String countryName)
    "Takes a collection of cricketers and a country name, and gives back a Double (total performance average for that country)."
Watch Out — Double vs double

double (lowercase) is a primitive type — it can only hold a number, never null.

Double (uppercase) is a wrapper class — it can hold a number OR null.

Some PRA methods return null when no match is found. In those cases, the return type is Double (uppercase). If the question says "return null if not found", use Double.

Section 03

Return Types in PRA

PRA methods return one of a few types. Knowing what to return determines how you structure the method body.

Return Type When Used Example Method What You Do Inside
double Averages, sums, totals averageOfPositiveGains() Calculate sum and count, return sum / count
Double Aggregates that can return null findTotalOfPerformanceAverage() Same as double, but return null if no match found
Collection<T> Filtered lists of objects findBooksByGenre() Create result list, loop and filter, return list
Collection<String> Filtered lists of names/strings negativeGainsInvestorNames() Create String list, extract names from matching objects
void Print directly (rare in PRA) Uncommon — most PRA methods return something Print inside the method instead of returning

How Return Type Determines Method Structure

Returns a Number (double/Double)
double sum = 0;
int count = 0;
for (Item item : list) {
    if (condition) {
        sum += item.getValue();
        count++;
    }
}
return sum / count;
Returns a Collection
ArrayList<Item> result = new ArrayList<>();
for (Item item : list) {
    if (condition) {
        result.add(item);
    }
}
return result;
PRA Exam Tip

Look at the return type FIRST when reading the method signature. It tells you the entire structure of the method:

  • Returns double? You need sum and count variables.
  • Returns Collection? You need a new ArrayList<>() to collect results.
  • Returns null in some case? Use Double (uppercase) as return type.
Section 04

Non-Static Methods

Non-static methods are called on a specific object instance. In PRA, the most common non-static methods are the getters and setters you wrote in your POJO class.

Where Non-Static Methods Appear in PRA

You rarely write non-static methods yourself in the Solution class. But you USE them constantly:

NonStaticUsage.java 
// Inside your static method, you CALL non-static methods on objects:
for (Investor inv : investors) {
    // inv.getGains() is a NON-STATIC method
    // It is called ON the specific investor object "inv"
    if (inv.getGains() > 0) {
        sum += inv.getGains();  // non-static: called on "inv"
        count++;
    }
}

// inv.getGains()       → non-static (called on object)
// Solution.calculate() → static (called on class)
Key Distinction

In PRA, you write static methods (in Solution class) but you call non-static methods (getters on POJO objects). Understanding both is essential because they work together in every solution.

Section 05

Static vs Non-Static — Side by Side

A complete comparison to cement the difference in your mind.

Static Methods
  • Belongs to: The class itself
  • Called with: ClassName.method()
  • Needs object? No
  • Can access: Only static members, and parameters
  • PRA usage: Solution methods (filter, aggregate)
  • Example: Solution.averageOfPositiveGains(list)
  • Think of it as: A tool in a toolbox — the tool does not own data
Non-Static Methods
  • Belongs to: A specific object instance
  • Called with: object.method()
  • Needs object? Yes, must create with new
  • Can access: Object's own fields (this.field)
  • PRA usage: Getters, setters on POJO class
  • Example: investor.getGains()
  • Think of it as: A person's skill — the person (object) uses it

Quick Memory Rule

Static  → "Shared" — one copy, everyone uses it, no object needed
          Example: Math.sqrt(16) — you don't create a Math object

Non-static → "Personal" — each object has its own copy
             Example: inv.getName() — each investor has their own name
Section 06

PRA Method Templates — Copy-Paste Ready

These two templates cover approximately 80% of all PRA methods. Memorize them and adapt them to any question.

Template 1: Filter Method

Use when the question says: "find all items that match...", "return items where...", "get a list of..."

FilterTemplate.java 
// FILTER TEMPLATE — returns matching items from a collection
public static Collection<ClassName> filterByCondition(
        Collection<ClassName> list, Type filterValue) {

    // Step 1: Create empty result list
    ArrayList<ClassName> result = new ArrayList<>();

    // Step 2: Loop through every item
    for (ClassName item : list) {

        // Step 3: Check if this item matches the condition
        // For String comparison, ALWAYS use equalsIgnoreCase
        if (item.getField().equalsIgnoreCase(filterValue)) {
            // Step 4: Add matching item to result
            result.add(item);
        }
    }

    // Step 5: Return the filtered list
    return result;
}
When to use what comparison
1
item.getField().equalsIgnoreCase(filterValue)
Use for String comparisons. Handles "Fiction" == "fiction" == "FICTION".
2
item.getValue() > threshold
Use for numeric comparisons (int, double). Works with >, <, >=, <=, ==.
3
item.getField().contains(searchTerm)
Use when checking if a string CONTAINS another string (partial match).

Variation: Return Names Instead of Objects

Sometimes PRA asks for a list of names (strings) instead of objects:

FilterNamesTemplate.java 
// Returns names (strings) of matching items
public static Collection<String> getMatchingNames(
        Collection<ClassName> list) {

    ArrayList<String> names = new ArrayList<>();
    for (ClassName item : list) {
        if (item.getValue() < 0) { // condition
            names.add(item.getName()); // add the NAME, not the object
        }
    }
    return names;
}

Template 2: Aggregate Method

Use when the question says: "calculate the average...", "find the sum of...", "count the number of..."

AggregateTemplate.java 
// AGGREGATE TEMPLATE — calculates a value from a collection
public static double calculateAggregate(Collection<ClassName> list) {

    // Step 1: Initialize accumulators
    double sum = 0;
    int count = 0;

    // Step 2: Loop through every item
    for (ClassName item : list) {

        // Step 3: Check condition (if any)
        if (item.getValue() > 0) {
            // Step 4: Accumulate
            sum += item.getValue();
            count++;
        }
    }

    // Step 5: Calculate and return (WATCH for division by zero!)
    return count > 0 ? sum / count : 0;
}

Variation: Return null If No Match (uses Double wrapper)

AggregateNullTemplate.java 
// Returns null if nothing matches — note the uppercase Double
public static Double findTotal(Collection<ClassName> list, String searchValue) {

    double total = 0;
    boolean found = false;

    for (ClassName item : list) {
        if (item.getField().equalsIgnoreCase(searchValue)) {
            total += item.getValue();
            found = true;
        }
    }

    // Return null if nothing matched, otherwise return the total
    return found ? total : null;
}
PRA Exam Strategy

When you read a PRA method signature, immediately identify which template to use:

  • Returns Collection<...>? Use the Filter Template.
  • Returns double or Double? Use the Aggregate Template.

Then just fill in the condition and the field names from the question. The structure never changes.

Real PRA Method Examples

Here is how the templates map to actual PRA methods:

PRA Method Template Condition
negativeGainsInvestorNames() Filter (names) gains < 0
averageOfPositiveGains() Aggregate (average) gains > 0
findBooksByGenre() Filter (objects) genre.equalsIgnoreCase(input)
findTotalOfPerformanceAverage() Aggregate (total, nullable) countryName.equalsIgnoreCase(input)
Practice Exercise

Without looking at the templates above, write a static method:

public static Collection<Student> findToppers(Collection<Student> students, double threshold)

It should return all students whose marks are above the threshold.

Then write another:

public static double averageMarks(Collection<Student> students, String department)

It should return the average marks of students in the given department.

Both use the templates above — just change the field names and conditions.
