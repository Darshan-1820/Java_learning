1 Type Promotion in Arithmetic Easy

What will this print?

TypePromotion.java 
public class TypePromotion {
    public static void main(String[] args) {
        byte a = 10;
        byte b = 20;
        // byte c = a + b;  // Would this compile?
        int c = a + b;
        System.out.println(c);

        int x = 7;
        int y = 2;
        System.out.println(x / y);
        System.out.println((double) x / y);
    }
}
Answer & Explanation
1
30
byte + byte promotes to int. So a + b = 30 as an int. The commented line would NOT compile — you can't assign an int result back to byte without casting.
2
3
int / int = int. Integer division truncates — 7 / 2 = 3, not 3.5.
3
3.5
(double) x casts x to 7.0 first. Then 7.0 / 2 promotes 2 to double → 3.5.
2 String Pool & Reference Comparison Medium

What will this print?

StringCompare.java 
public class StringCompare {
    public static void main(String[] args) {
        String a = "Nimbus";
        String b = "Nimbus";
        String c = new String("Nimbus");

        System.out.println(a == b);
        System.out.println(a == c);
        System.out.println(a.equals(c));
    }
}
Answer & Explanation
1
true
Both a and b are string literals — Java puts them in the String Pool. Same literal = same reference. == checks reference, so true.
2
false
new String() forces a new object on the heap, bypassing the pool. Different reference → == returns false.
3
true
.equals() compares content. Both contain "Nimbus" → true. Always use .equals() for String comparison.

public class CastingDemo {

4 Constructor Chaining Medium

What will this print?

ConstructorChain.java 
class Vehicle {
    Vehicle() {
        System.out.println("Vehicle created");
    }
}

class Car extends Vehicle {
    Car() {
        System.out.println("Car created");
    }
}

class ElectricCar extends Car {
    ElectricCar() {
        System.out.println("ElectricCar created");
    }
}

public class ConstructorChain {
    public static void main(String[] args) {
        ElectricCar tesla = new ElectricCar();
    }
}
Answer & Explanation
1
Vehicle created
Constructors chain upward. ElectricCar() calls Car() which calls Vehicle(). Parent constructors execute first — top-down. Java inserts an invisible super() as the first line of every constructor.
2
Car created
After Vehicle's constructor finishes, Car's constructor body runs.
3
ElectricCar created
Finally, ElectricCar's own constructor body runs last.
5 Polymorphism — Which Method Runs? Hard

What will this print?

PolyDispatch.java 
class Parcel {
    String getType() { return "Generic Parcel"; }
}

class ExpressParcel extends Parcel {
    String getType() { return "Express Parcel"; }
    String getSpeed() { return "Fast"; }
}

public class PolyDispatch {
    public static void main(String[] args) {
        Parcel p = new ExpressParcel();
        System.out.println(p.getType());
        // System.out.println(p.getSpeed()); // Would this compile?
    }
}
Answer & Explanation
1
Express Parcel
Reference type is Parcel, but actual object is ExpressParcel. Method dispatch uses the actual object type at runtime (dynamic binding). The overridden version in ExpressParcel runs.
2
p.getSpeed() — WON'T COMPILE
The compiler sees p as type Parcel. Parcel has no getSpeed() method. You'd need to cast: ((ExpressParcel) p).getSpeed(). Rule: reference type controls what you CAN call; object type controls what RUNS.
6 Exception Flow — What Executes? Medium

What will this print?

ExceptionFlow.java 
public class ExceptionFlow {
    public static void main(String[] args) {
        System.out.println("Start");

            int result = 10 / 0;
            System.out.println("After division");

Start

Try block

7 Collection Iteration & Modification Hard

What will this print?

CollectionIter.java 
import java.util.*;

public class CollectionIter {
    public static void main(String[] args) {
        List<String> statuses = new ArrayList<>();
        statuses.add("BOOKED");
        statuses.add("IN_TRANSIT");
        statuses.add("DELIVERED");

        for (String s : statuses) {
            if (s.equals("IN_TRANSIT")) {
                statuses.remove(s);
            }
        }
        System.out.println(statuses);
    }
}
Answer & Explanation
1
ConcurrentModificationException!
You CANNOT modify a collection while iterating it with a for-each loop. The for-each loop uses an Iterator internally, and statuses.remove() modifies the list behind the iterator's back. Fix: use Iterator.remove() or statuses.removeIf(s -> s.equals("IN_TRANSIT")).
8 Thread Interleaving Hard

What will this print? (Trick question)

ThreadInterleave.java 
public class ThreadInterleave {
    public static void main(String[] args) {
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 3; i++) System.out.print("A");
        });
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 3; i++) System.out.print("B");

    }
}
Answer & Explanation
1
UNPREDICTABLE — could be AAABBB, BBBAAA, ABABAB, AABBBA, etc.

10 Post-increment vs Pre-increment Medium

What will this print?

IncrementDemo.java 
public class IncrementDemo {
    public static void main(String[] args) {
        int a = 5;
        int b = a++;
        System.out.println("a=" + a + " b=" + b);

        int c = 5;
        int d = ++c;
        System.out.println("c=" + c + " d=" + d);

        int x = 10;
        System.out.println(x++ + " " + x);
    }
}
Answer & Explanation
1
a=6 b=5
a++ is post-increment: use current value (5) FIRST, then increment. So b gets 5, then a becomes 6.
2
c=6 d=6
++c is pre-increment: increment FIRST, then use new value. So c becomes 6, d gets 6.
3
10 11
x++ evaluates to 10 (old value), then x becomes 11. The second x in the concatenation sees the already-incremented value 11.
11 Autoboxing & Integer Cache Hard

What will this print?

AutoboxTrap.java 
public class AutoboxTrap {
    public static void main(String[] args) {
        Integer a = 100;
        Integer b = 100;
        System.out.println(a == b);

        Integer c = 200;
        Integer d = 200;
        System.out.println(c == d);

        System.out.println(c.equals(d));
    }
}
Answer & Explanation
1
true
Java caches Integer objects for values -128 to 127. Both a and b point to the same cached object. == checks reference → same object → true.
2
false
200 is outside the cache range. Two separate Integer objects are created. Different references → == returns false. This is the classic autoboxing trap.
3
true
.equals() compares values, not references. Always use .equals() for wrapper types.

13 Finally vs Return Hard

What will this method return?

FinallyReturn.java 
public class FinallyReturn {
    static int getValue() {
        try {
            return 1;
        } catch (Exception e) {
            return 2;
        } finally {
            return 3;
        }
    }

    public static void main(String[] args) {
        System.out.println(getValue());
    }
}
Answer & Explanation
1
3
Finally block ALWAYS executes, and a return in finally overrides any return in try or catch. The try says "return 1" but before that return completes, finally runs and says "return 3". The 3 wins. This is generally considered bad practice — never put return in a finally block.

1 NullPointerException Easy

This code crashes at runtime. Why?

NullBug.java 
public class NullBug {
    public static void main(String[] args) {
        String trackingId = getTrackingId(999);
        System.out.println(trackingId.toUpperCase());
    }

    static String getTrackingId(int bookingId) {
        if (bookingId < 100) {
            return "NMB-" + bookingId;
        }
        return null;
    }
}
Bug & Fix
🐛
Bug: getTrackingId(999) returns null
999 > 100, so the if-block is skipped and null is returned. Calling .toUpperCase() on null → NullPointerException.
✅
Fix: Always null-check before calling methods on returned values
if (trackingId != null) { System.out.println(trackingId.toUpperCase()); } or make the method never return null by throwing an exception or returning a default value.

3 == Instead of .equals() Easy

This validation always fails even when the status is correct. Why?

EqualsBug.java 
public class EqualsBug {
    public static void main(String[] args) {
        String userInput = new Scanner(System.in).nextLine(); // User types "DELIVERED"
        String expected = "DELIVERED";

            System.out.println("Status matched!");
        } else {

Bug & Fix

Bug: == compares references, not content

4 Scanner Newline Trap Medium

The program skips the name input entirely. User enters "25" for age but never gets to type their name. Why?

ScannerBug.java 
import java.util.Scanner;

public class ScannerBug {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter age: ");
        int age = sc.nextInt();

        System.out.print("Enter name: ");
        String name = sc.nextLine();

        System.out.println("Name: " + name + ", Age: " + age);
    }
}
Bug & Fix
🐛
Bug: nextInt() leaves the newline character in the buffer
When user types "25\n", nextInt() reads "25" but leaves the \n in the buffer. nextLine() then immediately reads that leftover \n as an empty string.
✅
Fix: Add sc.nextLine(); after sc.nextInt() to consume the leftover newline
int age = sc.nextInt(); then sc.nextLine(); // consume newline then String name = sc.nextLine(); — this is the classic Scanner trap in Java.

6 ArrayIndexOutOfBounds Easy

This code crashes on the last iteration. Why?

ArrayBug.java 
public class ArrayBug {
    public static void main(String[] args) {
        String[] cities = {"Mumbai", "Delhi", "Bangalore", "Chennai"};

        for (int i = 0; i <= cities.length; i++) {
            System.out.println(cities[i]);
        }
    }
}
Bug & Fix
🐛
Bug: i <= cities.length goes one index too far
Array length is 4, valid indices are 0-3. <= means i reaches 4 → cities[4] → ArrayIndexOutOfBoundsException.
✅
Fix: Use i < cities.length (strict less-than)
This is the #1 beginner bug. Arrays are 0-indexed. Length 4 = indices 0,1,2,3. Always use < not <=.
7 SQL Injection Vulnerability Hard

This code "works" but has a critical security vulnerability. What is it?

SqlInjection.java 
public List<Booking> searchBookings(String userInput) {
    String sql = "SELECT * FROM bookings WHERE tracking_id = '" + userInput + "'";
    Statement stmt = connection.createStatement();
    ResultSet rs = stmt.executeQuery(sql);
    // ... process results
}
Bug & Fix
🐛
Bug: String concatenation in SQL = SQL Injection
If user enters ' OR '1'='1, the query becomes SELECT * FROM bookings WHERE tracking_id = '' OR '1'='1' — returns ALL bookings. Worse: '; DROP TABLE bookings; -- could destroy data.

8 Unclosed Connection (Resource Leak) Medium

This works fine for 10 users. With 100 users, the app crashes with "Too many connections". Why?

ConnectionLeak.java 
public Booking getBooking(int id) throws SQLException {
    Connection conn = DriverManager.getConnection(URL, USER, PASS);
    PreparedStatement ps = conn.prepareStatement("SELECT * FROM bookings WHERE id = ?");
    ps.setInt(1, id);
    ResultSet rs = ps.executeQuery();

    if (rs.next()) {
        return new Booking(rs.getString("tracking_id"), rs.getDouble("cost"));
    }
    return null;
    // Where's conn.close()? rs.close()? ps.close()?
}
Bug & Fix
🐛
Bug: Connection, PreparedStatement, ResultSet are never closed
Each call opens a new connection and never releases it. Databases have a connection limit (typically 100-150). After enough calls, no more connections can be opened → crash.
✅
Fix: Use try-with-resources to auto-close all resources
try (Connection conn = ...; PreparedStatement ps = ...; ResultSet rs = ...) { ... } — all three implement AutoCloseable and will be closed automatically, even if an exception occurs.

10 Missing @Override — Silent Bug Hard

Two Users with the same email should be "equal", but HashSet stores both as separate entries. Why?

OverrideBug.java 
import java.util.*;

class User {
    String email;
    User(String email) { this.email = email; }

    public boolean equals(User other) {  // Spot the bug!
        return this.email.equals(other.email);
    }
}

public class OverrideBug {
    public static void main(String[] args) {
        Set<User> users = new HashSet<>();
        users.add(new User("darshan@nimbus.com"));
        users.add(new User("darshan@nimbus.com"));

        System.out.println(users.size()); // Expected 1, gets 2!
    }
}
Bug & Fix
🐛
Bug: equals(User) is NOT overriding Object.equals(Object)
The parameter type is User instead of Object. This creates a new overloaded method, not an override. HashSet calls equals(Object) — which is still the default reference comparison from Object. Also missing: hashCode() — must override both.
✅
Fix: Use @Override with correct signature + implement hashCode()
@Override public boolean equals(Object obj) with an instanceof check. If you'd used @Override, the compiler would have caught the wrong parameter type immediately. Always use @Override annotation.

1 Nimbus Password Validator Medium

Write a method boolean isValidPassword(String password) that enforces Nimbus password rules:

  • 8 to 30 characters long
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one special character (!@#$%^&*)
  • No three sequential characters (abc, 123, etc.)
Solution
✓
PasswordValidator.java 
public static boolean isValidPassword(String password) {
    if (password == null) return false;
    if (password.length() < 8 || password.length() > 30) return false;

    boolean hasUpper = false, hasLower = false, hasSpecial = false;
    String specials = "!@#$%^&*";

    for (char c : password.toCharArray()) {
        if (Character.isUpperCase(c)) hasUpper = true;
        if (Character.isLowerCase(c)) hasLower = true;
        if (specials.indexOf(c) != -1) hasSpecial = true;
    }

    if (!hasUpper || !hasLower || !hasSpecial) return false;

    // Check for 3 sequential characters
    for (int i = 0; i < password.length() - 2; i++) {
        char c1 = password.charAt(i);
        char c2 = password.charAt(i + 1);
        char c3 = password.charAt(i + 2);
        if (c2 == c1 + 1 && c3 == c2 + 1) return false;
    }

    return true;
}
2 User.equals() and hashCode() Medium

@Override
public boolean equals(Object obj) {
    if (this == obj) return true;              // Same reference
    if (obj == null) return false;             // Null check

    User other = (User) obj;                   // Safe cast

@Override
public int hashCode() {
    return email.hashCode();  // Must match equals logic
    // If equals uses email, hashCode MUST use email
}

// Method 1: Lambda (cleanest)

        }
    });

public static Map<String, Integer> countByStatus(List<String> statuses) {
    Map<String, Integer> counts = new HashMap<>();

    for (String status : statuses) {
        // getOrDefault returns 0 if key doesn't exist yet
        counts.put(status, counts.getOrDefault(status, 0) + 1);
    }

    return counts;
}

public static Map<String, Integer> countByStatus2(List<String> statuses) {
    Map<String, Integer> counts = new HashMap<>();
    for (String status : statuses) {
        counts.merge(status, 1, Integer::sum);
    }
    return counts;

10 Booking ID Generator (NMB-YYMMDD-NNNN) Medium

Write a method String generateBookingId(int sequenceNumber) that generates IDs in the format NMB-260423-0001 where:

  • NMB is the prefix
  • 260423 is today's date in YYMMDD
  • 0001 is the sequence number zero-padded to 4 digits
Solution
✓
BookingIdGenerator.java 
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;

public class BookingIdGenerator {

    public static String generateBookingId(int sequenceNumber) {
        // Get today's date in YYMMDD format
        LocalDate today = LocalDate.now();
        String dateStr = today.format(DateTimeFormatter.ofPattern("yyMMdd"));

        // Zero-pad sequence number to 4 digits
        String seqStr = String.format("%04d", sequenceNumber);

        return "NMB-" + dateStr + "-" + seqStr;
    }

    public static void main(String[] args) {
        System.out.println(generateBookingId(1));    // NMB-260423-0001
        System.out.println(generateBookingId(42));   // NMB-260423-0042
        System.out.println(generateBookingId(9999)); // NMB-260423-9999
    }
}

1 Broken Parcel Cost Calculator 4 bugs

This program should calculate parcel shipping cost based on weight and type. Find all 4 bugs.

CostCalculator.java

            case "STANDARD":
                rate = 10.0;
            case "EXPRESS":
                rate = 25.0;

1
Missing break after STANDARD case

2
Variable rate may not be initialized
