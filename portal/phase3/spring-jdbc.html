Section 01

Problems with Raw JDBC

Raw JDBC works but forces you to write the same boilerplate for every single query. Spring JDBC eliminates that.

Raw JDBC — Look at all this boilerplate 
public User findById(int id) {
    Connection conn = null;
    PreparedStatement ps = null;
    ResultSet rs = null;
    try {
        conn = DriverManager.getConnection(url, user, pass);  // boilerplate
        ps = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
        ps.setInt(1, id);
        rs = ps.executeQuery();
        if (rs.next()) {
            User u = new User();
            u.setId(rs.getInt("id"));           // manual mapping
            u.setName(rs.getString("name"));     // manual mapping
            u.setEmail(rs.getString("email"));   // manual mapping
            return u;
        }
        return null;
    } catch (SQLException e) {                   // boilerplate
        throw new RuntimeException(e);
    } finally {
        try { if (rs != null) rs.close(); } catch (SQLException e) {}   // boilerplate
        try { if (ps != null) ps.close(); } catch (SQLException e) {}   // boilerplate
        try { if (conn != null) conn.close(); } catch (SQLException e) {} // boilerplate
    }
}

Every query has the same pattern: get connection, prepare statement, execute, map results, handle exceptions, close everything. Spring JDBC removes all of it.

ProblemRaw JDBCSpring JDBC
Connection managementManual open/closeAutomatic via DataSource
Exception handlingChecked SQLException everywhereUnchecked DataAccessException
ResultSet mappingManual rs.getString() for each columnRowMapper or auto-map
Resource cleanuptry-finally for every resourceAutomatic
Lines of code~25 per query~3 per query
Section 02

JdbcTemplate

The core class of Spring JDBC. Handles connection, execution, mapping, and cleanup in one call.

query() — Multiple Rows

Get all users 
List<User> users = jdbcTemplate.query(
    "SELECT * FROM users WHERE role = ?",
    new UserRowMapper(),   // maps each row to User object
    "ADMIN"                // parameter value for ?
);

queryForObject() — Single Row

Get one user by ID 
User user = jdbcTemplate.queryForObject(
    "SELECT * FROM users WHERE id = ?",
    new UserRowMapper(),
    id
);

// For simple values (count, max, etc.)
int count = jdbcTemplate.queryForObject(
    "SELECT COUNT(*) FROM bookings WHERE user_id = ?",
    Integer.class,
    userId
);

update() — INSERT, UPDATE, DELETE

Modify data 
// INSERT
int rowsInserted = jdbcTemplate.update(
    "INSERT INTO users (name, email, role) VALUES (?, ?, ?)",
    user.getName(), user.getEmail(), user.getRole()
);

// UPDATE
int rowsUpdated = jdbcTemplate.update(
    "UPDATE bookings SET status = ? WHERE tracking_number = ?",
    "DELIVERED", trackingNumber
);

// DELETE
int rowsDeleted = jdbcTemplate.update(
    "DELETE FROM bookings WHERE status = ? AND created_at < ?",
    "CANCELLED", cutoffDate
);
Section 03

RowMapper & BeanPropertyRowMapper

RowMapper tells Spring how to convert a ResultSet row into a Java object.

Custom RowMapper

UserRowMapper.java 
public class UserRowMapper implements RowMapper<User> {

    @Override
    public User mapRow(ResultSet rs, int rowNum) throws SQLException {
        User user = new User();
        user.setId(rs.getInt("id"));
        user.setName(rs.getString("name"));
        user.setEmail(rs.getString("email"));
        user.setRole(rs.getString("role"));
        user.setCreatedAt(rs.getTimestamp("created_at").toLocalDateTime());
        return user;
    }
}

BeanPropertyRowMapper (Zero Code)

If your column names match your field names (or use underscore-to-camelCase), Spring maps automatically:

Auto-mapping 
// Column "created_at" maps to field "createdAt" automatically
List<User> users = jdbcTemplate.query(
    "SELECT * FROM users",
    new BeanPropertyRowMapper<>(User.class)
);

// Works when:
// DB column: user_name  →  Java field: userName
// DB column: id         →  Java field: id
// DB column: email      →  Java field: email
When to Use Which

BeanPropertyRowMapper: Quick prototyping, columns match fields. Custom RowMapper: Complex mappings, computed fields, JOINs that span multiple objects, or when you need max control.

Section 04

DataSource Configuration

DataSource is the connection factory. Configure it once, inject everywhere.

DataSourceConfig.java 
@Configuration
@PropertySource("classpath:application.properties")
public class DataSourceConfig {

    @Value("${db.url}")
    private String url;

    @Value("${db.user}")
    private String username;

    @Value("${db.password}")
    private String password;

    @Bean
    public DataSource dataSource() {
        DriverManagerDataSource ds = new DriverManagerDataSource();
        ds.setDriverClassName("com.mysql.cj.jdbc.Driver");
        ds.setUrl(url);
        ds.setUsername(username);
        ds.setPassword(password);
        return ds;
    }

    @Bean
    public JdbcTemplate jdbcTemplate(DataSource dataSource) {
        return new JdbcTemplate(dataSource);
    }
}
application.properties 
db.url=jdbc:mysql://localhost:3306/nimbus_db?useSSL=false&serverTimezone=UTC
db.user=root
db.password=secret
In Nimbus

The DataSource bean is configured once in DataSourceConfig. Spring injects it into JdbcTemplate, which is then injected into all DAOs. No DAO ever touches DriverManager directly.

Section 05

NamedParameterJdbcTemplate

Use named parameters (:name) instead of positional ? marks. Much more readable for complex queries.

JdbcTemplate (positional ?)
jdbc.query(
  "SELECT * FROM bookings " +
  "WHERE status = ? " +
  "AND user_id = ? " +
  "AND weight > ?",
  mapper, status, userId, minWeight
);

Which ? is which? Count carefully.

NamedParameter (named)
namedJdbc.query(
  "SELECT * FROM bookings " +
  "WHERE status = :status " +
  "AND user_id = :userId " +
  "AND weight > :minWeight",
  params, mapper
);

Crystal clear what each value maps to.

NamedParameterJdbcTemplate Usage 
@Repository
public class BookingDAOImpl implements BookingDAO {

    private final NamedParameterJdbcTemplate namedJdbc;

    public BookingDAOImpl(DataSource dataSource) {
        this.namedJdbc = new NamedParameterJdbcTemplate(dataSource);
    }

    public List<Booking> search(String status, int userId, double minWeight) {
        String sql = "SELECT * FROM bookings " +
                     "WHERE status = :status " +
                     "AND user_id = :userId " +
                     "AND weight > :minWeight";

        MapSqlParameterSource params = new MapSqlParameterSource()
            .addValue("status", status)
            .addValue("userId", userId)
            .addValue("minWeight", minWeight);

        return namedJdbc.query(sql, params, new BookingRowMapper());
    }
}

Nimbus UserDAO: Before & After

The same DAO, raw JDBC vs Spring JDBC. See the difference.

In Nimbus

The raw JDBC version of UserDAO was 80+ lines for 3 methods. Spring JDBC brings it down to ~30 lines with zero boilerplate. Same functionality, fraction of the code.

Raw JDBC (~25 lines per method)
public User findById(int id) {
  Connection conn = null;
  PreparedStatement ps = null;
  ResultSet rs = null;
  try {
    conn = DBUtil.getConnection();
    ps = conn.prepareStatement(
      "SELECT * FROM users WHERE id=?");
    ps.setInt(1, id);
    rs = ps.executeQuery();
    if (rs.next()) {
      User u = new User();
      u.setId(rs.getInt("id"));
      u.setName(rs.getString("name"));
      u.setEmail(rs.getString("email"));
      return u;
    }
    return null;
  } catch (SQLException e) {
    throw new RuntimeException(e);
  } finally {
    DBUtil.close(rs, ps, conn);
  }
}
Spring JDBC (~3 lines per method)
public User findById(int id) {
  return jdbcTemplate.queryForObject(
    "SELECT * FROM users WHERE id = ?",
    new UserRowMapper(),
    id
  );
}

// Connection? Automatic.
// PreparedStatement? Automatic.
// ResultSet closing? Automatic.
// Exception handling? Automatic.
// That's it. Done.

Complete Spring JDBC UserDAO

com/nimbus/dao/UserDAOImpl.java 
@Repository
public class UserDAOImpl implements UserDAO {

    private final JdbcTemplate jdbc;
    private final RowMapper<User> rowMapper = new UserRowMapper();

    public UserDAOImpl(JdbcTemplate jdbc) {
        this.jdbc = jdbc;
    }

    @Override
    public User findById(int id) {
        return jdbc.queryForObject(
            "SELECT * FROM users WHERE id = ?", rowMapper, id);
    }

    @Override
    public User findByEmail(String email) {
        return jdbc.queryForObject(
            "SELECT * FROM users WHERE email = ?", rowMapper, email);
    }

    @Override
    public List<User> findAll() {
        return jdbc.query("SELECT * FROM users ORDER BY name", rowMapper);
    }

    @Override
    public void save(User user) {
        jdbc.update(
            "INSERT INTO users (name, email, role) VALUES (?, ?, ?)",
            user.getName(), user.getEmail(), user.getRole());
    }

    @Override
    public void update(User user) {
        jdbc.update(
            "UPDATE users SET name = ?, email = ?, role = ? WHERE id = ?",
            user.getName(), user.getEmail(), user.getRole(), user.getId());
    }

    @Override
    public void deleteById(int id) {
        jdbc.update("DELETE FROM users WHERE id = ?", id);
    }
}

BookingDAO with JdbcTemplate

Full BookingDAO for the Nimbus parcel system.

com/nimbus/dao/BookingDAOImpl.java 
@Repository
public class BookingDAOImpl implements BookingDAO {

    private final JdbcTemplate jdbc;

    public BookingDAOImpl(JdbcTemplate jdbc) {
        this.jdbc = jdbc;
    }

    @Override
    public Booking findById(int id) {
        return jdbc.queryForObject(
            "SELECT * FROM bookings WHERE id = ?",
            new BookingRowMapper(), id);
    }

    @Override
    public Booking findByTrackingNumber(String trackingNumber) {
        return jdbc.queryForObject(
            "SELECT * FROM bookings WHERE tracking_number = ?",
            new BookingRowMapper(), trackingNumber);
    }

    @Override
    public List<Booking> findByUserId(int userId) {
        return jdbc.query(
            "SELECT * FROM bookings WHERE user_id = ? ORDER BY created_at DESC",
            new BookingRowMapper(), userId);
    }

    @Override
    public List<Booking> findByStatus(String status) {
        return jdbc.query(
            "SELECT * FROM bookings WHERE status = ?",
            new BookingRowMapper(), status);
    }

    @Override
    public void save(Booking booking) {
        jdbc.update(
            "INSERT INTO bookings (user_id, tracking_number, status, weight) " +
            "VALUES (?, ?, ?, ?)",
            booking.getUserId(), booking.getTrackingNumber(),
            booking.getStatus(), booking.getWeight());
    }

    @Override
    public void updateStatus(String trackingNumber, String status) {
        jdbc.update(
            "UPDATE bookings SET status = ? WHERE tracking_number = ?",
            status, trackingNumber);
    }

    @Override
    public int countByUserId(int userId) {
        return jdbc.queryForObject(
            "SELECT COUNT(*) FROM bookings WHERE user_id = ?",
            Integer.class, userId);
    }
}

Section 08

@Transactional

When multiple DB operations must succeed or fail together. All-or-nothing.

BookingService.java 
@Service
public class BookingService {

    private final BookingDAO bookingDAO;
    private final TrackingService trackingService;

    public BookingService(BookingDAO bookingDAO, TrackingService trackingService) {
        this.bookingDAO = bookingDAO;
        this.trackingService = trackingService;
    }

    @Transactional  // if ANY exception, ALL changes roll back
    public Booking createBooking(int userId, double weight) {
        // Step 1: Generate tracking number
        String trackingNum = trackingService.generateTrackingNumber();

        // Step 2: Create booking
        Booking booking = new Booking();
        booking.setUserId(userId);
        booking.setTrackingNumber(trackingNum);
        booking.setStatus("PENDING");
        booking.setWeight(weight);
        bookingDAO.save(booking);

        // Step 3: Update user's booking count
        // If this fails, Step 2 also rolls back!
        bookingDAO.incrementUserBookingCount(userId);

        return booking;
    }

    @Transactional
    public void cancelBooking(String trackingNumber) {
        Booking booking = bookingDAO.findByTrackingNumber(trackingNumber);
        if (!"PENDING".equals(booking.getStatus())) {
            throw new IllegalStateException("Can only cancel PENDING bookings");
        }
        bookingDAO.updateStatus(trackingNumber, "CANCELLED");
        bookingDAO.decrementUserBookingCount(booking.getUserId());
    }
}

How @Transactional Works

Method called
→
BEGIN TRANSACTION
→
Execute method body
↓
Success? COMMIT
  or  
Exception? ROLLBACK
Spring wraps the method in a proxy that manages the transaction.
@Transactional Gotchas
  • Only works on public methods (Spring proxies can't intercept private/protected).
  • By default, rolls back on unchecked exceptions (RuntimeException). For checked exceptions, use @Transactional(rollbackFor = Exception.class).
  • Self-invocation doesn't work: if method A calls method B in the same class, B's @Transactional is ignored because the proxy isn't involved.
In Nimbus

BookingService.createBooking() is transactional. If generating the tracking number succeeds but saving the booking fails, the tracking number allocation also rolls back. No orphaned tracking numbers.

Section 09

Exercises

Convert raw JDBC to Spring JDBC and practice transactions.

Exercise 1 — Convert UserDAO

Take any raw JDBC DAO you've written and convert it to use JdbcTemplate. Compare the line counts. It should be at least 50% fewer lines.

Exercise 2 — Custom RowMapper

Create a BookingRowMapper that maps a JOIN query result: SELECT b.*, u.name as user_name FROM bookings b JOIN users u ON b.user_id = u.id. The Booking object should have a userName field populated from the join.

Exercise 3 — NamedParameterJdbcTemplate Search

Build a searchBookings() method that accepts optional filters: status, minWeight, maxWeight, userId. Use NamedParameterJdbcTemplate and dynamically build the WHERE clause based on which parameters are provided.

Exercise 4 — @Transactional Transfer

Create a transferBooking(String trackingNumber, int fromUserId, int toUserId) method that moves a booking from one user to another. It must update the booking's user_id AND decrement the old user's count AND increment the new user's count. Wrap it in @Transactional.

Exercise 5 — BeanPropertyRowMapper

Create a new ParcelEvent table (id, booking_id, event_type, description, event_time). Write a DAO using BeanPropertyRowMapper instead of a custom RowMapper. Verify that underscore column names auto-map to camelCase fields.
