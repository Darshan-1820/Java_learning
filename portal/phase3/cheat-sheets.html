JSP Directives, EL & JSTL

Complete quick reference for JSP syntax.

JSP Directives

DirectiveExamplePurpose
page<%@ page contentType="text/html" %>Page settings (content type, imports, error page)
page import<%@ page import="java.util.List" %>Import Java classes
page errorPage<%@ page errorPage="error.jsp" %>Redirect to error page on exception
page isErrorPage<%@ page isErrorPage="true" %>Makes ${exception} available
taglib<%@ taglib prefix="c" uri="...jstl/core" %>Import JSTL tag library
include<%@ include file="header.jsp" %>Static include (compile-time)

Expression Language (EL)

ExpressionMeaning
${user.name}Calls user.getName()
${parcel.price * 1.18}Math expression (price + 18% GST)
${empty list}True if list is null or empty
${not empty user}True if user is not null
${status == 'DELIVERED' ? 'green' : 'red'}Ternary operator
${param.username}Same as request.getParameter("username")
${sessionScope.loggedInUser}Explicitly access session scope
${requestScope.error}Explicitly access request scope
${pageContext.request.contextPath}Application context path (e.g., /nimbus)
EL Scope Search Order

${name} searches: pageScope → requestScope → sessionScope → applicationScope. First match wins. Use explicit scope (e.g., ${sessionScope.name}) when attributes exist in multiple scopes.

JSTL Core Tags

Import JSTL first 
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
TagExamplePurpose
<c:if><c:if test="${user != null}">...</c:if>Conditional rendering (no else)
<c:choose> <c:choose><c:when test="...">...</c:when><c:otherwise>...</c:otherwise></c:choose> If/else-if/else
<c:forEach><c:forEach var="p" items="${parcels}">...</c:forEach>Loop over collections
<c:forEach> (range)<c:forEach begin="1" end="10" var="i">...</c:forEach>Counter loop
<c:set><c:set var="tax" value="${price * 0.18}" />Create a variable
<c:out><c:out value="${user.name}" default="Guest" />Output with XSS escaping
<c:url><c:url value="/booking/new" />Build URL with context path
<c:redirect><c:redirect url="/login" />Redirect from JSP

JSTL Format Tags

Import format taglib 
<%@ taglib prefix="fmt" uri="http://java.sun.com/jsp/jstl/fmt" %>

<!-- Format date -->
<fmt:formatDate value="${parcel.bookingDate}"
                pattern="dd-MMM-yyyy" />

<!-- Format number -->
<fmt:formatNumber value="${parcel.price}"
                  type="currency" currencyCode="INR" />

Spring Annotations

Every annotation used in Phase 3, grouped by purpose.

Configuration

AnnotationPurpose
@ConfigurationMarks class as Spring configuration (replaces XML)
@EnableWebMvcEnables Spring MVC (registers handlers, converters)
@ComponentScan("pkg")Tells Spring to scan package for beans
@BeanMethod return value becomes a Spring-managed bean

Stereotypes (Bean Registration)

AnnotationLayerPurpose
@ComponentAnyGeneric Spring-managed bean
@ControllerWebHandles HTTP requests, returns view names
@ServiceBusinessBusiness logic layer
@RepositoryDataData access layer (translates DB exceptions)

Dependency Injection

AnnotationPurpose
@AutowiredInject a dependency (field, constructor, or setter)
@Qualifier("name")Specify which bean when multiple exist
@PrimaryMake this the default bean for its type
@Value("${prop}")Inject a property value

Request Mapping

AnnotationHTTP MethodExample
@RequestMapping("/path")All methodsClass-level base path
@GetMapping("/path")GETShow a page or fetch data
@PostMapping("/path")POSTSubmit a form or create data
@PutMapping("/path")PUTUpdate existing data
@DeleteMapping("/path")DELETEDelete data

Parameter Binding

AnnotationSourceExample
@RequestParamQuery/form param@RequestParam String city ← ?city=Mumbai
@PathVariableURL path@PathVariable int id ← /parcel/{id}
@ModelAttributeForm fields → object@ModelAttribute Parcel p ← all form fields auto-mapped
@RequestBodyJSON body@RequestBody Parcel p ← JSON parsed to object

Error Handling

AnnotationPurpose
@ExceptionHandler(Exception.class)Handle specific exception in controller
@ControllerAdviceGlobal exception handler across all controllers
Reference 04

HTTP Status Codes

The status codes you will encounter in web development.

2xx — Success

CodeNameMeaning
200OKRequest succeeded, response body has data
201CreatedNew resource created (POST success)
204No ContentSuccess but no response body (DELETE)

3xx — Redirection

CodeNameMeaning
301Moved PermanentlyResource moved forever, update bookmarks
302Found (Redirect)Temporary redirect (sendRedirect() sends this)
304Not ModifiedUse cached version, nothing changed

4xx — Client Error

CodeNameMeaning
400Bad RequestInvalid request data (validation failed)
401UnauthorizedNot authenticated (no login)
403ForbiddenAuthenticated but not allowed (wrong role)
404Not FoundResource does not exist
405Method Not AllowedWrong HTTP method (POST to GET-only endpoint)
409ConflictDuplicate entry (username already exists)

5xx — Server Error

CodeNameMeaning
500Internal Server ErrorUnhandled exception in server code
502Bad GatewayReverse proxy got bad response from backend
503Service UnavailableServer overloaded or in maintenance
Reference 05

web.xml Config Patterns

Common web.xml configurations you need to know.

Servlet Registration

Servlet mapping 
<!-- Register a servlet -->
<servlet>
    <servlet-name>loginServlet</servlet-name>
    <servlet-class>com.nimbus.LoginServlet</servlet-class>
    <load-on-startup>1</load-on-startup>
</servlet>
<servlet-mapping>
    <servlet-name>loginServlet</servlet-name>
    <url-pattern>/login</url-pattern>
</servlet-mapping>

Filter Registration

Filter mapping 
<filter>
    <filter-name>authFilter</filter-name>
    <filter-class>com.nimbus.filter.AuthFilter</filter-class>
</filter>
<filter-mapping>
    <filter-name>authFilter</filter-name>
    <url-pattern>/dashboard/*</url-pattern>
</filter-mapping>

Context Parameters

Global parameters 
<context-param>
    <param-name>appVersion</param-name>
    <param-value>1.0</param-value>
</context-param>

<!-- Read in servlet: getServletContext().getInitParameter("appVersion") -->

Session Configuration

Session timeout 
<session-config>
    <session-timeout>30</session-timeout>  <!-- minutes -->
</session-config>

Error Pages

Custom error pages 
<error-page>
    <error-code>404</error-code>
    <location>/WEB-INF/views/404.jsp</location>
</error-page>
<error-page>
    <error-code>500</error-code>
    <location>/WEB-INF/views/500.jsp</location>
</error-page>
<error-page>
    <exception-type>java.lang.Exception</exception-type>
    <location>/WEB-INF/views/error.jsp</location>
</error-page>

Welcome File

Default page 
<welcome-file-list>
    <welcome-file>index.jsp</welcome-file>
    <welcome-file>index.html</welcome-file>
</welcome-file-list>

<servlet-mapping>
