Section 01

What is Angular?

Angular is a frontend framework by Google for building Single Page Applications (SPAs). Instead of the server rendering HTML (like JSP), Angular runs in the browser and builds the UI dynamically using JavaScript.

SPA vs Traditional Web App

Traditional (Phase 3)
  • Server renders HTML (JSP)
  • Every click = full page reload
  • Server does everything
  • Feels like a website
SPA (Phase 4 — Angular)
  • Browser renders HTML (Angular)
  • Only data is fetched (JSON via REST)
  • Server = API only, no HTML
  • Feels like a desktop app
Analogy

Traditional (JSP): Like ordering food at a restaurant. Every time you want something, the waiter goes to the kitchen, gets the full plate, brings it back. Full round trip each time.

SPA (Angular): Like a buffet. The kitchen loads all the serving stations once. You walk up and pick what you want instantly. The kitchen only restocks specific items (data) when needed.

Component-Based Architecture

Everything in Angular is a component. A component is a reusable piece of UI with its own HTML template, CSS styles, and TypeScript logic. You build an app by composing components like LEGO blocks.

AppComponentRoot (shell)
↓
NavbarComponentNavigation bar
 
RouterOutletPage content area
↓
LoginComponent
 
DashboardComponent
 
BookingComponent
Angular app = tree of components. AppComponent is the root. RouterOutlet swaps pages without full reload.
In Nimbus

Our Phase 3 app had 7 JSP files (login.jsp, register.jsp, dashboard.jsp, etc.). In Phase 4, each becomes an Angular component: LoginComponent, RegisterComponent, DashboardComponent, BookingComponent, TrackingComponent, HistoryComponent, InvoiceComponent. Same features, completely different architecture.

Key Insight

index.html is the only HTML file the browser loads. It contains <app-root></app-root>. Angular takes over from there and dynamically renders all components inside that tag. That's why it's called a Single Page Application — one page, all content swapped by JavaScript.

The Module System

Every Angular app has at least one module — AppModule. A module groups related components, services, and imports together.

src/app/app.module.ts 
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { HttpClientModule } from '@angular/common/http';

import { AppRoutingModule } from './app-routing.module';
import { AppComponent } from './app.component';
import { LoginComponent } from './login/login.component';
import { DashboardComponent } from './dashboard/dashboard.component';

@NgModule({
  declarations: [          // Components belonging to this module
    AppComponent,
    LoginComponent,
    DashboardComponent
  ],
  imports: [               // Other modules this module needs
    BrowserModule,         // Required for browser apps
    FormsModule,           // For ngModel (two-way binding)
    HttpClientModule,      // For HTTP requests to REST API
    AppRoutingModule       // Routes
  ],
  providers: [],           // Services (usually auto-provided)
  bootstrap: [AppComponent] // Root component to start with
})
export class AppModule { }
Section 04

Components

A component = template (HTML) + class (TypeScript) + styles (CSS). The @Component decorator connects them.

Anatomy of a Component

login.component.ts 
import { Component } from '@angular/core';

@Component({
  selector: 'app-login',              // HTML tag name
  templateUrl: './login.component.html', // HTML template
  styleUrls: ['./login.component.css']   // Scoped styles
})
export class LoginComponent {
  // Properties (= the data)
  username: string = '';
  password: string = '';
  errorMessage: string = '';

  // Methods (= the behavior)
  onLogin(): void {
    if (this.username === '' || this.password === '') {
      this.errorMessage = 'Please fill in all fields';
      return;
    }
    // Call auth service (covered in Services chapter)
    console.log('Logging in:', this.username);
  }
}
Line-by-line explanation
1
@Component({ ... })
Decorator that tells Angular "this class is a component." Similar to @Controller in Spring.
2
selector: 'app-login'
Use this component in HTML as <app-login></app-login>. Like a custom HTML tag.
3
templateUrl: './login.component.html'
Points to the HTML file for this component's view. Like a JSP view in Spring MVC.
4
username: string = ''
A typed property. TypeScript enforces this is always a string. This is the "model" data.
5
onLogin(): void
Method that gets called when the login button is clicked (event binding). Return type void = no return value.

Section 05

Data Binding

Data binding connects the component class (TypeScript) to the template (HTML). Angular has four types of binding.

1. Interpolation — {{ }}

Display a property value in the template. One-way: class → template.

Interpolation examples 
<!-- Display property -->
<h2>Welcome, {{ user.fullName }}!</h2>

<!-- Expression -->
<p>Total: Rs. {{ price * 1.18 }}</p>

<!-- Method call -->
<p>Status: {{ getStatusLabel() }}</p>

<!-- Like JSP's ${user.fullName} but with {{ }} -->

2. Property Binding — [property]

Bind a DOM property to a component property. One-way: class → template.

Property binding examples 
<!-- Bind image source -->
<img [src]="parcel.imageUrl">

<!-- Disable button conditionally -->
<button [disabled]="!isFormValid">Submit</button>

<!-- Dynamic CSS class -->
<span [class.delivered]="parcel.status === 'DELIVERED'">
  {{ parcel.status }}
</span>

<!-- Dynamic style -->
<div [style.background-color]="isUrgent ? 'red' : 'green'">
  Priority
</div>

3. Event Binding — (event)

Listen to DOM events and call component methods. One-way: template → class.

Event binding examples 
<!-- Click event -->
<button (click)="onBookParcel()">Book Now</button>

<!-- Form submit -->
<form (ngSubmit)="onLogin()">...</form>

<!-- Keyup event -->
<input (keyup.enter)="onSearch()">

<!-- Pass the event object -->
<input (input)="onType($event)">

4. Two-Way Binding — [(ngModel)]

Combines property + event binding. Changes in the input update the class property AND changes in the class update the input. Requires FormsModule import.

Two-way binding 
<!-- Whatever user types instantly updates this.trackingId -->
<input [(ngModel)]="trackingId" name="trackingId">

<!-- This shows the value in real-time -->
<p>Searching for: {{ trackingId }}</p>

<!-- [(ngModel)] is shorthand for: -->
<input [ngModel]="trackingId"
       (ngModelChange)="trackingId = $event">

Binding Cheat Sheet

SyntaxDirectionJSP Equivalent
{{ value }}Class → View${value} (EL expression)
[property]="expr"Class → ViewNo direct equivalent
(event)="method()"View → Classonclick="..." (JS)
[(ngModel)]="prop"Both waysNo equivalent (JSP has no two-way binding)
Section 06

Directives

Directives are instructions in the template that change how elements are rendered. They are Angular's version of JSTL tags.

*ngIf — Conditional Rendering

*ngIf examples 
<!-- Show only if error exists -->
<div *ngIf="errorMessage" class="error">
  {{ errorMessage }}
</div>

<!-- If/else -->
<div *ngIf="isLoggedIn; else loginBlock">
  Welcome back, {{ user.name }}!
</div>
<ng-template #loginBlock>
  <p>Please log in.</p>
</ng-template>

<!-- JSP equivalent:
     <c:if test="${not empty error}">...</c:if> -->

*ngFor — Loop

*ngFor examples 
<!-- Loop over parcels array -->
<table>
  <tr *ngFor="let parcel of parcels; let i = index">
    <td>{{ i + 1 }}</td>
    <td>{{ parcel.trackingId }}</td>
    <td>{{ parcel.recipientName }}</td>
    <td>{{ parcel.status }}</td>
  </tr>
</table>

<!-- With trackBy for performance -->
<div *ngFor="let p of parcels; trackBy: trackByTrackingId">
  {{ p.trackingId }}
</div>

<!-- JSP equivalent:
     <c:forEach var="p" items="${parcels}" varStatus="s">
       ...
     </c:forEach> -->

*ngSwitch — Multi-Case

*ngSwitch example 
<div [ngSwitch]="parcel.status">
  <span *ngSwitchCase="'BOOKED'" class="badge orange">
    Booked
  </span>
  <span *ngSwitchCase="'IN_TRANSIT'" class="badge blue">
    In Transit
  </span>
  <span *ngSwitchCase="'DELIVERED'" class="badge green">
    Delivered
  </span>
  <span *ngSwitchDefault class="badge gray">
    Unknown
  </span>
</div>

<!-- JSP equivalent:
     <c:choose>
       <c:when test="${status == 'BOOKED'}">...</c:when>
       <c:otherwise>...</c:otherwise>
     </c:choose> -->

Directives vs JSTL

Angular DirectiveJSTL TagPurpose
*ngIf<c:if>Conditional render
*ngIf; else<c:choose>If/else
*ngFor<c:forEach>Loop over list
[ngSwitch]<c:choose>/<c:when>Multi-case
[ngClass]No equivalentDynamic CSS classes
[ngStyle]No equivalentDynamic inline styles
Section 07

Pipes

Pipes transform data in the template. They format dates, numbers, strings, and more — without modifying the original data.

Built-in pipes 
<!-- uppercase / lowercase -->
<p>{{ parcel.status | uppercase }}</p>
<!-- Output: "DELIVERED" -->

<!-- titlecase -->
<p>{{ 'rahul sharma' | titlecase }}</p>
<!-- Output: "Rahul Sharma" -->

<!-- date formatting -->
<p>{{ parcel.bookingDate | date:'dd MMM yyyy' }}</p>
<!-- Output: "20 Apr 2026" -->

<p>{{ parcel.bookingDate | date:'fullDate' }}</p>
<!-- Output: "Monday, April 20, 2026" -->

<!-- currency -->
<p>{{ parcel.price | currency:'INR' }}</p>
<!-- Output: "Rs.237.50" -->

<p>{{ parcel.price | currency:'USD':'symbol':'1.2-2' }}</p>
<!-- Output: "$237.50" -->

<!-- number (decimal) -->
<p>{{ parcel.weight | number:'1.1-2' }}</p>
<!-- Output: "2.50" (min 1 integer, 1-2 decimals) -->

<!-- percent -->
<p>{{ 0.85 | percent }}</p>
<!-- Output: "85%" -->

<!-- json (great for debugging) -->
<pre>{{ parcel | json }}</pre>
<!-- Output: full JSON of the object -->

<!-- Chaining pipes -->
<p>{{ parcel.recipientName | uppercase | slice:0:10 }}</p>
PipeUsageJSTL Equivalent
date{{ d | date:'dd-MMM-yyyy' }}<fmt:formatDate>
currency{{ p | currency:'INR' }}<fmt:formatNumber type="currency">
uppercase{{ s | uppercase }}${fn:toUpperCase(s)}
json{{ obj | json }}No equivalent
Section 08

Component Communication

Components need to talk to each other. Parent components pass data down with @Input. Child components send events up with @Output.

@Input — Parent to Child

parcel-card.component.ts (child) 
import { Component, Input } from '@angular/core';
import { Parcel } from '../models/parcel';

@Component({
  selector: 'app-parcel-card',
  template: `
    <div class="card">
      <h3>{{ parcel.trackingId }}</h3>
      <p>To: {{ parcel.recipientName }}</p>
      <p>Status: {{ parcel.status }}</p>
    </div>
  `
})
export class ParcelCardComponent {
  @Input() parcel!: Parcel;  // Receives data from parent
}
dashboard.component.html (parent) 
<!-- Parent passes each parcel to the child component -->
<app-parcel-card
  *ngFor="let p of parcels"
  [parcel]="p">
</app-parcel-card>

@Output — Child to Parent

parcel-card.component.ts (child emits event) 
import { Component, Input, Output, EventEmitter } from '@angular/core';
import { Parcel } from '../models/parcel';

@Component({
  selector: 'app-parcel-card',
  template: `
    <div class="card">
      <h3>{{ parcel.trackingId }}</h3>
      <button (click)="onTrack()">Track</button>
    </div>
  `
})
export class ParcelCardComponent {
  @Input() parcel!: Parcel;
  @Output() trackClicked = new EventEmitter<string>();

  onTrack(): void {
    // Emit the tracking ID to the parent
    this.trackClicked.emit(this.parcel.trackingId);
  }
}
dashboard.component.html (parent listens) 
<app-parcel-card
  *ngFor="let p of parcels"
  [parcel]="p"
  (trackClicked)="goToTracking($event)">
</app-parcel-card>

<!-- $event contains the tracking ID string emitted by child -->
Parent ComponentDashboardComponent
↓
@Input (data down)
@Output (events up)
↑
Child ComponentParcelCardComponent
Data flows DOWN via @Input. Events flow UP via @Output. Never the other way.

LoginComponent

login/login.component.ts 
import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-login',
  templateUrl: './login.component.html',
  styleUrls: ['./login.component.css']
})
export class LoginComponent {
  username: string = '';
  password: string = '';
  errorMessage: string = '';

  constructor(private router: Router) {}

  onLogin(): void {
    if (!this.username || !this.password) {
      this.errorMessage = 'Please fill in all fields';
      return;
    }

    // In Phase 4, this calls AuthService.login()
    // which makes an HTTP POST to /api/auth/login
    // For now, simple check:
    if (this.username === 'customer1'
        && this.password === 'pass123') {
      localStorage.setItem('user',
        JSON.stringify({ username: this.username,
                         role: 'CUSTOMER' }));
      this.router.navigate(['/dashboard']);
    } else {
      this.errorMessage = 'Invalid username or password';
    }
  }
}

BookingListComponent

booking-list/booking-list.component.ts 
import { Component, OnInit } from '@angular/core';
import { Parcel } from '../models/parcel';

@Component({
  selector: 'app-booking-list',
  templateUrl: './booking-list.component.html',
  styleUrls: ['./booking-list.component.css']
})
export class BookingListComponent implements OnInit {
  parcels: Parcel[] = [];
  filteredParcels: Parcel[] = [];
  searchTerm: string = '';
  filterStatus: string = 'ALL';

  ngOnInit(): void {
    // In Phase 4: this.parcelService.getMyParcels()
    //   .subscribe(data => this.parcels = data);
    this.loadParcels();
  }

  loadParcels(): void {
    // Sample data — replaced by HTTP call later
    this.parcels = [
      {
        trackingId: 'NIM1A2B3C4D',
        recipientName: 'Priya Singh',
        recipientCity: 'Bangalore',
        parcelType: 'MEDIUM',
        weight: 2.5,
        price: 237.5,
        status: 'IN_TRANSIT',
        bookingDate: new Date('2026-04-20')
      } as Parcel,
      {
        trackingId: 'NIMX9Y8Z7W6',
        recipientName: 'Amit Verma',
        recipientCity: 'Mumbai',
        parcelType: 'DOCUMENT',
        weight: 0.5,
        price: 57.5,
        status: 'DELIVERED',
        bookingDate: new Date('2026-04-18')
      } as Parcel
    ];
    this.filteredParcels = [...this.parcels];
  }

  onSearch(): void {
    this.applyFilters();
  }

  onFilterChange(): void {
    this.applyFilters();
  }

  private applyFilters(): void {
    this.filteredParcels = this.parcels.filter(p => {
      const matchesSearch = !this.searchTerm
        || p.trackingId.toLowerCase()
             .includes(this.searchTerm.toLowerCase())
        || p.recipientName.toLowerCase()
             .includes(this.searchTerm.toLowerCase());

      const matchesStatus = this.filterStatus === 'ALL'
        || p.status === this.filterStatus;

      return matchesSearch && matchesStatus;
    });
  }
}

Phase 3 vs Phase 4

Notice the pattern: in Phase 3, filtering meant submitting a form to the server and reloading the whole page. In Angular, onSearch() filters the array instantly in the browser — no server round trip, no page reload. The data is already loaded. This is why SPAs feel faster.

Section 10

Exercises

Build these components to practice Angular fundamentals.

Exercise 1: Greeting Component

Task

Create a GreetingComponent with an input field bound to a name property via [(ngModel)]. Below the input, display "Hello, [name]!" using interpolation. If name is empty, show "Hello, World!".

Use ng g c greeting to generate the component. Don't forget to import FormsModule in AppModule.

Exercise 2: Parcel Status Badge

Task

Create a StatusBadgeComponent that takes a status string via @Input(). Use [ngSwitch] to display different colored badges: green for DELIVERED, blue for IN_TRANSIT, orange for BOOKED. Use it inside a parent component that passes different statuses.

Exercise 3: Parcel Counter

Task

Create a component with an items: string[] array. Display the array using *ngFor. Add an input field and an "Add" button. When clicked, push the input value to the array. Show the total count above the list. Add a "Remove" button next to each item that removes it from the array.

Exercise 4: Price Calculator

Task

Create a component with a weight input (number) and a type dropdown (DOCUMENT, SMALL, MEDIUM, LARGE). Calculate price using the Nimbus formula: base + (weight * 15). Display the price formatted with the currency:'INR' pipe. Price should update in real-time as the user changes weight or type (use two-way binding).

Exercise 5: Parent-Child Communication

Task

Create two components:

  1. ParcelFormComponent (child) — has a form with tracking ID and recipient name fields. On submit, emits the form data to the parent via @Output().
  2. ParcelListComponent (parent) — maintains a parcels array. Listens for the child's event and adds the new parcel to the array. Displays all parcels in a table using *ngFor.

This mirrors the real Nimbus flow: booking form emits data, parent component adds it to the list.
